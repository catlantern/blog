# Linux 5.15 内核同步机制详解

## 一、从一个问题开始：并发访问

### 1.1 假设你管理一个银行账户

```
场景：银行账户并发问题
+===========================================================================+
|                                                                           |
|   账户余额：1000 元                                                       |
|                                                                           |
|   时刻 T1：ATM 机 A 读取余额 = 1000                                       |
|   时刻 T2：ATM 机 B 读取余额 = 1000                                       |
|   时刻 T3：ATM 机 A 存入 100，计算新余额 = 1100                           |
|   时刻 T4：ATM 机 B 取出 200，计算新余额 = 800                            |
|   时刻 T5：ATM 机 A 写入余额 = 1100                                       |
|   时刻 T6：ATM 机 B 写入余额 = 800                                        |
|                                                                           |
|   结果：余额变成 800 元，但应该是 900 元！                                |
|   问题：100 元凭空消失了！                                                |
|                                                                           |
+===========================================================================+
```

这就是**竞态条件（Race Condition）**：多个执行流同时访问共享资源，结果取决于执行顺序。

### 1.2 Linux 内核面临的并发问题

```
Linux 内核并发源：
+===========================================================================+
|                                                                           |
|   1. 多核 CPU（SMP）                                                      |
|   +-------------------+                                                    |
|   | CPU0 和 CPU1 同时执行 |  可能同时访问同一变量                        |
|   +-------------------+                                                    |
|                                                                           |
|   2. 中断                                                                 |
|   +-------------------+                                                    |
|   | 进程执行时中断发生 |  中断处理程序可能访问同一数据                    |
|   +-------------------+                                                    |
|                                                                           |
|   3. 抢占                                                                 |
|   +-------------------+                                                    |
|   | 进程被抢占切换     |  另一个进程可能访问同一数据                    |
|   +-------------------+                                                    |
|                                                                           |
|   4. 软中断/任务队列                                                      |
|   +-------------------+                                                    |
|   | 延迟执行的工作     |  可能与进程并发访问                            |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 1.3 解决方案：同步机制

Linux 提供了多种同步机制，各有特点：

```
同步机制对比：
+===========================================================================+
|                                                                           |
|   机制          适用场景              特点                                |
|   +-------------------+-------------------+-------------------+            |
|   | 原子操作     | 简单计数器       | 最轻量，无锁      |            |
|   +-------------------+-------------------+-------------------+            |
|   | 自旋锁       | 短临界区         | 忙等待，不睡眠    |            |
|   +-------------------+-------------------+-------------------+            |
|   | 互斥锁       | 长临界区         | 可睡眠，开销大    |            |
|   +-------------------+-------------------+-------------------+            |
|   | 读写锁       | 读多写少         | 读读不互斥        |            |
|   +-------------------+-------------------+-------------------+            |
|   | 信号量       | 资源计数         | 可睡眠，计数      |            |
|   +-------------------+-------------------+-------------------+            |
|   | 完成量       | 事件通知         | 一次性的等待      |            |
|   +-------------------+-------------------+-------------------+            |
|                                                                           |
+===========================================================================+
```

## 二、原子操作：最基础的同步

### 2.1 什么是原子操作？

**原子操作**：不可分割的操作，要么全部完成，要么全部不完成，不会被中断。

```
非原子操作 vs 原子操作：
+===========================================================================+
|                                                                           |
|   非原子操作：count++                                                     |
|   +-------------------+                                                    |
|   | 步骤 1：读取 count 值到寄存器                                       |
|   | 步骤 2：寄存器值加 1                                                |
|   | 步骤 3：写回内存                                                    |
|   +-------------------+                                                    |
|   问题：步骤之间可能被中断！                                              |
|                                                                           |
|   原子操作：atomic_inc(&count)                                            |
|   +-------------------+                                                    |
|   | 一条指令完成：读取-修改-写入                                        |
|   | 不会被中断，其他 CPU 无法同时访问                                   |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 2.2 硬件支持：LOCK 前缀

x86 架构通过 **LOCK 前缀**实现原子操作：

```c
/*
 * atomic_add - 原子加法
 * 
 * LOCK_PREFIX 展开为 "lock" 指令前缀
 * 在多核 CPU 上锁定总线，确保操作的原子性
 */
static __always_inline void arch_atomic_add(int i, atomic_t *v)
{
    asm volatile(LOCK_PREFIX "addl %1,%0"
                 : "+m" (v->counter)    /* 输出：内存操作数 */
                 : "ir" (i)             /* 输入：立即数或寄存器 */
                 : "memory");           /* 破坏描述符：内存被修改 */
}
```

```
LOCK 前缀的工作原理：
+===========================================================================+
|                                                                           |
|   单核 CPU：                                                              |
|   +-------------------+                                                    |
|   | LOCK 前缀 = 禁止中断 |  防止指令执行中被中断                        |
|   +-------------------+                                                    |
|                                                                           |
|   多核 CPU：                                                              |
|   +-------------------+                                                    |
|   | LOCK 前缀 = 锁定总线 |  阻止其他 CPU 访问该内存地址                |
|   | 或使用缓存锁定     |  现代 CPU 使用缓存一致性协议优化              |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 2.3 常用原子操作 API

```c

 * 原子变量类型
 */

    int counter;
} atomic_t;

/*
 * 初始化
 */
#define ATOMIC_INIT(i)  { (i) }

/*
 * 基本操作
 */
/* 读取值 */
int atomic_read(atomic_t *v);

/* 设置值 */
void atomic_set(atomic_t *v, int i);

/* 加 i */
void atomic_add(int i, atomic_t *v);

/* 减 i */
void atomic_sub(int i, atomic_t *v);

/* 自增 1 */
void atomic_inc(atomic_t *v);

/* 自减 1 */

/* 自减并测试是否为 0 */
int atomic_dec_and_test(atomic_t *v);  /* 返回 true 表示结果为 0 */

/* 加 i 并测试是否为负 */

/* 自减并返回新值 */

/* 如果 v 的值等于 old，则设置为 new，返回旧值 */
```

### 2.4 比较并交换（CAS）：原子操作的核心

CAS 是实现无锁算法的基础：

```
CAS 操作流程：
+===========================================================================+
|                                                                           |
|   atomic_cmpxchg(v, old, new)                                             |
|                                                                           |
|   步骤 1：原子地读取 v 的值                                               |
|   步骤 2：比较读取的值是否等于 old                                        |
|   步骤 3：如果相等，原子地写入 new                                        |
|   步骤 4：返回 v 的旧值                                                   |
|                                                                           |
|   使用示例：无锁计数器                                                    |
|   +-------------------+                                                    |
|   | void atomic_inc(atomic_t *v) {                                     |
|   |     int old, new;                                                  |
|   |     do {                                                           |
|   |         old = atomic_read(v);    /* 读取当前值 */                  |
|   |         new = old + 1;           /* 计算新值 */                    |
|   |     } while (atomic_cmpxchg(v, old, new) != old);                  |
|   |     /* 如果 CAS 失败，说明有其他线程修改了值，重试 */              |
|   | }                                                                  |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

## 三、自旋锁：忙等待的锁

### 3.1 什么是自旋锁？

**自旋锁**：获取不到锁时，不睡眠，而是循环等待（自旋）。

```
自旋锁工作原理：
+===========================================================================+
|                                                                           |
|   CPU A 尝试获取锁                                                        |
|   +-------------------+                                                    |
|   | 检查锁状态       |                                                    |
|   +-------------------+                                                    |
|           |                                                               |
|           +-----+------------------+                                       |
|           | 未锁定              | 已锁定                                  |
|           v                     v                                         |
|   +-------------------+   +-------------------+                            |
|   | 获取锁，进入临界区 |   | 循环检查锁状态   |                            |
|   +-------------------+   | (自旋等待)        |                            |
|           |               +-------------------+                            |
|           v                       |                                       |
|   +-------------------+           | 锁释放                                |
|   | 执行临界区代码   |           v                                       |
|   +-------------------+   +-------------------+                            |
|           |               | 获取锁，进入临界区 |                            |
|           v               +-------------------+                            |
|   +-------------------+                                                    |
|   | 释放锁           |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 3.2 为什么叫"自旋"？

```
自旋的含义：
+===========================================================================+
|                                                                           |
|   获取锁失败时的行为：                                                    |
|   +-------------------+                                                    |
|   | while (锁被占用) {                                                  |
|   |     cpu_relax();  /* 让 CPU 休息一下，减少功耗 */                   |
|   | }                                                                   |
|   | /* 锁释放后立即获取 */                                             |
|   +-------------------+                                                    |
|                                                                           |
|   就像一个人在门口转圈等待，所以叫"自旋"                                 |
|                                                                           |
+===========================================================================+
```

### 3.3 数据结构

```c
/*
 * 自旋锁结构（简化版）
 * 定义在 include/linux/spinlock_types.h
 */
typedef struct spinlock {
    union {
        struct raw_spinlock rlock;  /* 底层锁实现 */
    };
} spinlock_t;

/*
 * 底层锁结构（x86 架构）
 * 使用队列自旋锁（qspinlock）
 */
typedef struct qspinlock {
    atomic_t val;  /* 锁状态：0=未锁定，其他=锁定 */
} arch_spinlock_t;
```

### 3.4 基本使用

```c
#include <linux/spinlock.h>

/* 定义并初始化自旋锁 */
spinlock_t my_lock;

void my_function(void)
{
    /* 初始化锁 */
    spin_lock_init(&my_lock);
    
    /* 获取锁 */
    spin_lock(&my_lock);
    
    /* 临界区：访问共享资源 */
    /* ... */
    
    /* 释放锁 */
    spin_unlock(&my_lock);
}
```

### 3.5 自旋锁变体

```
自旋锁变体：
+===========================================================================+
|                                                                           |
|   1. 基本自旋锁                                                           |
|   +-------------------+                                                    |
|   | spin_lock()      |  获取锁                                          |
|   | spin_unlock()    |  释放锁                                          |
|   +-------------------+                                                    |
|   适用：临界区不涉及中断                                                  |
|                                                                           |
|   2. 禁用中断的自旋锁                                                     |
|   +-------------------+                                                    |
|   | spin_lock_irq()  |  获取锁并禁用本地 CPU 中断                       |
|   | spin_unlock_irq()|  释放锁并恢复中断                                |
|   +-------------------+                                                    |
|   适用：临界区可能被中断处理程序访问                                      |
|                                                                           |
|   3. 保存中断状态的自旋锁                                                 |
|   +-------------------+                                                    |
|   | spin_lock_irqsave() |  保存中断状态，获取锁，禁用中断              |
|   | spin_unlock_irqrestore() |  释放锁，恢复中断状态                   |
|   +-------------------+                                                    |
|   适用：不确定当前中断状态                                                |
|                                                                           |
|   4. 禁用底半部的自旋锁                                                   |
|   +-------------------+                                                    |
|   | spin_lock_bh()   |  获取锁并禁用软中断                              |
|   | spin_unlock_bh() |  释放锁并恢复软中断                              |
|   +-------------------+                                                    |
|   适用：临界区可能被软中断访问                                            |
|                                                                           |
+===========================================================================+
```

### 3.6 自旋锁的实现（简化版）

```c
/*
 * 自旋锁获取（简化版）
 */
static __always_inline void spin_lock(spinlock_t *lock)
{
    /*
     * 禁用内核抢占
     * 因为持有自旋锁时不能睡眠，也不能被调度出去
     */
    preempt_disable();
    
    /*
     * 尝试获取底层锁
     * 如果锁已被占用，则自旋等待
     */
    __raw_spin_lock(&lock->rlock);
}

/*
 * 底层锁获取（x86 架构简化版）
 */
static __always_inline void __raw_spin_lock(raw_spinlock_t *lock)
{
    /*
     * 尝试快速获取锁
     * 如果锁是空闲的（值为 0），原子地设置为 1
     */
    if (likely(atomic_cmpxchg(&lock->val, 0, 1) == 0)) {
        return;  /* 获取成功 */
    }
    
    /*
     * 快速获取失败，进入慢速路径
     * 在慢速路径中自旋等待
     */
    __raw_spin_lock_slowpath(lock);
}

/*
 * 慢速路径：自旋等待
 */
void __raw_spin_lock_slowpath(raw_spinlock_t *lock)
{
    while (1) {
        /*
         * 自旋等待锁变为空闲
         * 只读取，不修改，减少总线争用
         */
        while (atomic_read(&lock->val) != 0) {
            cpu_relax();  /* 让 CPU 休息，减少功耗 */
        }
        
        /*
         * 锁看起来空闲了，尝试获取
         * 注意：这里可能有竞争，所以要用原子操作
         */
        if (atomic_cmpxchg(&lock->val, 0, 1) == 0) {
            return;  /* 获取成功 */
        }
        /* 获取失败，继续自旋 */
    }
}
```

### 3.7 自旋锁使用规则

```
自旋锁使用规则：
+===========================================================================+
|                                                                           |
|   ✗ 禁止：在持有自旋锁时睡眠                                              |
|   +-------------------+                                                    |
|   | 原因：睡眠会触发调度，其他 CPU 可能永远等不到锁释放                  |
|   | 后果：系统死锁或崩溃                                                |
|   +-------------------+                                                    |
|                                                                           |
|   ✗ 禁止：在持有自旋锁时调用可能睡眠的函数                                |
|   +-------------------+                                                    |
|   | 例如：kmalloc(GFP_KERNEL), copy_from_user()                         |
|   +-------------------+                                                    |
|                                                                           |
|   ✓ 建议：临界区尽量短                                                    |
|   +-------------------+                                                    |
|   | 原因：自旋等待浪费 CPU 时间                                         |
|   +-------------------+                                                    |
|                                                                           |
|   ✓ 建议：临界区只包含必要的操作                                          |
|   +-------------------+                                                    |
|   | 不要在临界区做复杂计算、I/O 操作                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

## 四、互斥锁：可睡眠的锁

### 4.1 什么是互斥锁？

**互斥锁（Mutex）**：获取不到锁时，当前线程睡眠，让出 CPU。

```
互斥锁 vs 自旋锁：
+===========================================================================+
|                                                                           |
|   自旋锁：                                                                |
|   +-------------------+                                                    |
|   | 获取失败 → 循环等待 |  CPU 忙等待，消耗 CPU 时间                    |
|   | 适用：等待时间很短 |  临界区执行时间 < 两次上下文切换时间           |
|   +-------------------+                                                    |
|                                                                           |
|   互斥锁：                                                                |
|   +-------------------+                                                    |
|   | 获取失败 → 睡眠等待 |  让出 CPU，不消耗 CPU 时间                    |
|   | 适用：等待时间较长 |  临界区执行时间较长                            |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 4.2 数据结构

```c
/*
 * 互斥锁结构
 * 定义在 include/linux/mutex.h
 */
struct mutex {
    atomic_long_t       owner;      /* 锁持有者的 task_struct 指针 */
    raw_spinlock_t      wait_lock;  /* 保护等待队列的自旋锁 */
    struct list_head    wait_list;  /* 等待队列 */
#ifdef CONFIG_MUTEX_SPIN_ON_OWNER
    struct optimistic_spin_queue osq;  /* 乐观自旋队列 */
#endif
};
```

```
互斥锁内部结构：
+===========================================================================+
|                                                                           |
|   mutex                                                                   |
|   +-------------------------------------------------------------------+   |
|   | owner = NULL (未锁定) 或 task_struct* (锁定)                      |   |
|   +-------------------------------------------------------------------+   |
|   | wait_lock (自旋锁，保护 wait_list)                                |   |
|   +-------------------------------------------------------------------+   |
|   | wait_list → [waiter1] → [waiter2] → [waiter3] → NULL              |   |
|   +-------------------------------------------------------------------+   |
|                                                                           |
|   waiter 结构：                                                           |
|   +-------------------------------------------------------------------+   |
|   | struct mutex_waiter {                                              |   |
|   |     struct list_head list;    /* 链入 wait_list */                |   |
|   |     struct task_struct *task; /* 等待的进程 */                    |   |
|   | };                                                                 |   |
|   +-------------------------------------------------------------------+   |
|                                                                           |
+===========================================================================+
```

### 4.3 基本使用

```c
#include <linux/mutex.h>

/* 静态定义并初始化 */
DEFINE_MUTEX(my_mutex);

/* 或者动态初始化 */
struct mutex my_mutex;
mutex_init(&my_mutex);

void my_function(void)
{
    /* 获取锁（可能睡眠） */
    mutex_lock(&my_mutex);
    
    /* 临界区：可以睡眠，可以执行较长时间的操作 */
    /* ... */
    
    /* 释放锁 */
    mutex_unlock(&my_mutex);
}
```

### 4.4 互斥锁获取流程

```
mutex_lock() 流程：
+===========================================================================+
|                                                                           |
|   mutex_lock(&lock)                                                       |
|           |                                                               |
|           v                                                               |
|   +-------------------+                                                    |
|   | 快速路径：尝试原子获取 |                                              |
|   | owner == NULL ?   |                                                    |
|   +-------------------+                                                    |
|           |                                                               |
|           +-----+------------------+                                       |
|           | 是                  | 否                                      |
|           v                     v                                         |
|   +-------------------+   +-------------------+                            |
|   | 原子设置 owner    |   | 慢速路径         |                            |
|   | 获取成功          |   | __mutex_lock_slowpath |                        |
|   +-------------------+   +-------------------+                            |
|                                   |                                       |
|                                   v                                       |
|                           +-------------------+                            |
|                           | 乐观自旋（可选） |                            |
|                           | 等待 owner 释放  |                            |
|                           +-------------------+                            |
|                                   |                                       |
|                                   +-----+------------------+               |
|                                   | owner 释放        | 超时              |
|                                   v                   v                   |
|                           +-------------------+   +-------------------+    |
|                           | 尝试获取          |   | 加入等待队列     |    |
|                           +-------------------+   | 睡眠等待         |    |
|                                   |               +-------------------+    |
|                                   |                       |               |
|                                   +-----------------------+               |
|                                   |                                       |
|                                   v                                       |
|                           +-------------------+                            |
|                           | 被唤醒，获取锁    |                            |
|                           +-------------------+                            |
|                                                                           |
+===========================================================================+
```

### 4.5 互斥锁实现（简化版）

```c
/*
 * mutex_lock - 获取互斥锁
 */
void __sched mutex_lock(struct mutex *lock)
{
    might_sleep();  /* 检查是否在原子上下文，警告非法睡眠 */
    
    /*
     * 快速路径：尝试无竞争获取
     * 如果 owner 为 NULL，原子地设置为当前进程
     */
    if (!__mutex_trylock_fast(lock))
        __mutex_lock_slowpath(lock);  /* 快速路径失败，走慢速路径 */
}

/*
 * 快速路径尝试获取
 */
static inline bool __mutex_trylock_fast(struct mutex *lock)
{
    unsigned long curr = (unsigned long)current;  /* 当前进程指针 */
    unsigned long zero = 0UL;
    
    /*
     * 原子比较并交换
     * 如果 owner == 0，设置为当前进程
     */
    return atomic_long_try_cmpxchg_acquire(&lock->owner, &zero, curr);
}

/*
 * 慢速路径
 */
static noinline void __sched __mutex_lock_slowpath(struct mutex *lock)
{
    struct mutex_waiter waiter;
    
    /* 初始化等待者 */
    INIT_LIST_HEAD(&waiter.list);
    waiter.task = current;
    
    /* 获取保护等待队列的自旋锁 */
    raw_spin_lock(&lock->wait_lock);
    
    /* 再次尝试获取锁 */
    if (__mutex_trylock(lock)) {
        raw_spin_unlock(&lock->wait_lock);
        return;
    }
    
    /* 加入等待队列 */
    list_add_tail(&waiter.list, &lock->wait_list);
    
    /* 循环等待 */
    while (1) {
        /* 设置进程状态为不可中断睡眠 */
        set_current_state(TASK_UNINTERRUPTIBLE);
        
        /* 释放自旋锁 */
        raw_spin_unlock(&lock->wait_lock);
        
        /* 调度出去，让出 CPU */
        schedule();  /* 睡眠等待 */
        
        /* 被唤醒后，重新获取自旋锁 */
        raw_spin_lock(&lock->wait_lock);
        
        /* 尝试获取锁 */
        if (__mutex_trylock(lock)) {
            /* 从等待队列移除 */
            list_del(&waiter.list);
            raw_spin_unlock(&lock->wait_lock);
            return;
        }
    }
}

/*
 * mutex_unlock - 释放互斥锁
 */
void __sched mutex_unlock(struct mutex *lock)
{
    /*
     * 快速路径：如果没有等待者，直接释放
     */
    if (__mutex_unlock_fast(lock))
        return;
    
    __mutex_unlock_slowpath(lock, _RET_IP_);
}

/*
 * 慢速释放路径：唤醒等待者
 */
static void __mutex_unlock_slowpath(struct mutex *lock, unsigned long ip)
{
    struct mutex_waiter *waiter;
    
    raw_spin_lock(&lock->wait_lock);
    
    /* 清除 owner */
    atomic_long_set(&lock->owner, 0);
    
    /* 如果有等待者，唤醒第一个 */
    if (!list_empty(&lock->wait_list)) {
        waiter = list_first_entry(&lock->wait_list, struct mutex_waiter, list);
        
        /* 从等待队列移除 */
        list_del(&waiter.list);
        
        /* 唤醒等待的进程 */
        wake_up_process(waiter.task);
    }
    
    raw_spin_unlock(&lock->wait_lock);
}
```

### 4.6 乐观自旋优化

现代 Linux 内核对互斥锁做了优化：在睡眠前先自旋等待一小段时间。

```
乐观自旋优化：
+===========================================================================+
|                                                                           |
|   问题：如果锁持有者很快就会释放，睡眠/唤醒的开销太大                    |
|                                                                           |
|   解决：在睡眠前先自旋等待一段时间                                        |
|   +-------------------+                                                    |
|   | 检查锁持有者是否在运行 |  如果是，说明很快会释放                    |
|   | 如果是，自旋等待   |  避免睡眠开销                                  |
|   | 如果不是，才睡眠   |  锁持有者被调度走了，等待无意义                |
|   +-------------------+                                                    |
|                                                                           |
|   优点：减少上下文切换开销                                                |
|   缺点：可能浪费一些 CPU 时间                                             |
|                                                                           |
+===========================================================================+
```

## 五、读写锁：读读不互斥

### 5.1 为什么需要读写锁？

```
问题场景：
+===========================================================================+
|                                                                           |
|   共享数据结构：链表                                                      |
|   +-------------------+                                                    |
|   | 读操作：遍历链表 |  多个读者可以同时进行，互不影响                  |
|   | 写操作：修改链表 |  写者需要独占访问                                |
|   +-------------------+                                                    |
|                                                                           |
|   如果用普通锁：                                                          |
|   +-------------------+                                                    |
|   | 读者 A 获取锁     |                                                    |
|   | 读者 B 等待       |  不必要！读者之间不冲突                          |
|   | 读者 C 等待       |  性能浪费                                        |
|   +-------------------+                                                    |
|                                                                           |
|   使用读写锁：                                                            |
|   +-------------------+                                                    |
|   | 读者 A 获取读锁   |                                                    |
|   | 读者 B 获取读锁   |  允许！多个读者可以同时持有                      |
|   | 读者 C 获取读锁   |  允许！                                          |
|   | 写者 D 等待       |  等待所有读者释放                                |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 5.2 数据结构

```c

 * 读写信号量结构

    atomic_long_t owner;         /* 写者或读者 */

    struct optimistic_spin_queue osq;  /* 乐观自旋队列 */
#endif
    raw_spinlock_t wait_lock;    /* 保护等待队列 */
    struct list_head wait_list;  /* 等待队列 */
};
```

```
count 字段的含义：
+===========================================================================+
|                                                                           |
|   count 值          含义                                                  |
|   +-------------------+-------------------+                                |
|   | 0              | 未锁定                                            |
|   +-------------------+-------------------+                                |
|   | 正数 (1, 2, ...) | 有 N 个读者持有锁                                 |
|   +-------------------+-------------------+                                |
|   | 负数            | 有写者持有锁或等待                                |
|   +-------------------+-------------------+                                |
|                                                                           |
+===========================================================================+
```

### 5.4 读写锁规则

```
读写锁规则：
+===========================================================================+
|                                                                           |
|   +-------------------+-------------------+-------------------+            |
|   | 当前状态         | 读者请求         | 写者请求         |            |
|   +-------------------+-------------------+-------------------+            |
```

## 六、信号量：可计数的锁

### 6.1 什么是信号量？

**信号量**：一个计数器，控制同时访问资源的线程数量。

```
信号量 vs 互斥锁：
+===========================================================================+
|                                                                           |
|   互斥锁：                                                                |
|   +-------------------+                                                    |
|   | 计数 = 0 或 1     |  只允许一个线程访问                              |
|   | 二元信号量        |  特殊情况：计数为 1 的信号量                    |
|   +-------------------+                                                    |
|                                                                           |
|   信号量：                                                                |
|   +-------------------+                                                    |
|   | 计数 = 任意值     |  允许 N 个线程同时访问                          |
|   | 资源计数器        |  用于资源管理                                   |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 6.2 数据结构

```c
/*
 * 信号量结构
 * 定义在 include/linux/semaphore.h
 */
struct semaphore {
    raw_spinlock_t  lock;       /* 保护信号量的自旋锁 */
    unsigned int    count;      /* 计数器 */
    struct list_head wait_list; /* 等待队列 */
};
```

### 6.3 基本使用

```c
#include <linux/semaphore.h>

/* 定义并初始化（计数为 1，相当于互斥锁） */
DEFINE_SEMAPHORE(my_sem);

/* 定义并初始化（计数为 N） */
struct semaphore my_sem;
sema_init(&my_sem, 3);  /* 允许 3 个线程同时访问 */

/* 获取信号量（P 操作） */
void get_resource(void)
{
    down(&my_sem);  /* 计数减 1，如果为负则睡眠 */
    
    /* 访问资源 */
}

/* 释放信号量（V 操作） */
void put_resource(void)
{
    up(&my_sem);  /* 计数加 1，唤醒等待者 */
}
```

### 6.5 信号量实现（简化版）

```c
/*
 * down - 获取信号量
 */
void down(struct semaphore *sem)
{
    unsigned long flags;
    
    might_sleep();
    
    /* 获取保护信号量的自旋锁 */
    raw_spin_lock_irqsave(&sem->lock, flags);
    
    if (likely(sem->count > 0)) {
        /* 计数 > 0，直接获取 */
        sem->count--;
    } else {
        /* 计数 = 0，需要睡眠等待 */
        __down(sem);
    }
    
    raw_spin_unlock_irqrestore(&sem->lock, flags);
}

/*
 * __down - 睡眠等待
 */
static void __down(struct semaphore *sem)
{
    struct semaphore_waiter waiter;
    
    /* 初始化等待者 */
    list_add_tail(&waiter.list, &sem->wait_list);
    waiter.task = current;
    waiter.up = false;
    
    while (1) {
        set_current_state(TASK_UNINTERRUPTIBLE);
        raw_spin_unlock_irq(&sem->lock);
        
        schedule();  /* 睡眠 */
        
        raw_spin_lock_irq(&sem->lock);
        
        if (waiter.up)  /* 被唤醒 */
            return;
    }
}

/*
 * up - 释放信号量
 */
void up(struct semaphore *sem)
{
    unsigned long flags;
    
    raw_spin_lock_irqsave(&sem->lock, flags);
    
    if (list_empty(&sem->wait_list)) {
        /* 没有等待者，计数加 1 */
        sem->count++;
    } else {
        /* 有等待者，唤醒第一个 */
        __up(sem);
    }
    
    raw_spin_unlock_irqrestore(&sem->lock, flags);
}

/*
 * __up - 唤醒等待者
 */
static void __up(struct semaphore *sem)
{
    struct semaphore_waiter *waiter;
    
    waiter = list_first_entry(&sem->wait_list, struct semaphore_waiter, list);
    list_del(&waiter.list);
    waiter->up = true;
    wake_up_process(waiter->task);
}
```

### 6.6 信号量应用场景

```
信号量典型应用：
+===========================================================================+
|                                                                           |
|   场景 1：资源池管理                                                      |
|   +-------------------+                                                    |
|   | 有 N 个数据库连接 |  信号量计数 = N                                  |
|   | 每个线程获取一个 |  down() 获取连接                                  |
|   | 用完放回         |  up() 释放连接                                   |
|   +-------------------+                                                    |
|                                                                           |
|   场景 2：生产者-消费者                                                   |
|   +-------------------+                                                    |
|   | 有界缓冲区       |  空槽信号量 + 满槽信号量                        |
|   | 生产者：down(空) |  获取空槽，放入数据                              |
|   | 消费者：down(满) |  获取满槽，取出数据                              |
|   +-------------------+                                                    |
|                                                                           |
|   场景 3：限流                                                            |
|   +-------------------+                                                    |
|   | 限制并发访问数   |  信号量计数 = 最大并发数                        |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

## 七、完成量：事件通知机制

### 7.1 什么是完成量？

**完成量**：用于一个执行流通知另一个执行流某事已完成。

```
完成量 vs 信号量：
+===========================================================================+
|                                                                           |
|   信号量：                                                                |
|   +-------------------+                                                    |
|   | 可重复使用       |  up/down 可以多次调用                          |
|   | 计数器           |  管理资源数量                                   |
|   +-------------------+                                                    |
|                                                                           |
|   完成量：                                                                |
|   +-------------------+                                                    |
|   | 一次性使用       |  complete 后必须 reinit 才能重用               |
|   | 事件通知         |  等待特定事件完成                               |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 7.2 数据结构

```c
/*
 * 完成量结构
 * 定义在 include/linux/completion.h
 */
struct completion {
    unsigned int done;           /* 完成标志 */
    struct swait_queue_head wait; /* 等待队列 */
};
```

### 7.3 基本使用

```c
#include <linux/completion.h>

/* 定义并初始化 */
DECLARE_COMPLETION(my_comp);

/* 或者动态初始化 */
struct completion my_comp;
init_completion(&my_comp);

/* 等待完成 */
void wait_for_event(void)
{
    /* 等待完成，不可中断 */
    wait_for_completion(&my_comp);
    
    /* 或者可中断等待 */
    if (wait_for_completion_interruptible(&my_comp)) {
        /* 被信号中断 */
        return -ERESTARTSYS;
    }
    
    /* 或者带超时等待 */
    if (!wait_for_completion_timeout(&my_comp, timeout)) {
        /* 超时 */
        return -ETIME;
    }
}

/* 通知完成 */
void notify_event(void)
{
    complete(&my_comp);  /* 唤醒一个等待者 */
    // 或
    complete_all(&my_comp);  /* 唤醒所有等待者 */
}
```

### 7.4 完成量实现（简化版）

```c
/*
 * wait_for_completion - 等待完成
 */
void wait_for_completion(struct completion *x)
{
    might_sleep();
    
    /* 如果已完成，直接返回 */
    if (x->done)
        return;
    
    /* 否则等待 */
    __wait_for_common(x, schedule, TASK_UNINTERRUPTIBLE, MAX_SCHEDULE_TIMEOUT);
}

/*
 * complete - 通知完成
 */
void complete(struct completion *x)
{
    unsigned long flags;
    
    raw_spin_lock_irqsave(&x->wait.lock, flags);
    
    /* 设置完成标志 */
    x->done++;
    
    /* 唤醒一个等待者 */
    swake_up_locked(&x->wait);
    
    raw_spin_unlock_irqrestore(&x->wait.lock, flags);
}

/*
 * complete_all - 唤醒所有等待者
 */
void complete_all(struct completion *x)
{
    unsigned long flags;
    
    raw_spin_lock_irqsave(&x->wait.lock, flags);
    
    /* 设置完成标志为很大的值 */
    x->done += UINT_MAX/2;
    
    /* 唤醒所有等待者 */
    swake_up_all_locked(&x->wait);
    
    raw_spin_unlock_irqrestore(&x->wait.lock, flags);
}
```

## 八、如何选择同步机制

### 8.1 选择流程图

```
同步机制选择流程：
+===========================================================================+
|                                                                           |
|   需要同步吗？                                                            |
|       |                                                                   |
|       +-----+------------------+                                           |
|       | 否                  | 是                                          |
|       v                     v                                             |
|   +--------+          +-------------------+                                |
|   | 不需要 |          | 操作是否简单？   |                                |
|   +--------+          +-------------------+                                |
|                             |                                             |
|                             +-----+------------------+                     |
|                             | 是（计数器/位操作）| 否                      |
|                             v                    v                         |
|                     +-------------------+   +-------------------+          |
|                     | 原子操作         |   | 临界区是否很短？ |          |
|                     +-------------------+   +-------------------+          |
|                                                 |                         |
|                                                 +-----+------------------+ |
|                                                 | 是                  | 否 |
|                                                 v                    v   |
|                                         +-------------------+  +--------+  |
|                                         | 是否可睡眠？     |  | 互斥锁 |  |
|                                         +-------------------+  +--------+  |
|                                             |                              |
|                                             +-----+------------------+     |
|                                             | 否                  | 是    |
|                                             v                    v         |
|                                     +-------------------+  +--------+      |
|                                     | 自旋锁           |  | 互斥锁 |      |
|                                     +-------------------+  +--------+      |
|                                                                           |
|   特殊情况：                                                              |
|   - 读多写少 → 读写锁                                                     |

|   - 事件通知 → 完成量                                                     |
```

### 8.2 对比总结

| 机制 | 等待方式 | 可睡眠 | 适用场景 | 开销 |
|------|----------|--------|----------|------|
| 原子操作 | 无等待 | 否 | 简单计数器 | 最低 |
| 自旋锁 | 忙等待 | 否 | 短临界区、中断上下文 | 低 |
| 互斥锁 | 睡眠 | 是 | 长临界区、可睡眠上下文 | 中 |
| 读写锁 | 睡眠 | 是 | 读多写少 | 中 |
| 信号量 | 睡眠 | 是 | 资源计数 | 中 |
| 完成量 | 睡眠 | 是 | 事件通知 | 中 |

### 8.3 使用示例

```c

#include <linux/atomic.h>

/* 场景 1：简单计数器 - 使用原子操作 */

{
    atomic_inc(&counter);
}

/* 场景 2：短临界区、中断上下文 - 使用自旋锁 */
static spinlock_t my_spinlock;
static int shared_data;

void update_shared_data(int val)
{
    unsigned long flags;
    
    spin_lock_irqsave(&my_spinlock, flags);
    shared_data = val;
    spin_unlock_irqrestore(&my_spinlock, flags);

/* 场景 3：长临界区、可睡眠 - 使用互斥锁 */
static DEFINE_MUTEX(my_mutex);
static struct list_head my_list;

void add_to_list(struct my_struct *item)
{
    mutex_lock(&my_mutex);
    list_add(&item->list, &my_list);
    /* 可以在这里做复杂操作，甚至睡眠 */
    mutex_unlock(&my_mutex);
}

/* 场景 4：读多写少 - 使用读写锁 */
static DECLARE_RWSEM(my_rwsem);
static struct config my_config;

void read_config(void)
{
    down_read(&my_rwsem);
    /* 读取配置 */
    up_read(&my_rwsem);
}

void write_config(struct config *new_config)
{
    down_write(&my_rwsem);
    /* 更新配置 */
    up_write(&my_rwsem);
}

/* 场景 5：资源池 - 使用信号量 */
static DEFINE_SEMAPHORE(connection_pool);  /* 假设有 1 个连接 */

void use_connection(void)
{
    down(&connection_pool);
    /* 使用连接 */
    up(&connection_pool);
}

/* 场景 6：事件通知 - 使用完成量 */
static DECLARE_COMPLETION(init_done);

int init_thread(void *data)
{
    /* 执行初始化 */
    complete(&init_done);
    return 0;
}

void start_init(void)
{
    kthread_run(init_thread, NULL, "init");
    wait_for_completion(&init_done);
}
```

## 九、总结

### 9.1 核心概念

```
同步机制核心概念：
+===========================================================================+
|                                                                           |
|   1. 原子操作                                                             |
|   +-------------------+                                                    |
|   | 核心：硬件级原子性 |  LOCK 前缀、CAS 指令                            |
|   | 用途：简单计数器   |  引用计数、标志位                              |
|   +-------------------+                                                    |
|                                                                           |
|   2. 自旋锁                                                               |
|   +-------------------+                                                    |
|   | 核心：忙等待       |  获取不到锁时循环检查                          |
|   | 用途：短临界区     |  中断上下文、不可睡眠                          |
|   +-------------------+                                                    |
|                                                                           |
|   3. 互斥锁                                                               |
|   +-------------------+                                                    |
|   | 核心：睡眠等待     |  获取不到锁时让出 CPU                          |
|   | 用途：长临界区     |  可睡眠上下文                                  |
|   +-------------------+                                                    |
|                                                                           |
|   4. 读写锁                                                               |
|   +-------------------+                                                    |
|   | 核心：读读不互斥   |  多读者可同时访问                              |
|   | 用途：读多写少     |  配置数据、缓存                                 |
|   +-------------------+                                                    |
|                                                                           |
|   5. 信号量                                                               |
|   +-------------------+                                                    |
|   | 核心：计数器       |  控制并发访问数量                              |
|   | 用途：资源管理     |  连接池、限流                                  |
|   +-------------------+                                                    |
|                                                                           |
|   6. 完成量                                                               |
|   +-------------------+                                                    |
|   | 核心：事件通知     |  一次性的等待/通知                             |
|   | 用途：同步点       |  驱动初始化、线程同步                          |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 9.2 使用原则

1. **选择最轻量的机制**：能用原子操作就不用锁
2. **自旋锁临界区要短**：避免浪费 CPU 时间
3. **互斥锁临界区可以长**：但不要持有太久
4. **避免死锁**：注意锁的获取顺序
5. **注意上下文**：中断上下文只能用自旋锁

Linux 内核提供了丰富的同步机制，正确选择和使用它们是编写高质量内核代码的关键。

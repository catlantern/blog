## 1. 什么是进程上下文切换

### 1.1 从生活场景理解

想象你在看书（进程 A），突然接到一个电话（中断/调度），你需要：

```
看书 → 接电话 → 继续看书

你需要做什么？
+-------------------+
| 1. 记住看到哪一页 |  <-- 保存程序计数器（EIP/RIP）
+-------------------+
         |
         v
+-------------------+
| 2. 记住书签位置   |  <-- 保存栈指针（ESP/RSP）
+-------------------+
         |
         v
+-------------------+
| 3. 记住思考的内容 |  <-- 保存寄存器
+-------------------+
         |
         v
+-------------------+
| 4. 放下书去接电话 |  <-- 切换到新进程
+-------------------+
```

**进程上下文切换**就是：保存当前进程的"工作状态"，恢复另一个进程的"工作状态"，让 CPU 能够继续执行另一个进程。

### 1.2 CPU 上下文 vs 进程上下文

```
CPU 上下文（硬件层面）：
+===========================================================================+
|                                                                           |
|   +------------------+                                                    |
|   | CPU 寄存器       |                                                    |
|   | - 通用寄存器     |  eax, ebx, ecx, edx, esi, edi, ebp, esp (x86)     |
|   |                  |  rax, rbx, ... r15 (x86_64)                       |
|   | - 段寄存器       |  cs, ds, es, fs, gs, ss                          |
|   | - 标志寄存器     |  eflags/rflags                                   |
|   | - 指令指针       |  eip/rip                                         |
|   +------------------+                                                    |
|                                                                           |
|   +------------------+                                                    |
| | 程序计数器 (PC)   |  下一条要执行的指令地址                          |
|   +------------------+                                                    |
|                                                                           |
+===========================================================================+

进程上下文（软件层面）：
+===========================================================================+
|                                                                           |
|   +------------------+                                                    |
|   | CPU 上下文       |  上面所有内容                                    |
|   +------------------+                                                    |
|                                                                           |
|   +------------------+                                                    |
|   | 内存管理信息     |  页表基址（CR3）、地址空间                       |
|   +------------------+                                                    |
|                                                                           |
|   +------------------+                                                    |
|   | 内核栈           |  每个进程独立的内核栈                            |
|   +------------------+                                                    |
|                                                                           |
|   +------------------+                                                    |
|   | 浮点状态         |  FPU/SSE/AVX 寄存器                             |
|   +------------------+                                                    |
|                                                                           |
|   +------------------+                                                    |
|   | 其他状态         |  TLS、调试寄存器等                              |
|   +------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 1.3 上下文切换的定义

```
上下文切换 = 保存当前进程的所有状态 + 恢复目标进程的所有状态

具体来说：
+===========================================================================+
|                                                                           |
|   进程 A 运行中                                                           |
|       |                                                                   |
|       v                                                                   |
|   +-------------------+                                                    |
|   | 触发调度          |  时间片用完、系统调用、中断等                    |
|   +-------------------+                                                    |
|       |                                                                   |
|       v                                                                   |
|   +-------------------+                                                    |
|   | 保存进程 A 的状态 |                                                    |
|   | - 寄存器          |                                                    |
|   | - 栈指针          |                                                    |
|   | - 页表基址        |                                                    |
|   | - 浮点状态        |                                                    |
|   +-------------------+                                                    |
|       |                                                                   |
|       v                                                                   |
|   +-------------------+                                                    |
|   | 恢复进程 B 的状态 |                                                    |
|   | - 寄存器          |                                                    |
|   | - 栈指针          |                                                    |
|   | - 页表基址        |                                                    |
|   | - 浮点状态        |                                                    |
|   +-------------------+                                                    |
|       |                                                                   |
|       v                                                                   |
|   进程 B 开始运行                                                         |
|                                                                           |
+===========================================================================+
```

## 2. 为什么需要上下文切换

### 2.1 多任务的需求

```
单任务系统：
+===========================================================================+
|                                                                           |
|   时间轴：                                                                |
|   |-------|-------|-------|-------|                                       |
|   | 进程A | 空闲  | 进程A | 空闲  |                                       |
|   |-------|-------|-------|-------|                                       |
|                                                                           |
|   问题：进程 A 等待 I/O 时，CPU 空闲，资源浪费                            |
|                                                                           |
+===========================================================================+

多任务系统：
+===========================================================================+
|                                                                           |
|   时间轴：                                                                |
|   |-------|-------|-------|-------|-------|-------|                       |
|   | 进程A | 进程B | 进程C | 进程A | 进程B | 进程C |                       |
|   |-------|-------|-------|-------|-------|-------|                       |
|                                                                           |
|   优点：CPU 利用率高，多个进程"同时"运行                                 |
|   需要：上下文切换来在进程间切换                                          |
|                                                                           |
+===========================================================================+
```

### 2.2 为什么不直接保存所有内容？

```
问题：为什么不每次切换都保存所有内容？

答案：性能考虑

+-------------------+
| 保存所有寄存器    |  约 100+ 字节
+-------------------+
         |
         v
+-------------------+
| 保存所有 FPU      |  约 512-8000 字节
+-------------------+
         |
         v
+-------------------+
| 保存所有内存      |  可能几 GB！
+-------------------+

解决方案：
1. 只保存必要的寄存器
2. 懒惰 FPU 切换
3. 不保存用户内存（页表切换即可）
```

### 2.3 设计目标

```
上下文切换的设计目标：

1. 正确性
   +-------------------+
   | 切换后进程能正确  |
   | 从断点继续执行    |
   +-------------------+

2. 高效性
   +-------------------+
   | 尽量减少切换开销  |
   | 只保存必要状态    |
   +-------------------+

3. 可扩展性
   +-------------------+
   | 支持多核处理器    |
   | 支持多种硬件特性  |
   +-------------------+

4. 安全性
   +-------------------+
   | 进程间隔离        |
   | 防止信息泄露      |
   +-------------------+
```

## 3. 进程上下文的组成

### 3.1 通用寄存器

```
x86_64 通用寄存器：
+===========================================================================+
|                                                                           |
|   被调用者保存寄存器（callee-saved）：                                    |
|   +-------------------+-------------------+                               |
|   | 寄存器            | 用途              |                               |
|   +-------------------+-------------------+                               |
|   | rbx               | 基址寄存器        |                               |
|   | rbp               | 帧指针            |                               |
|   | r12-r15           | 临时寄存器        |                               |
|   +-------------------+-------------------+                               |
|                                                                           |
|   调用者保存寄存器（caller-saved）：                                      |
|   +-------------------+-------------------+                               |
|   | 寄存器            | 用途              |                               |
|   +-------------------+-------------------+                               |
|   | rax               | 返回值            |                               |
|   | rcx, rdx          | 参数/临时         |                               |
|   | rsi, rdi          | 参数              |                               |
|   | r8-r11            | 参数/临时         |                               |
|   +-------------------+-------------------+                               |
|                                                                           |
|   关键区别：                                                              |
|   - 被调用者保存：函数调用后由被调用者恢复                                |
|   - 调用者保存：函数调用前由调用者保存                                    |
|   - 上下文切换只需保存被调用者保存寄存器！                                |
|                                                                           |
+===========================================================================+
```

### 4.3 调度入口

```c
/* 主调度器 */
static void __sched notrace __schedule(unsigned int sched_mode)
{
    struct task_struct *prev, *next;
    unsigned long *switch_count;
    unsigned long prev_state;
    struct rq_flags rf;
    struct rq *rq;
    int cpu;

    cpu = smp_processor_id();
    rq = cpu_rq(cpu);
    prev = rq->curr;

    /* ... */

    /* 选择下一个进程 */
    next = pick_next_task(rq, prev, &rf);

    if (likely(prev != next)) {
        /* 执行上下文切换 */
        rq = context_switch(rq, prev, next, &rf);
    }

    /* ... */
}
```

## 5. 核心数据结构

### 5.1 task_struct 关键字段

```c
struct task_struct {
    /* 进程状态 */

    struct mm_struct *active_mm;   /* 活跃地址空间（内核线程用） */

    /* 线程状态 */
    struct thread_struct thread;   /* 架构相关状态 */

    /* ... */
};
```

### 5.2 thread_struct 结构

```c
struct thread_struct {
    /* TLS 描述符 */
    struct desc_struct tls_array[3];

    unsigned long sp;              /* 用户栈指针 */
```

### 5.3 inactive_task_frame 结构

```c
/*
 * 这个结构体保存在内核栈顶部
 * 用于保存被切换出去的进程的寄存器状态
 * 字段顺序必须与 __switch_to_asm 中的保存顺序一致
 */
struct inactive_task_frame {
#ifdef CONFIG_X86_64
    unsigned long r15;
    unsigned long r14;
    unsigned long r13;
    unsigned long r12;
#else
    unsigned long flags;
    unsigned long si;
    unsigned long di;
#endif
    unsigned long bx;

    /* 这两个字段必须在一起，形成栈帧头 */
    unsigned long bp;              /* 帧指针 */
    unsigned long ret_addr;        /* 返回地址 */
};

struct fork_frame {
    struct inactive_task_frame frame;
    struct pt_regs regs;
};
```

### 6.2 context_switch 函数详解

```c
static __always_inline struct rq *
context_switch(struct rq *rq, struct task_struct *prev,
               struct task_struct *next, struct rq_flags *rf)
{
    /* 1. 准备任务切换 */
    prepare_task_switch(rq, prev, next);

    /* 2. 架构相关的开始切换 */
    arch_start_context_switch(prev);

    /*
     * 3. 地址空间切换
     *
     * kernel -> kernel   lazy + transfer active
     *   user -> kernel   lazy + mmgrab() active
     * kernel ->   user   switch + mmdrop() active
     *   user ->   user   switch
     */
    if (!next->mm) {                        /* 切换到内核线程 */
        /* 内核线程没有用户地址空间 */
        enter_lazy_tlb(prev->active_mm, next);

        /* 借用前一个进程的 active_mm */
        next->active_mm = prev->active_mm;
        if (prev->mm)                       /* 从用户进程切换 */
            mmgrab(prev->active_mm);        /* 增加引用计数 */
        else
            prev->active_mm = NULL;
    } else {                                /* 切换到用户进程 */
        membarrier_switch_mm(rq, prev->active_mm, next->mm);
        
        /* 切换页表 */
        switch_mm_irqs_off(prev->active_mm, next->mm, next);

        if (!prev->mm) {                    /* 从内核线程切换 */
            /* 将借用的 mm 放回 */
            rq->prev_mm = prev->active_mm;
            prev->active_mm = NULL;
        }
    }

    /* 4. 准备锁切换 */
    prepare_lock_switch(rq, next, rf);

    /* 5. 切换寄存器状态和栈 */
    switch_to(prev, next, prev);
    barrier();

    /* 6. 完成任务切换 */
    return finish_task_switch(prev);
}
```

## 7. switch_to 宏详解

### 7.1 宏定义

```c
/*
 * switch_to 宏
 * prev: 当前进程
 * next: 目标进程
 * last: 保存 prev 指针（用于返回后获取前一个进程）
 */
#define switch_to(prev, next, last)                  \
do {                                                 \
    ((last) = __switch_to_asm((prev), (next)));      \
} while (0)
```

### 7.2 __switch_to_asm 汇编实现（x86_64）

```asm
/*
 * __switch_to_asm
 * %rdi: prev task
 * %rsi: next task
 *
 * 返回值（%rax）: prev task
 */
SYM_FUNC_START(__switch_to_asm)
    /*
     * 保存被调用者保存寄存器
     * 顺序必须与 struct inactive_task_frame 一致
     */
    pushq   %rbp                /* 帧指针 */
    pushq   %rbx                /* 基址寄存器 */
    pushq   %r12                /* 被调用者保存寄存器 */
    pushq   %r13
    pushq   %r14
    pushq   %r15

    /* 关键：切换栈指针 */
    movq    %rsp, TASK_threadsp(%rdi)    /* 保存 prev 的栈指针 */
    movq    TASK_threadsp(%rsi), %rsp    /* 加载 next 的栈指针 */

#ifdef CONFIG_STACKPROTECTOR
    /* 更新栈保护 canary */
    movq    TASK_stack_canary(%rsi), %rbx
    movq    %rbx, PER_CPU_VAR(fixed_percpu_data) + stack_canary_offset
#endif

    /*
     * RSB（Return Stack Buffer）填充
     * 防止 Spectre v2 攻击
     */
    FILL_RETURN_BUFFER %r12, RSB_CLEAR_LOOPS, X86_FEATURE_RSB_CTXSW

    /* 恢复被调用者保存寄存器 */
    popq    %r15
    popq    %r14
    popq    %r13
    popq    %r12
    popq    %rbx
    popq    %rbp

    /* 跳转到 C 函数处理其他状态 */
    jmp     __switch_to
SYM_FUNC_END(__switch_to_asm)
```

### 7.4 __switch_to C 函数

```c
__visible __notrace_funcgraph struct task_struct *
__switch_to(struct task_struct *prev_p, struct task_struct *next_p)
{
    struct thread_struct *prev = &prev_p->thread;
    struct thread_struct *next = &next_p->thread;
    struct fpu *prev_fpu = &prev->fpu;
    struct fpu *next_fpu = &next->fpu;
    int cpu = smp_processor_id();

    /* 1. FPU 状态准备 */
    if (!test_thread_flag(TIF_NEED_FPU_LOAD))
        switch_fpu_prepare(prev_fpu, cpu);

    /* 2. 保存 FS/GS 段 */
    save_fsgs(prev_p);

    /* 3. 加载 TLS */
    load_TLS(next, cpu);

    /* 4. 结束 lazy 模式 */
    arch_end_context_switch(next_p);

    /* 5. 切换 DS/ES 段寄存器 */
    savesegment(es, prev->es);
    if (unlikely(next->es | prev->es))
        loadsegment(es, next->es);

    savesegment(ds, prev->ds);
    if (unlikely(next->ds | prev->ds))
        loadsegment(ds, next->ds);

    /* 6. 加载 FS/GS 基址 */
    x86_fsgsbase_load(prev, next);

    /* 7. 加载 PKRU（保护密钥） */
    x86_pkru_load(prev, next);

    /* 8. 更新当前任务指针 */
    this_cpu_write(current_task, next_p);
    this_cpu_write(cpu_current_top_of_stack, task_top_of_stack(next_p));

    /* 9. 完成 FPU 切换 */
    switch_fpu_finish(next_fpu);

    /* 10. 更新内核栈指针 */
    update_task_stack(next_p);

    /* 11. 其他切换 */
    switch_to_extra(prev_p, next_p);

    /* 12. 处理 AMD CPU 的 SYSRET SS 问题 */
    if (static_cpu_has_bug(X86_BUG_SYSRET_SS_ATTRS)) {
        unsigned short ss_sel;
        savesegment(ss, ss_sel);
        if (ss_sel != __KERNEL_DS)
            loadsegment(ss, __KERNEL_DS);
    }

    /* 13. 加载缓存分配 PQR MSR */
    resctrl_sched_in(next_p);

    return prev_p;
}
```

### 8.3 switch_mm 实现

```c
static inline void switch_mm(struct mm_struct *prev, 
                             struct mm_struct *next,
                             struct task_struct *tsk)
{
    unsigned cpu = smp_processor_id();

    if (likely(prev != next)) {
        /* 检查是否需要刷新 TLB */
        if (IS_ENABLED(CONFIG_VMAP_STACK)) {
            unsigned long stack = 
                (unsigned long)task_stack_page(tsk);
            if (stack > (1ULL << 48))
                leave_mm(cpu);
        }

        /* 加载新的页表 */
        load_mm_cr4(next);
        switch_ldt(prev, next);
        
        /* 设置 CR3 */
        write_cr3(__sme_pa(next->pgd) | 
                  (next->context.pcid & ~PCID_INV));

        /* 更新活跃 mm */
        this_cpu_write(cpu_tlbstate.active_mm, next);
        
        /* 更新 CPU 掩码 */
        cpumask_set_cpu(cpu, mm_cpumask(next));
    }
}
```

```c
/* 切换前准备 */
void switch_fpu_prepare(struct fpu *old_fpu, int cpu)
{
    if (static_cpu_has(X86_FEATURE_FPU) &&
        !(current->flags & PF_KTHREAD)) {
        /* 保存 FPU 状态 */
        copy_fpregs_to_fpstate(old_fpu);
        /* 标记 FPU 状态需要加载 */
        __cpu_invalidate_fpregs_state();
    }
}

/* 切换后完成 */
void switch_fpu_finish(struct fpu *new_fpu)
{
    if (test_thread_flag(TIF_NEED_FPU_LOAD)) {
        /* 恢复 FPU 状态 */
        __copy_fpregs_to_fpregs(new_fpu);
        clear_thread_flag(TIF_NEED_FPU_LOAD);
    }
}
```

```c
    const cpumask_t *cpus_ptr;     /* 允许的 CPU 掩码 */
    cpumask_t cpus_mask;
};
```

## 总结

### 核心要点

1. **上下文切换的本质**：保存当前进程状态，恢复目标进程状态，让 CPU 能够继续执行另一个进程。

2. **软件切换的优势**：Linux 5.15 使用软件实现上下文切换，比硬件 TSS 切换更灵活、更高效。

3. **关键数据结构**：
   - task_struct：进程控制块
   - thread_struct：线程相关状态
   - inactive_task_frame：保存在栈上的寄存器状态

4. **切换流程**：
   - pick_next_task() 选择下一个进程
   - context_switch() 执行切换
   - switch_mm() 切换地址空间
   - switch_to() 切换寄存器和栈

5. **性能优化**：
   - 只保存必要寄存器
   - 懒惰 FPU 切换
   - Lazy TLB
   - 内核线程借用 mm

## 1. 概述

### 1.1 Mount 与链接机制的重要性

```
Mount 与链接机制的重要性：
+===========================================================================+
|                                                                           |
|   Mount（挂载）：                                                         |
|   +-------------------+                                                    |
|   | 将文件系统关联到目录树 |                                                |
|   | 实现多文件系统支持     |                                                |
|   | 构建统一的命名空间     |                                                |
|   +-------------------+                                                    |
|                                                                           |
|   硬链接（Hard Link）：                                                   |
|   +-------------------+                                                    |
|   | 多个文件名指向同一 inode |                                              |
|   | 共享文件数据           |                                                |
|   | 删除原文件不影响链接   |                                                |
|   +-------------------+                                                    |
|                                                                           |
|   软链接（Symbolic Link）：                                               |
|   +-------------------+                                                    |
|   | 特殊文件存储目标路径   |                                                |
|   | 可以跨文件系统         |                                                |
|   | 可以链接目录           |                                                |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 1.2 版本对比总览

| 特性 | Linux 0.11 | Linux 5.15 |
|------|------------|------------|
| 挂载结构 | 简单的超级块数组 | 复杂的 mount 命名空间 |
| 挂载点表示 | i_mount 标志 | vfsmount 结构 |
| 硬链接 | 基本支持 | 完整支持 + 安全检查 |
| 软链接 | 不支持 | 完整支持 |
| 路径解析 | 简单的 namei() | 复杂的 path_lookup() |
| 命名空间 | 不支持 | 完整的命名空间隔离 |

## 2. 挂载机制对比

### 2.1 数据结构对比

```
Linux 0.11 挂载数据结构：
+===========================================================================+
|                                                                           |
|   超级块数组：                                                            |
|   +-------------------+                                                    |
|   | super_block[8]    |  最多 8 个文件系统                                |
|   +-------------------+                                                    |
|                                                                           |
|   挂载关系：                                                              |
|   +-------------------+                                                    |
|   | s_imount          |  挂载点 inode                                     |
|   | s_isup            |  被挂载文件系统的根 inode                         |
|   | i_mount           |  inode 的挂载标志                                 |
|   +-------------------+                                                    |
|                                                                           |
|   结构简单，功能有限：                                                    |
|   +-------------------+                                                    |
|   | 不支持挂载命名空间 |                                                    |
|   | 不支持绑定挂载     |                                                    |
|   | 不支持挂载传播     |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

Linux 5.15 挂载数据结构：
+===========================================================================+
|                                                                           |
|   vfsmount 结构：                                                         |
|   +-------------------+                                                    |
|   | mnt_root          |  挂载树的根 dentry                                |
|   | mnt_sb            |  超级块指针                                       |
|   | mnt_flags         |  挂载标志                                         |
|   | mnt_userns        |  用户命名空间                                     |
|   +-------------------+                                                    |
|                                                                           |
|   mount 结构（内部）：                                                    |
|   +-------------------+                                                    |
|   | mnt_parent        |  父挂载点                                         |
|   | mnt_mountpoint    |  挂载点 dentry                                    |
|   | mnt_mp            |  挂载点结构                                       |
|   | mnt_mp_list       |  同一挂载点的挂载链表                             |
|   | mnt_slave_list    |  从属挂载链表                                     |
|   | mnt_slave         |  从属挂载节点                                     |
|   | mnt_master        |  主挂载点                                         |
|   | mnt_ns            |  挂载命名空间                                     |
|   +-------------------+                                                    |
|                                                                           |
|   功能丰富：                                                              |
|   +-------------------+                                                    |
|   | 支持挂载命名空间   |                                                    |
|   | 支持绑定挂载       |                                                    |
|   | 支持挂载传播       |                                                    |
|   | 支持共享子树       |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 2.2 挂载操作对比

```
Linux 0.11 挂载操作：
+===========================================================================+
|                                                                           |
|   sys_mount() 流程：                                                      |
|   +-------------------+                                                    |
|   | 1. 获取设备 inode |                                                    |
|   | 2. 获取挂载点 inode |                                                  |
|   | 3. 读取超级块     |                                                    |
|   | 4. 建立挂载关系   |                                                    |
|   +-------------------+                                                    |
|                                                                           |
|   代码片段：                                                              |
|   +-------------------+                                                    |
|   | sb->s_imount = dir_i; |                                                |
|   | dir_i->i_mount = 1;   |                                                |
|   +-------------------+                                                    |
|                                                                           |
|   限制：                                                                  |
|   +-------------------+                                                    |
|   | 只能在目录上挂载   |                                                    |
|   | 不支持递归挂载     |                                                    |
|   | 不支持只读挂载     |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

Linux 5.15 挂载操作：
+===========================================================================+
|                                                                           |
|   sys_mount() 流程：                                                      |
|   +-------------------+                                                    |
|   | 1. 复制用户空间参数 |                                                    |
|   | 2. 解析文件系统类型 |                                                    |
|   | 3. 创建文件系统上下文 |                                                 |
|   | 4. 执行挂载操作     |                                                    |
|   | 5. 附加到命名空间   |                                                    |
|   +-------------------+                                                    |
|                                                                           |
|   支持的挂载类型：                                                        |
|   +-------------------+                                                    |
|   | MS_BIND           |  绑定挂载                                         |
|   | MS_MOVE           |  移动挂载                                         |
|   | MS_REC            |  递归挂载                                         |
|   | MS_PRIVATE        |  私有挂载                                         |
|   | MS_SLAVE          |  从属挂载                                         |
|   | MS_SHARED         |  共享挂载                                         |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 2.3 挂载点查找对比

```
Linux 0.11 挂载点查找：
+===========================================================================+
|                                                                           |
|   在 iget() 中处理：                                                      |
|   +-------------------+                                                    |
|   | if (inode->i_mount) { |                                                 |
|   |     /* 查找挂载的文件系统 */ |                                          |
|   |     for (sb = super_block; sb; sb++) { |                               |
|   |         if (sb->s_imount == inode) { |                                 |
|   |             /* 返回挂载文件系统的根 inode */ |                          |
|   |             return iget(sb->s_dev, ROOT_INO); |                        |
|   |         } |                                                            |
|   |     } |                                                                |
|   | } |                                                                    |
|   +-------------------+                                                    |
|                                                                           |
|   特点：                                                                  |
|   +-------------------+                                                    |
|   | 线性搜索超级块数组 |                                                    |
|   | 效率低             |                                                    |
|   | 只支持单个挂载点   |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

Linux 5.15 挂载点查找：
+===========================================================================+
|                                                                           |
|   在 path_lookup() 中处理：                                               |
|   +-------------------+                                                    |
|   | lookup_mnt()      |  查找挂载点                                       |
|   | follow_down()     |  进入挂载点                                       |
|   | follow_up()       |  返回上层挂载                                     |
|   +-------------------+                                                    |
|                                                                           |
|   挂载点哈希表：                                                          |
|   +-------------------+                                                    |
|   | mount_hashtable   |  快速查找挂载点                                   |
|   | 多级哈希          |  高效                                             |
|   +-------------------+                                                    |
|                                                                           |
|   特点：                                                                  |
|   +-------------------+                                                    |
|   | 哈希表快速查找     |                                                    |
|   | 支持多层挂载       |                                                    |
|   | 支持挂载传播       |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

## 3. 硬链接机制对比

### 3.1 硬链接概念

```
硬链接原理：
+===========================================================================+
|                                                                           |
|   硬链接是多个文件名指向同一个 inode：                                     |
|   +-------------------+                                                    |
|   | 文件名1 ─────┐    |                                                    |
|   | 文件名2 ─────┼──→ inode ──→ 数据块                                     |
|   | 文件名3 ─────┘    |                                                    |
|   +-------------------+                                                    |
|                                                                           |
|   特点：                                                                  |
|   +-------------------+                                                    |
|   | 共享相同的 inode   |                                                    |
|   | 共享相同的数据     |                                                    |
|   | i_nlinks 记录链接数 |                                                   |
|   | 删除文件时减少链接数 |                                                  |
|   | 链接数为 0 时才真正删除 |                                               |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 3.2 硬链接操作对比

```
Linux 0.11 硬链接：
+===========================================================================+
|                                                                           |
|   不支持 sys_link() 系统调用                                              |
|   +-------------------+                                                    |
|   | Minix 文件系统限制 |                                                    |
|   | 只支持目录项       |                                                    |
|   +-------------------+                                                    |
|                                                                           |
|   链接计数：                                                              |
|   +-------------------+                                                    |
|   | i_nlinks 字段     |  记录硬链接数                                     |
|   | 创建目录时设为 2  |  . 和 ..                                          |
|   | 创建文件时设为 1  |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

Linux 5.15 硬链接：
+===========================================================================+
|                                                                           |
|   sys_link() 系统调用：                                                   |
|   +-------------------+                                                    |
|   | SYSCALL_DEFINE2(link, oldname, newname) |                              |
|   +-------------------+                                                    |
|                                                                           |
|   vfs_link() 函数：                                                       |
|   +-------------------+                                                    |
|   | 检查权限         |                                                    |
|   | 检查文件系统     |  必须在同一文件系统                                |
|   | 检查链接数       |  不能超过最大链接数                                |
|   | 调用 i_op->link  |  文件系统特定操作                                  |
|   +-------------------+                                                    |
|                                                                           |
|   安全检查：                                                              |
|   +-------------------+                                                    |
|   | 不能链接目录      |  防止循环                                         |
|   | 不能跨文件系统    |  inode 不能跨文件系统                             |
|   | 不能链接不可变文件 |  安全考虑                                         |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

## 4. 软链接机制对比

### 4.1 软链接概念

```
软链接原理：
+===========================================================================+
|                                                                           |
|   软链接是特殊文件，存储目标文件的路径：                                   |
|   +-------------------+                                                    |
|   | 软链接文件       |                                                    |
|   | +-------------+ |                                                    |
|   | | inode       | |                                                    |
|   | | 类型: S_IFLNK | |                                                    |
|   | | 数据: 路径字符串 |                                                   |
|   | +-------------+ |                                                    |
|   +-------------------+                                                    |
|           |                                                               |
           | 存储的路径
|           v                                                               |
|   +-------------------+                                                    |
|   | 目标文件         |                                                    |
|   +-------------------+                                                    |
|                                                                           |
|   特点：                                                                  |
|   +-------------------+                                                    |
|   | 有自己的 inode     |                                                    |
|   | 存储目标路径       |                                                    |
|   | 可以跨文件系统     |                                                    |
|   | 可以链接目录       |                                                    |
|   | 目标删除后成为悬空链接 |                                                |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 4.2 软链接支持对比

```
Linux 0.11 软链接：
+===========================================================================+
|                                                                           |
|   不支持软链接                                                            |
|   +-------------------+                                                    |
|   | Minix 文件系统限制 |                                                    |
|   | 没有 S_IFLNK 类型  |                                                    |
|   | 没有 symlink 系统调用 |                                                 |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

Linux 5.15 软链接：
+===========================================================================+
|                                                                           |
|   sys_symlink() 系统调用：                                                |
|   +-------------------+                                                    |
|   | SYSCALL_DEFINE2(symlink, oldname, newname) |                           |
|   +-------------------+                                                    |
|                                                                           |
|   vfs_symlink() 函数：                                                    |
|   +-------------------+                                                    |
|   | 检查权限         |                                                    |
|   | 创建特殊 inode   |  类型为 S_IFLNK                                    |
|   | 存储目标路径     |  在数据块中                                        |
|   +-------------------+                                                    |
|                                                                           |
|   软链接解析：                                                            |
|   +-------------------+                                                    |
|   | follow_link()    |  跟随软链接                                        |
|   | 递归解析         |  支持多层软链接                                    |
|   | 循环检测         |  防止无限循环                                      |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

## 5. 路径解析对比

### 5.1 路径解析流程对比

```
Linux 0.11 路径解析（namei）：
+===========================================================================+
|                                                                           |
|   简单的线性解析：                                                        |
|   +-------------------+                                                    |
|   | 1. 从根目录或当前目录开始 |                                            |
|   | 2. 逐级解析路径分量     |                                              |
|   | 3. 查找目录项           |                                              |
|   | 4. 获取 inode           |                                              |
|   | 5. 处理挂载点           |                                              |
|   +-------------------+                                                    |
|                                                                           |
|   不支持：                                                                |
|   +-------------------+                                                    |
|   | 软链接解析       |                                                    |
|   | 挂载命名空间     |                                                    |
|   | 原子操作         |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

Linux 5.15 路径解析（path_lookup）：
+===========================================================================+
|                                                                           |
|   复杂的路径解析：                                                        |
|   +-------------------+                                                    |
|   | 1. 确定起始点     |  根目录、当前目录、文件描述符                      |
|   | 2. 逐级解析路径分量 |                                                  |
|   | 3. 处理软链接     |  递归解析                                         |
|   | 4. 处理挂载点     |  进入/退出挂载点                                  |
|   | 5. 权限检查       |  每一级都检查                                     |
|   | 6. 处理特殊情况   |  .. 、. 、符号链接                                |
|   +-------------------+                                                    |
|                                                                           |
|   支持的特性：                                                            |
|   +-------------------+                                                    |
|   | 软链接解析       |                                                    |
|   | 挂载命名空间     |                                                    |
|   | RCU 路径查找     |  无锁快速查找                                      |
|   | 原子操作         |                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 5.2 挂载点处理对比

```
Linux 0.11 挂载点处理：
+===========================================================================+
|                                                                           |
|   在 iget() 中检查 i_mount 标志：                                         |
|   +-------------------+                                                    |
|   | if (inode->i_mount) { |                                                 |
|   |     /* 查找挂载的文件系统 */ |                                          |
|   |     for (i = 0; i < NR_SUPER; i++) { |                                 |
|   |         if (super_block[i].s_imount == inode) { |                      |
|   |             iput(inode); |                                              |
|   |             inode = iget(super_block[i].s_dev, ROOT_INO); |            |
|   |             break; |                                                    |
|   |         } |                                                            |
|   |     } |                                                                |
|   | } |                                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

Linux 5.15 挂载点处理：
+===========================================================================+
|                                                                           |
|   在 follow_managed() 中处理：                                            |
|   +-------------------+                                                    |
|   | lookup_mnt()     |  查找挂载点                                       |
|   | follow_down()    |  进入挂载点                                       |
|   | d_automount()    |  自动挂载                                         |
|   | d_manage()       |  管理操作                                         |
|   +-------------------+                                                    |
|                                                                           |
|   挂载点查找：                                                            |
|   +-------------------+                                                    |
|   | struct mount *lookup_mnt(struct path *path) |                          |
|   | { |                                                                    |
|   |     struct mount *child; |                                              |
|   |     unsigned hash = hash(path); |                                       |
|   |     hlist_for_each_entry(child, hash_head, mnt_hash) { |               |
|   |         if (child->mnt_parent == mnt && |                              |
|   |             child->mnt_mountpoint == dentry) |                         |
|   |             return child; |                                             |
|   |     } |                                                                |
|   |     return NULL; |                                                      |
|   | } |                                                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

## 6. 总结

### 6.1 主要差异总结

```
主要差异总结：
+===========================================================================+
|                                                                           |
|   挂载机制：                                                              |
|   +-------------------+-------------------+                               |
|   | Linux 0.11       | Linux 5.15        |                               |
|   +-------------------+-------------------+                               |
|   | 简单的超级块数组 | 复杂的 mount 结构 |                                |
|   | i_mount 标志     | vfsmount 结构     |                                |
|   | 不支持命名空间   | 完整命名空间支持  |                                |
|   | 不支持绑定挂载   | 支持绑定挂载      |                                |
|   | 不支持挂载传播   | 支持挂载传播      |                                |
|   +-------------------+-------------------+                               |
|                                                                           |
|   硬链接：                                                                |
|   +-------------------+-------------------+                               |
|   | Linux 0.11       | Linux 5.15        |                               |
|   +-------------------+-------------------+                               |
|   | 不支持           | 完整支持          |                                |
|   | 只有链接计数     | 完整的 link 系统调用 |                             |
|   +-------------------+-------------------+                               |
|                                                                           |
|   软链接：                                                                |
|   +-------------------+-------------------+                               |
|   | Linux 0.11       | Linux 5.15        |                               |
|   +-------------------+-------------------+                               |
|   | 不支持           | 完整支持          |                                |
|   | 无 symlink 类型  | S_IFLNK 类型      |                                |
|   +-------------------+-------------------+                               |
|                                                                           |
|   路径解析：                                                              |
|   +-------------------+-------------------+                               |
|   | Linux 0.11       | Linux 5.15        |                               |
|   +-------------------+-------------------+                               |
|   | 简单的 namei()   | 复杂的 path_lookup() |                             |
|   | 不支持软链接解析 | 支持软链接解析    |                                |
|   | 线性搜索挂载点   | 哈希表查找挂载点  |                                |
|   +-------------------+-------------------+                               |
|                                                                           |
+===========================================================================+
```

### 6.2 演进意义

```
演进意义：
+===========================================================================+
|                                                                           |
|   1. 功能完善：                                                           |
|   +-------------------+                                                    |
|   | 支持软链接       |  灵活的文件引用                                   |
|   | 支持硬链接       |  高效的文件共享                                   |
|   | 支持多种挂载类型 |  灵活的文件系统管理                               |
|   +-------------------+                                                    |
|                                                                           |
|   2. 性能优化：                                                           |
|   +-------------------+                                                    |
|   | 哈希表查找       |  快速定位挂载点                                   |
|   | RCU 路径查找     |  无锁快速查找                                     |
|   | 缓存优化         |  dentry 缓存、inode 缓存                         |
|   +-------------------+                                                    |
|                                                                           |
|   3. 安全增强：                                                           |
|   +-------------------+                                                    |
|   | 权限检查         |  每一步都检查权限                                 |
|   | 命名空间隔离     |  容器化支持                                       |
|   | 挂载传播控制     |  精细的挂载管理                                   |
|   +-------------------+                                                    |
|                                                                           |
|   4. 可扩展性：                                                           |
|   +-------------------+                                                    |
|   | 支持多种文件系统 |  ext4、btrfs、xfs 等                              |

|                                                                           |
+===========================================================================+
```

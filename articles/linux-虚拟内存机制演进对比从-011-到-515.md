## 概述

Linux 内核从 0.11 版本（1991年）发展到 5.15 版本（2021年），虚拟内存管理机制经历了巨大的演进。本文档详细对比两个版本的核心差异。

### 时间跨度

- **Linux 0.11**：1991年发布，Linus Torvalds 的早期作品
- **Linux 5.15**：2021年发布，现代企业级内核

### 核心目标对比

| 版本 | 核心目标 |
|------|---------|
| Linux 0.11 | 实现基本的虚拟内存功能，支持最多 16MB 物理内存 |
| Linux 5.15 | 支持海量内存、多核系统、容器化、安全性、性能优化 |

## 核心差异对比

### 1. 支持的内存规模

| 特性 | Linux 0.11 | Linux 5.15 |
|------|-----------|-----------|
| 最大物理内存 | 16 MB | 数 TB（取决于架构） |
| 地址空间位数 | 32 位 | 64 位（x86_64） |
| 最大进程数 | 受内存限制 | 几乎无限制 |
| 页面大小 | 4 KB | 4 KB / 2 MB / 1 GB（大页） |

### 2. 页表层级

| 版本 | 页表层级 | 说明 |
|------|---------|------|
| Linux 0.11 | 2 级 | 页目录 + 页表 |
| Linux 5.15 | 4-5 级 | PGD + P4D + PUD + PMD + PTE |

### 3. 核心数据结构

| 数据结构 | Linux 0.11 | Linux 5.15 |
|---------|-----------|-----------|
| 进程内存描述 | 简单的页目录指针 | mm_struct（复杂的内存描述符） |
| 虚拟内存区域 | 无 | vm_area_struct（VMA） |
| 页面描述 | mem_map 数组 | struct page（复杂的联合体） |
| 内存节点 | 无 | pglist_data（NUMA 支持） |

### 4. 关键特性对比

| 特性 | Linux 0.11 | Linux 5.15 |
|------|-----------|-----------|
| 写时复制（COW） | ✓ 基本实现 | ✓ 完善，支持多种场景 |
| 请求调页 | ✓ 基本实现 | ✓ 完善，支持预读 |
| 内存映射 | ✗ | ✓ 支持 mmap、共享内存 |
| 交换分区 | ✗ | ✓ 完善的交换机制 |
| 内存压缩 | ✗ | ✓ zswap、zram |
| 大页支持 | ✗ | ✓ hugetlbfs、THP |
| 内存控制器 | ✗ | ✓ cgroups 内存控制 |
| NUMA 支持 | ✗ | ✓ 完整的 NUMA 支持 |
| 内存热插拔 | ✗ | ✓ 支持动态添加/移除内存 |
| 内存加密 | ✗ | ✓ SME、SEV 等加密技术 |

## 页表结构演进

### Linux 0.11：二级页表

```
线性地址 (32位):
┌─────────────────┬─────────────────┬─────────────────┐
│   页目录索引     │    页表索引      │    页内偏移      │
│    (10位)       │     (10位)      │     (12位)      │
│   31    22      │   21      12    │   11        0   │
└─────────────────┴─────────────────┴─────────────────┘

地址转换流程：
CR3 → 页目录(PDE) → 页表(PTE) → 物理页面

每个进程一个页目录
页目录大小：4KB (1024 项)
页表大小：4KB (1024 项)
每页大小：4KB
```

**特点**：
- 简单直接，适合早期硬件
- 地址空间限制：4GB
- 每个进程需要完整的页目录和页表

### Linux 5.15：四级/五级页表

```
线性地址 (64位):
┌────────┬────────┬────────┬────────┬────────┬────────┐
│ 保留   │ P4D索引 │ PUD索引│ PMD索引│ PTE索引│ 页内偏移│
│ (16位) │ (9位)  │ (9位)  │ (9位)  │ (9位)  │ (12位) │
└────────┴────────┴────────┴────────┴────────┴────────┘

地址转换流程：
CR3 → PGD → P4D → PUD → PMD → PTE → 物理页面

五级页表支持：
- PGD (Page Global Directory)：页全局目录
- P4D (Page 4th Level Directory)：页四级目录
- PUD (Page Upper Directory)：页上级目录
- PMD (Page Middle Directory)：页中间目录
- PTE (Page Table Entry)：页表项
```

**特点**：
- 支持 64 位地址空间（实际使用 48-57 位）
- 支持大页：2MB（PMD级）、1GB（PUD级）
- 按需分配页表，节省内存
- 支持页表共享（共享内存）

### 页表项标志位对比

| 标志位 | Linux 0.11 | Linux 5.15 | 说明 |
|--------|-----------|-----------|------|
| Present (P) | ✓ | ✓ | 页是否存在 |
| Read/Write (R/W) | ✓ | ✓ | 读写权限 |
| User/Supervisor (U/S) | ✓ | ✓ | 用户/内核权限 |
| Accessed (A) | ✓ | ✓ | 是否被访问 |
| Dirty (D) | ✓ | ✓ | 是否被修改 |
| Page Size (PS) | ✗ | ✓ | 大页标志 |
| Global (G) | ✗ | ✓ | 全局页 |
| NX (No Execute) | ✗ | ✓ | 禁止执行 |
| Soft Dirty | ✗ | ✓ | 软脏位 |
| NUMA | ✗ | ✓ | NUMA 平衡 |

## 内存管理数据结构演进

### Linux 0.11：简单的 mem_map 数组

```c
/* 简单的页面引用计数数组 */
static unsigned char mem_map[PAGING_PAGES] = {0,};

/* 
 * mem_map[i] 的值：
 * 0：页面空闲
 * 1-99：页面引用计数
 * 100：页面保留（内核使用）
 */
```

**特点**：
- 结构简单，只是一个字节数组
- 只记录引用计数
- 不支持复杂的页面状态管理
- 不支持 NUMA

### Linux 5.15：复杂的 struct page

```c
struct page {
    unsigned long flags;        /* 页面标志位 */
    
    union {
        struct {    /* 页缓存和匿名页 */
            struct list_head lru;
            struct address_space *mapping;
            pgoff_t index;
            unsigned long private;
        };
        struct {    /* slab 分配器 */
            struct list_head slab_list;
            struct kmem_cache *slab_cache;
            void *freelist;
            union {
                void *s_mem;
                unsigned long counters;
                struct {
                    unsigned inuse:16;
                    unsigned objects:15;
                    unsigned frozen:1;
                };
            };
        };
        struct {    /* 大页 */
            unsigned long compound_head;
            unsigned char compound_dtor;
            unsigned char compound_order;
            atomic_t compound_mapcount;
        };
        struct {    /* 页表页 */
            unsigned long _pt_pad_1;
            pgtable_t pmd_huge_pte;
            unsigned long _pt_pad_2;
            union {
                struct mm_struct *pt_mm;
                atomic_t pt_frag_refcount;
            };
            spinlock_t ptl;
        };
        /* ... 其他用途 ... */
    };
    
    union {
        atomic_t _mapcount;      /* 映射计数 */
        unsigned int page_type;   /* 页面类型 */
    };
    
    atomic_t _refcount;          /* 引用计数 */
    
#ifdef CONFIG_MEMCG
    unsigned long memcg_data;    /* 内存控制器数据 */
#endif
};
```

**特点**：
- 使用联合体节省空间
- 支持多种用途（页缓存、slab、大页、页表等）
- 完善的引用计数机制
- 支持内存控制器
- 支持多种页面状态标志

### 内存节点：NUMA 支持

**Linux 0.11**：无 NUMA 概念

**Linux 5.15**：

```c
typedef struct pglist_data {
```

## 页面分配与回收演进

### Linux 0.11：简单的线性搜索

```c
/* 从 mem_map 数组末尾向前搜索空闲页 */
unsigned long get_free_page(void)
{
    /* 使用汇编从后向前搜索 mem_map */
    /* 找到值为 0 的项，标记为 1 */
    /* 计算物理地址，清零页面 */
    /* 返回物理地址 */
}
```

**特点**：
- 简单的线性搜索
- O(n) 时间复杂度
- 不考虑页面连续性
- 无伙伴系统

### Linux 5.15：伙伴系统 + Slab 分配器

#### 伙伴系统（Buddy System）

```c
/*
 * 伙伴系统将内存按 2^n 大小分组
 * 支持 order 0 到 order MAX_ORDER-1
 * order 0: 1 页 (4KB)
 * order 1: 2 页 (8KB)
 * order 2: 4 页 (16KB)
 * ...
 * order 10: 1024 页 (4MB)
 */

struct free_area {
    struct list_head free_list[MIGRATE_TYPES]; /* 空闲链表 */
    unsigned long nr_free;                      /* 空闲块数量 */
};

struct zone {
    /* ... */
    struct free_area free_area[MAX_ORDER];     /* 伙伴系统 */
    /* ... */
};
```

**特点**：
- O(log n) 时间复杂度
- 自动合并相邻空闲块
- 减少外部碎片
- 支持迁移类型（可移动、可回收、不可移动）

#### Slab 分配器

```c
/*
 * Slab 分配器用于小对象分配
 * 从伙伴系统获取页面，切分成小对象
 */

struct kmem_cache {
    const char *name;              /* 缓存名称 */
    unsigned int object_size;      /* 对象大小 */
    unsigned int size;             /* 总大小 */
    unsigned int align;            /* 对齐 */
    
    struct kmem_cache_node *node[MAX_NUMNODES]; /* 节点数组 */
    struct kmem_cache_cpu __percpu *cpu_slab;   /* CPU 缓存 */
    
    /* ... */
};
```

**特点**：
- 减少内部碎片
- 对象缓存，提高性能
- 支持 per-CPU 缓存
- 支持多种分配器（SLAB、SLUB、SLOB）

```c
/* 页错误入口（架构相关） */
void do_page_fault(struct pt_regs *regs, unsigned long error_code)
{
    /* 1. 获取错误地址 */
    unsigned long address = read_cr2();
    
    /* 2. 查找 VMA */
    struct vm_area_struct *vma = find_vma(mm, address);
    
    /* 3. 检查权限 */
    if (unlikely(!vma || address < vma->vm_start)) {
        /* 处理无效地址 */
        bad_area(regs, error_code, address);
        return;
    }
    
    /* 4. 调用通用处理函数 */
    handle_mm_fault(vma, address, flags, regs);
}

/* 通用页错误处理 */
vm_fault_t handle_mm_fault(struct vm_area_struct *vma,
                           unsigned long address, unsigned int flags)
{
    /* 1. 更新统计信息 */
    /* 2. 检查内存控制器限制 */
    /* 3. 调用 __handle_mm_fault */
}

/* 实际的页错误处理 */
static vm_fault_t __handle_mm_fault(struct vm_area_struct *vma,
                                    unsigned long address, unsigned int flags)
{
    /* 1. 获取或分配 PGD */
    /* 2. 获取或分配 P4D */
    /* 3. 获取或分配 PUD */
    /* 4. 获取或分配 PMD */
    /* 5. 获取或分配 PTE */
    /* 6. 调用 handle_pte_fault */
}

/* PTE 级别的错误处理 */
static vm_fault_t handle_pte_fault(struct vm_fault *vmf)
{
    /* 1. 页不存在 */
    if (!vmf->pte) {
        /* 匿名页：do_anonymous_page */
        /* 文件映射：do_fault */
    }
    
    /* 2. 页存在但需要处理 */
    if (!pte_present(*vmf->pte)) {
        /* 交换页：do_swap_page */
    }
    
    /* 3. 写保护 */
    if (vmf->flags & FAULT_FLAG_WRITE) {
        /* 写时复制：do_wp_page */
    }
    
    /* 4. NUMA 平衡 */
    if (pte_protnone(vmf->orig_pte) && vma_is_accessible(vmf->vma)) {
        /* NUMA 迁移：do_numa_page */
    }
}
```

**特点**：
- 完善的错误分类和处理
- 支持交换分区
- 支持文件映射
- 支持 NUMA 平衡
- 支持用户态页错误处理（userfaultfd）

## 内存映射演进

### Linux 0.11：无内存映射

Linux 0.11 不支持 mmap() 系统调用，进程的地址空间是固定的。

### Linux 5.15：完善的内存映射

```c
/* VMA 结构 */
struct vm_area_struct {
    unsigned long vm_start;        /* 起始地址 */
    unsigned long vm_end;          /* 结束地址 */
    
    struct vm_area_struct *vm_next, *vm_prev; /* 链表 */
    struct rb_node vm_rb;          /* 红黑树节点 */
    
    struct mm_struct *vm_mm;       /* 所属的 mm_struct */
    pgprot_t vm_page_prot;         /* 页面保护 */
    unsigned long vm_flags;        /* 标志位 */
    
    const struct vm_operations_struct *vm_ops; /* 操作函数 */
    unsigned long vm_pgoff;        /* 文件偏移 */
    struct file *vm_file;          /* 映射的文件 */
    void *vm_private_data;         /* 私有数据 */
};

/* VMA 操作函数 */
struct vm_operations_struct {
    void (*open)(struct vm_area_struct * area);
    void (*close)(struct vm_area_struct * area);
    vm_fault_t (*fault)(struct vm_fault *vmf);
    void (*map_pages)(struct vm_fault *vmf,
                      pgoff_t start_pgoff, pgoff_t end_pgoff);
    /* ... */
};
```

**特点**：
- 支持文件映射和匿名映射
- 支持共享映射和私有映射
- 使用红黑树快速查找 VMA
- 支持多种映射类型

## 总结

### 演进历程

```
Linux 0.11 (1991)
    ↓
简单的虚拟内存管理
- 二级页表
- 基本的 COW
- 最多 16MB 内存
    ↓
Linux 2.4 (2001)
    ↓
完善虚拟内存
- 三级页表
- 完善的交换机制
- NUMA 支持
    ↓
Linux 2.6 (2003)
    ↓
性能优化
- 四级页表
- 大页支持
- 内存控制器
    ↓
Linux 5.15 (2021)
    ↓
现代内存管理
- 四/五级页表
- 透明大页
- 内存加密
- 内存热插拔
- 完善的 NUMA 支持
```

### 核心改进

1. **可扩展性**：从 16MB 到数 TB
2. **性能**：伙伴系统、Slab、大页
3. **安全性**：内存加密、权限控制
4. **灵活性**：内存热插拔、容器化
5. **可靠性**：完善的错误处理

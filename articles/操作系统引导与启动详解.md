## 引导概述

### 什么是引导

引导（Boot）是指计算机从加电到操作系统内核开始运行的过程。这个过程包括：

1. **BIOS 自检** - 加电自检（POST）
2. **加载引导扇区** - BIOS 读取磁盘第一个扇区
3. **加载操作系统** - 引导程序加载内核
4. **切换执行模式** - 从实模式切换到保护模式
5. **内核初始化** - 初始化各子系统

### Linux 0.11 引导程序组成

| 文件 | 位置 | 功能 |
|------|------|------|
| bootsect.s | 引导扇区 | 加载 setup 和内核 |
| setup.s | 4个扇区 | 获取硬件信息，进入保护模式 |
| head.s | 内核开头 | 初始化 GDT、IDT、分页 |
| main.c | 内核主体 | 初始化各子系统 |

## bootsect.s - 引导扇区

### 文件位置

**定义位置**：**`boot/bootsect.s`**

### 主要功能

1. 将自身从 0x7C00 移动到 0x90000
2. 加载 setup.s 到 0x90200
3. 加载内核到 0x10000
4. 确定根文件系统设备
5. 跳转到 setup.s

### 常量定义

**定义位置**：**`boot/bootsect.s:26-37`**

```asm
SETUPLEN = 4                ! setup 占用的扇区数
BOOTSEG  = 0x07c0           ! 引导扇区原始地址
INITSEG  = 0x9000           ! 引导程序移动后的地址
SETUPSEG = 0x9020           ! setup 程序的地址
SYSSEG   = 0x1000           ! 内核加载地址 (0x10000)
ENDSEG   = SYSSEG + SYSSIZE ! 内核结束地址
ROOT_DEV = 0x306            ! 根文件系统设备
```

### 1. 引导扇区自移动

**定义位置**：**`boot/bootsect.s:43-56`**

```asm
entry start
start:
	mov	ax, #BOOTSEG      ; 源段地址 = 0x07c0
	mov	ds, ax
	mov	ax, #INITSEG      ; 目的段地址 = 0x9000
	mov	es, ax
	mov	cx, #256          ; 移动 256 字（512字节）
	sub	si, si            ; 源偏移 = 0
	sub	di, di            ; 目的偏移 = 0
	rep
	movw                   ; 复制
	jmpi	go, INITSEG     ; 跳转到新位置
```

```

移动后:
┌─────────────────────────────────────┐
│  0x90000 - 0x901FF │  bootsect.s   │ ← 新位置
├─────────────────────────────────────┤

├─────────────────────────────────────┤
```

### 2. 加载 setup 程序

**定义位置**：**`boot/bootsect.s:65-79`**

```asm
load_setup:
	mov	dx, #0x0000          ; 驱动器 0, 磁头 0
	mov	cx, #0x0002          ; 扇区 2, 磁道 0
	mov	bx, #0x0200          ; 缓冲区偏移 = 512
	mov	ax, #0x0200+SETUPLEN ; 功能号 2, 读取 4 个扇区
	int	0x13                 ; BIOS 磁盘中断
	jnc	ok_load_setup        ; 成功则继续
	mov	dx, #0x0000
	mov	ax, #0x0000          ; 复位磁盘
	int	0x13
	j	load_setup           ; 重试
```

**BIOS int 0x13 参数说明**：

| 寄存器 | 值 | 说明 |
|--------|-----|------|
| AH | 0x02 | 读扇区功能 |
| AL | 4 | 读取扇区数 |
| CH | 0 | 磁道号 |
| CL | 2 | 起始扇区号 |
| DH | 0 | 磁头号 |
| DL | 0 | 驱动器号 |
| ES:BX | 0x9000:0x0200 | 缓冲区地址 |

### 3. 加载 System 模块（内核）

**定义位置**：**`boot/bootsect.s:97-101`**

```asm
	mov	ax, #SYSSEG     ; ES = 0x1000
	mov	es, ax
	call	read_it        ; 读取内核
	call	kill_motor     ; 关闭软驱马达
```

#### 什么是 System 模块？

System 模块是 Linux 0.11 内核的核心部分，它是由多个子模块链接而成的 32 位保护模式代码。System 模块包含了内核的所有功能代码，是操作系统的主体。

**System 模块大小限制**：

**定义位置**：**`boot/bootsect.s:11`**

```asm
SYSSIZE = 0x3000    ; 0x30000 字节 = 192KB (最大限制)
```

#### System 模块的组成

System 模块由以下部分链接而成：


#### 加载 System 模块到内存

**read_it 函数**：**`boot/bootsect.s:150-195`**

```asm
read_it:
	mov ax, es
	test ax, #0x0fff
die:	jne die         ; ES 必须在 64KB 边界上
	xor bx, bx          ; BX = 0, 缓冲区偏移
rp_read:
	mov ax, es
	cmp ax, #ENDSEG     ; 是否加载完毕?
	jb ok1_read
	ret                 ; 加载完成
ok1_read:
	; ... 计算读取扇区数 ...
	call read_track     ; 读取磁道
	; ... 更新位置 ...
	jmp rp_read
```

## setup.s - 硬件设置

### 文件位置

**定义位置**：**`boot/setup.s`**

### 主要功能

1. 获取硬件参数（光标位置、内存大小、显卡参数等）
2. 获取硬盘参数
3. 将内核移动到 0x00000
4. 初始化 8259A 中断控制器
5. 启用 A20 地址线
6. 进入保护模式

### 1. 获取硬件参数

**定义位置**：**`boot/setup.s:30-65`**

```asm
start:
	mov	ax, #INITSEG
	mov	ds, ax

	; 获取光标位置
	mov	ah, #0x03
	xor	bh, bh
	int	0x10
	mov	[0], dx          ; 保存到 0x90000

	; 获取扩展内存大小 (KB)
	mov	ah, #0x88
	int	0x15
	mov	[2], ax          ; 保存到 0x90002

	; 获取显卡数据
	mov	ah, #0x0f
	int	0x10
	mov	[4], bx          ; 显示页
	mov	[6], ax          ; 视频模式, 窗口宽度

	; 检查 EGA/VGA 参数
	mov	ah, #0x12
	mov	bl, #0x10
	int	0x10
	mov	[8], ax
	mov	[10], bx
	mov	[12], cx
```

**硬件参数保存位置**：

| 地址 | 内容 | 说明 |
|------|------|------|
| 0x90000 | 光标位置 | 2字节 |
| 0x90002 | 扩展内存大小 | 2字节，单位KB |
| 0x90004 | 显示页 | 2字节 |
| 0x90006 | 视频模式 | 2字节 |
| 0x90008 | 显卡特征 | 2字节 |
| 0x90080 | 硬盘0参数表 | 16字节 |
| 0x90090 | 硬盘1参数表 | 16字节 |
| 0x901FC | 根设备号 | 2字节 |

### 2. 获取硬盘参数

**定义位置**：**`boot/setup.s:50-75`**

```asm
	; 获取 hd0 数据
	mov	ax, #0x0000
	mov	ds, ax
	lds	si, [4*0x41]        ; 中断向量 0x41 指向硬盘参数表
	mov	ax, #INITSEG
	mov	es, ax
	mov	di, #0x0080
	mov	cx, #0x10
	rep	movsb               ; 复制到 0x90080

	; 获取 hd1 数据
	mov	ax, #0x0000
	mov	ds, ax
	lds	si, [4*0x46]        ; 中断向量 0x46
	mov	ax, #INITSEG
	mov	es, ax
	mov	di, #0x0090
	mov	cx, #0x10
	rep	movsb               ; 复制到 0x90090
```

### 3. 移动内核到 0x00000

**定义位置**：**`boot/setup.s:87-102`**

```asm
	cli                     ; 关中断

	mov	ax, #0x0000
	cld                     ; 方向标志 = 0 (向前)
do_move:
	mov	es, ax              ; 目的段
	add	ax, #0x1000
	cmp	ax, #0x9000
	jz	end_move            ; 移动完成
	mov	ds, ax              ; 源段
	sub	di, di
	sub	si, si
	mov	cx, #0x8000         ; 32KB
	rep	movsw               ; 移动
	jmp	do_move
```

### 4. 初始化 8259A 中断控制器

**定义位置**：**`boot/setup.s:157-188`**

```asm
	; 初始化 8259A-1 (主片)
	mov	al, #0x11           ; ICW1: 边沿触发, 级联模式
	out	#0x20, al
	.word	0x00eb, 0x00eb
	out	#0xA0, al           ; 8259A-2 (从片)
	
	mov	al, #0x20           ; ICW2: 中断号起始 0x20
	out	#0x21, al
	mov	al, #0x28           ; 从片中断号起始 0x28
	out	#0xA1, al
	
	mov	al, #0x04           ; ICW3: 主片 IR2 接从片
	out	#0x21, al
	mov	al, #0x02           ; 从片接主片 IR2
	out	#0xA1, al
	
	mov	al, #0x01           ; ICW4: 8086 模式
	out	#0x21, al
	out	#0xA1, al
	
	mov	al, #0xFF           ; 屏蔽所有中断
	out	#0x21, al
	out	#0xA1, al
```

**8259A 中断号映射**：

```
主片 (8259A-1):
IRQ0 (定时器)    → INT 0x20
IRQ1 (键盘)      → INT 0x21
IRQ2 (级联)      → INT 0x22
IRQ3 (串口2)     → INT 0x23
IRQ4 (串口1)     → INT 0x24
IRQ5 (并口2)     → INT 0x25
IRQ6 (软驱)      → INT 0x26
IRQ7 (并口1)     → INT 0x27

从片 (8259A-2):
IRQ8 (实时时钟)  → INT 0x28
IRQ9            → INT 0x29
IRQ10           → INT 0x2A
IRQ11           → INT 0x2B
IRQ12           → INT 0x2C
IRQ13 (协处理器) → INT 0x2D
IRQ14 (硬盘)     → INT 0x2E
IRQ15           → INT 0x2F
```

### 5. 启用 A20 地址线

**定义位置**：**`boot/setup.s:107-117`**

```asm
	call	empty_8042
	mov	al, #0xD1          ; 写命令
	out	#0x64, al
	call	empty_8042
	mov	al, #0xDF          ; A20 ON
	out	#0x60, al
	call	empty_8042

empty_8042:
	.word	0x00eb, 0x00eb
	in	al, #0x64          ; 读取状态
	test	al, #2             ; 输入缓冲区满?
	jnz	empty_8042         ; 等待
	ret
```

**A20 地址线的作用**：
- 8086/8088 只有 20 根地址线，地址超过 1MB 会回绕
- 80286+ 有更多地址线，需要启用 A20 访问超过 1MB 的内存
- 通过键盘控制器（8042）的端口控制 A20

### 6. 进入保护模式并跳转到 System 模块

**定义位置**：**`boot/setup.s:191-194`**

```asm
	mov	ax, #0x0001        ; PE 位 (保护模式启用)
	lmsw	ax                ; 加载机器状态字
	jmpi	0, 8              ; 跳转到段选择子 8, 偏移 0
```

**lmsw 指令**：
- 加载 CR0 寄存器的低 16 位
- 设置 PE 位（位 0）启用保护模式

**段选择子 8**：
```
段选择子格式: 15       3  2  1  0
              ├────────┤ ├┤ ├┤
              │ 索引   │TI│RPL│
              └────────┘ └┘ └┘

选择子 8 = 0b0000_0000_0000_1000
        = 索引 1, TI=0 (GDT), RPL=0
        → GDT 中第 1 个描述符 (代码段)
```

#### 跳转到 System 模块开头（head.s）

当 setup.s 执行 `jmpi 0, 8` 时，CPU 跳转到：

```
跳转目标计算:
┌─────────────────────────────────────────────────────────────────────────┐
│ 段选择子 8 → GDT[1] → 代码段描述符                                        │
│                                                                         │
│ GDT[1] 代码段描述符:                                                      │
```

```asm
	.word	0, 0, 0, 0         ; 描述符 0: 空描述符

	.word	0x07FF             ; 段限长 = 8MB

	.word	0x9A00             ; 基地址 [23:16], P=1, DPL=0, 代码/执行
	.word	0x00C0             ; 基地址 [31:24], G=1, 32位

	; 描述符 2: 数据段
	.word	0x07FF             ; 段限长 = 8MB
	.word	0x0000             ; 基地址 [15:0]
	.word	0x9200             ; P=1, DPL=0, 数据/读写
	.word	0x00C0             ; G=1, 32位

	.word	0x800              ; GDT 限长 = 2048
	.word	512+gdt, 0x9       ; GDT 基地址
```

```asm
	movl	$0x00080000, %eax     ; 段选择子 = 0x0008 (代码段)
```

### 5. 初始化页表

**定义位置**：**`boot/head.s:136-167`**

```asm
setup_paging:
	movl	$1024*5, %ecx      ; 5 页 (页目录 + 4 个页表)
	xorl	%eax, %eax
	xorl	%edi, %edi         ; 页目录在 0x0000
	cld
	rep	stosl                ; 清零

	; 设置页目录项
	movl	$pg0+7, _pg_dir       ; 页表0, P=1, R/W=1, U/S=1
	movl	$pg1+7, _pg_dir+4     ; 页表1
	movl	$pg2+7, _pg_dir+8     ; 页表2
	movl	$pg3+7, _pg_dir+12    ; 页表3

	; 设置页表项 (恒等映射)
	movl	$pg3+4092, %edi
	movl	$0xfff007, %eax       ; 16MB - 4096 + 7
	std
1:
	stosl                        ; 填充页表项
	subl	$0x1000, %eax
	jge	1b

	; 启用分页
	xorl	%eax, %eax
	movl	%eax, %cr3            ; 页目录基址
	movl	%cr0, %eax
	orl	$0x80000000, %eax     ; 设置 PG 位
	movl	%eax, %cr0
	ret
```


### 2. main() 函数

**定义位置**：**`init/main.c:108-140`**

```c
void main(void)
{
	ROOT_DEV = ORIG_ROOT_DEV;
	drive_info = DRIVE_INFO;
	
	/* 计算内存大小 */
	memory_end = (1<<20) + (EXT_MEM_K<<10);
	memory_end &= 0xfffff000;
	if (memory_end > 16*1024*1024)
		memory_end = 16*1024*1024;
	
	/* 设置缓冲区边界 */
	if (memory_end > 12*1024*1024) 
		buffer_memory_end = 4*1024*1024;
	else if (memory_end > 6*1024*1024)
		buffer_memory_end = 2*1024*1024;
	else
		buffer_memory_end = 1*1024*1024;
	
	main_memory_start = buffer_memory_end;
	
	/* 初始化各子系统 */
	mem_init(main_memory_start, memory_end);  /* 内存管理 */
	trap_init();                              /* 中断处理 */
	blk_dev_init();                           /* 块设备 */
	chr_dev_init();                           /* 字符设备 */
	tty_init();                               /* 终端 */
	time_init();                              /* 时间 */
	sched_init();                             /* 调度 */
	buffer_init(buffer_memory_end);           /* 缓冲区 */
	hd_init();                                /* 硬盘 */
	floppy_init();                            /* 软驱 */
	
	sti();                                    /* 开中断 */
	move_to_user_mode();                      /* 切换到用户态 */
	
	if (!fork()) {                            /* 创建进程1 */
		init();
	}
	
	for(;;) pause();                          /* 进程0 空闲循环 */
}
```

## 分页机制初始化

### 分页概述

分页机制将线性地址转换为物理地址，提供：
- 更灵活的内存管理
- 内存保护
- 虚拟内存支持

### 地址转换过程

```
线性地址 (32位)
┌─────────────────┬─────────────────┬─────────────────┐
│  页目录索引      │   页表索引       │   页内偏移       │
│   (10位)        │    (10位)       │    (12位)       │
└────────┬────────┴────────┬────────┴────────┬────────┘
         │                 │                 │
         ▼                 │                 │
    ┌─────────┐            │                 │
    │ 页目录  │            │                 │
    └────┬────┘            │                 │
         │                 ▼                 │
         │            ┌─────────┐            │
         └───────────►│  页表   │            │
                      └────┬────┘            │
                           │                 ▼
                           │            ┌─────────┐
                           └───────────►│ 物理页面│
                                        └─────────┘
```

### 页表初始化代码

```asm
setup_paging:
	; 清零页目录和页表
	movl	$1024*5, %ecx
	xorl	%eax, %eax
	xorl	%edi, %edi
	rep	stosl

	; 设置页目录项指向页表
	movl	$pg0+7, _pg_dir      ; P=1, R/W=1, U/S=1
	movl	$pg1+7, _pg_dir+4
	movl	$pg2+7, _pg_dir+8
	movl	$pg3+7, _pg_dir+12

	; 设置页表项 (恒等映射)
	movl	$pg3+4092, %edi
	movl	$0xfff007, %eax      ; 16MB - 4096 + 属性
	std
1:
	stosl
	subl	$0x1000, %eax
	jge	1b

	; 启用分页
	movl	%eax, %cr3           ; 页目录基址
	movl	%cr0, %eax
	orl	$0x80000000, %eax    ; PG 位
	movl	%eax, %cr0
	ret
```

### 恒等映射

Linux 0.11 初始化时使用恒等映射：
- 线性地址 = 物理地址
- 映射范围：0 - 16MB
- 每个页面属性：P=1, R/W=1, U/S=1

## 总结

### 启动阶段对比

| 阶段 | 文件 | 模式 | 地址 | 主要功能 |
|------|------|------|------|----------|
| 1 | bootsect.s | 实模式 | 0x7C00→0x90000 | 加载内核 |
| 2 | setup.s | 实模式→保护模式 | 0x90200 | 硬件检测、模式切换 |
| 3 | head.s | 保护模式 | 0x00000 | 初始化 GDT/IDT/页表 |
| 4 | main.c | 保护模式 | 内核空间 | 初始化子系统 |

### 关键技术点

| 技术 | 说明 |
|------|------|
| 实模式 | 16位，1MB 地址空间，无保护 |
| 保护模式 | 32位，4GB 地址空间，段级保护 |
| 分页 | 4KB 页面，二级页表，线性地址转换 |
| A20 地址线 | 访问超过 1MB 内存 |
| 8259A | 中断控制器，管理硬件中断 |
| GDT | 全局描述符表，定义段 |
| IDT | 中断描述符表，定义中断处理 |

### 关键代码文件

| 文件 | 功能 |
|------|------|
| **`boot/bootsect.s`** | 引导扇区 |
| **`boot/setup.s`** | 硬件设置 |
| **`boot/head.s`** | 32位启动 |
| **`init/main.c`** | 内核初始化 |

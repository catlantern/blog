## 1. 文件系统的学习意义

### 1.1 文件系统的重要性

文件系统是操作系统中**最贴近用户**的组成部分，理解文件系统对于系统开发至关重要：

```
+-------------------+     +-------------------+     +-------------------+
|    应用程序       |---->|    文件系统       |---->|    内核           |
+-------------------+     +-------------------+     +-------------------+
        |                         |                         |
        v                         v                         v
+-------------------+     +-------------------+     +-------------------+
| 存储在文件系统中  |     | 提供存储和访问    |     | 提供底层支持      |
| 使用文件系统功能  |     | 管理数据和资源    |     | 硬件驱动          |
+-------------------+     +-------------------+     +-------------------+
```

### 1.2 文件系统的应用场景

| 应用场景 | 说明 |
|---------|------|
| **系统界面** | 桌面环境、窗口管理器、主题配置 |
| **开机画面** | 启动动画、Logo 显示、启动进度 |
| **系统 ROM** | 只读系统分区、固件存储 |
| **系统功能** | 系统服务、守护进程、工具程序 |
| **预装软件** | 厂商预装应用、系统组件 |

### 1.3 嵌入式开发中的文件系统

大多数嵌入式公司都在做文件系统相关的工作：

```
嵌入式产品开发流程：
+-------------+     +-------------+     +-------------+     +-------------+
| 硬件设计    |---->| 内核移植    |---->| 文件系统    |---->| 应用开发    |
+-------------+     +-------------+     +-------------+     +-------------+
                                              |
                    +-------------------------+-------------------------+
                    |                         |                         |
            +-------v-------+         +-------v-------+         +-------v-------+
            | 根文件系统    |         | 系统配置      |         | 用户应用      |
            | busybox       |         | /etc/*        |         | /usr/bin/*    |
            +---------------+         +---------------+         +---------------+
```

```
glibc 功能层次：
+-------------------+

+---------+---------+
          |

+-------------------+

+---------+---------+
          |
          v
+-------------------+
|   系统调用封装    |  <-- syscall() 封装
|  read, write      |
|  open, close      |
+---------+---------+
          |
          v
+-------------------+
```

## 4. 配置文件系统

### 4.1 /etc 目录结构

```
/etc/
├── init.d/              # 启动脚本目录
│   └── rcS              # 主启动脚本
├── inittab              # init 进程配置
├── fstab                # 文件系统挂载表
├── passwd               # 用户账户信息
├── group                # 用户组信息
├── shadow               # 用户密码（加密）
├── hostname             # 主机名
├── hosts                # 主机名解析
├── profile              # 全局环境变量
├── shells               # 可用 shell 列表
└── sysconfig/           # 系统配置目录
    └── network-scripts/ # 网络配置
```

### 4.2 /etc/init.d/rcS 启动脚本

rcS 是系统启动时执行的第一个脚本：

```bash
#!/bin/sh

# 挂载虚拟文件系统
mount -t proc proc /proc
mount -t sysfs sysfs /sys
mount -t devtmpfs devtmpfs /dev

# 创建必要的设备节点
mknod /dev/console c 5 1
mknod /dev/null c 1 3
mknod /dev/tty0 c 4 0

# 设置主机名
hostname MyEmbeddedDevice

# 配置网络
ifconfig eth0 192.168.1.100 netmask 255.255.255.0 up
route add default gw 192.168.1.1

# 启动系统服务
telnetd &
httpd &

# 显示启动信息
echo "System started successfully!"
```

### 4.3 /etc/inittab 配置

inittab 定义 init 进程的行为：

```
# inittab 格式: <id>:<runlevels>:<action>:<process>

# 系统初始化
::sysinit:/etc/init.d/rcS

# 启动 getty（登录终端）
::askfirst:-/bin/sh
tty1::askfirst:-/bin/sh
ttyS0::askfirst:-/bin/sh

# 重启和关机
::ctrlaltdel:/sbin/reboot
::shutdown:/bin/umount -a -r

# 重启时执行
::restart:/sbin/init
```

## 5. 设备节点管理

### 5.1 设备节点概念

设备节点是用户空间访问硬件设备的接口：

```
设备节点属性：
+-------------------+-------------------+-------------------+
|    设备类型       |    主设备号       |    次设备号       |
+-------------------+-------------------+-------------------+
|    字符设备(c)    |    驱动程序标识   |    具体设备标识   |
|    块设备(b)      |                   |                   |
+-------------------+-------------------+-------------------+

示例：
crw-rw-rw- 1 root root 5, 1 Jan 1 00:00 console
brw-rw---- 1 root disk 8, 0 Jan 1 00:00 sda
          ↑           ↑   ↑
          类型        主  次
```

### 5.2 常用设备节点

| 设备节点 | 类型 | 主设备号 | 次设备号 | 说明 |
|---------|------|---------|---------|------|
| /dev/console | 字符 | 5 | 1 | 系统控制台 |
| /dev/null | 字符 | 1 | 3 | 空设备（丢弃所有写入） |
| /dev/zero | 字符 | 1 | 5 | 零设备（读取返回 0） |
| /dev/random | 字符 | 1 | 8 | 随机数设备 |
| /dev/tty | 字符 | 5 | 0 | 当前终端 |
| /dev/tty0 | 字符 | 4 | 0 | 当前虚拟控制台 |
| /dev/sda | 块 | 8 | 0 | 第一块 SCSI 硬盘 |
| /dev/mmcblk0 | 块 | 179 | 0 | 第一块 MMC 设备 |

### 5.3 创建设备节点

```bash
# mknod 命令格式
mknod <name> <type> <major> <minor>

# 创建控制台设备
mknod /dev/console c 5 1

# 创建空设备
mknod /dev/null c 1 3

# 创建块设备
mknod /dev/sda b 8 0
```

### 5.4 设备节点自动创建

**方式一：devtmpfs**

```bash
# 内核配置
CONFIG_DEVTMPFS=y
CONFIG_DEVTMPFS_MOUNT=y

# 挂载
mount -t devtmpfs devtmpfs /dev
```

**方式二：mdev (Busybox)**

```bash
# 挂载 sysfs
mount -t sysfs sysfs /sys

# 启动 mdev
echo /sbin/mdev > /proc/sys/kernel/hotplug
mdev -s
```

**方式三：udev**

```bash
# udev 是更复杂的设备管理器
# 通常在桌面 Linux 系统中使用
udevd --daemon
udevadm trigger
```

## 6. Shell 实现

### 6.1 Shell 的作用

Shell 是用户与内核交互的桥梁：

```
用户输入命令流程：
+-------------+     +-------------+     +-------------+     +-------------+
|   用户      |---->|   Shell     |---->|   系统调用  |---->|   内核      |
|   输入命令  |     |   解析命令  |     |   执行请求  |     |   执行操作  |
+-------------+     +-------------+     +-------------+     +-------------+
                           |
                    +------+------+
                    |             |
                    v             v
            +-------------+ +-------------+
            | 内置命令    | | 外部命令    |
            | cd, exit    | | ls, cat     |
            +-------------+ +-------------+
```

### 6.2 Shell 命令分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **内置命令** | Shell 内部实现，不创建新进程 | cd, exit, export, source |
| **外部命令** | 独立的可执行文件 | ls, cat, grep, find |
| **函数** | 用户定义的 Shell 函数 | 自定义脚本函数 |
| **别名** | 命令的快捷方式 | ll='ls -l' |

### 6.3 Shell 命令实现原理

```c
/* 简化的 Shell 主循环 */
int main(void)
{
    char cmdline[MAX_CMD_LEN];
    char *argv[MAX_ARGS];
    int argc;
    
    while (1) {
        /* 1. 显示提示符 */
        printf("# ");
        
        /* 2. 读取命令行 */
        if (fgets(cmdline, MAX_CMD_LEN, stdin) == NULL)
            break;
        
        /* 3. 解析命令 */
        argc = parse_cmdline(cmdline, argv);
        if (argc == 0)
            continue;
        
        /* 4. 检查内置命令 */
        if (strcmp(argv[0], "cd") == 0) {
            builtin_cd(argc, argv);
            continue;
        }
        if (strcmp(argv[0], "exit") == 0) {
            exit(0);
        }
        
        /* 5. 执行外部命令 */
        if (fork() == 0) {
            /* 子进程 */
            execvp(argv[0], argv);
            perror("exec failed");
            exit(1);
        } else {
            /* 父进程等待 */
            wait(NULL);
        }
    }
    
    return 0;
}
```

## 7. 文件系统目录结构规范

### 7.1 FHS (Filesystem Hierarchy Standard)

```
根文件系统目录结构：
/
├── bin/          # 基本用户命令（所有用户可用）
│   ├── ls
│   ├── cat
│   └── sh
├── sbin/         # 系统管理命令（仅 root）
│   ├── init
│   ├── mount
│   └── ifconfig
├── lib/          # 共享库文件
│   ├── libc.so.*
│   └── ld-linux.so.*
├── etc/          # 系统配置文件
│   ├── passwd
│   ├── fstab
│   └── init.d/
├── dev/          # 设备文件
│   ├── console
│   ├── null
│   └── tty*
├── proc/         # 进程信息（虚拟文件系统）
├── sys/          # 系统信息（虚拟文件系统）
├── tmp/          # 临时文件
├── var/          # 可变数据
│   ├── log/
│   └── run/
├── usr/          # 用户程序
│   ├── bin/
│   ├── lib/
│   └── local/
├── home/         # 用户主目录
└── root/         # root 用户主目录
```

### 7.2 各目录用途详解

| 目录 | 用途 | 必需性 |
|------|------|--------|
| /bin | 基本命令，单用户模式也需要 | 必需 |
| /sbin | 系统管理命令 | 必需 |
| /lib | 基本共享库 | 必需 |
| /etc | 系统配置文件 | 必需 |
| /dev | 设备文件 | 必需 |
| /proc | 内核/进程信息 | 推荐 |
| /sys | 内核对象信息 | 推荐 |
| /tmp | 临时文件 | 推荐 |
| /var | 日志、PID 文件等 | 推荐 |
| /usr | 用户程序和数据 | 可选 |
| /home | 普通用户主目录 | 可选 |
| /root | root 主目录 | 可选 |

### 7.3 嵌入式最小文件系统

嵌入式系统通常使用最小化的文件系统：

```
最小嵌入式文件系统：

│   ├── ls -> ../bin/busybox
│   └── ...
├── sbin/         # 系统命令
│   ├── init -> ../bin/busybox
│   └── ...
├── etc/
│   ├── init.d/rcS
│   ├── inittab
│   └── fstab
├── dev/
│   ├── console
│   ├── null
│   └── tty*
├── lib/          # 必要的库
│   ├── libc.so.*
│   └── ld-*.so*
└── proc/         # 挂载点（空目录）
```

### 8.2 关键知识点

1. **文件系统是应用程序的载体**：应用程序存储在文件系统中，并使用文件系统功能
2. **配置文件控制系统行为**：/etc 目录下的配置决定系统启动和运行方式
3. **设备节点提供硬件访问**：/dev 目录下的设备文件是用户空间访问硬件的接口
4. **Shell 是用户交互入口**：所有 shell 命令都在文件系统中
5. **标准库提供运行时支持**：glibc 等库封装了系统调用，简化应用开发

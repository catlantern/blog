## 1. 核心问题解答

### 1.1 /etc/rc 和 /bin/sh 是在哪里被初始化的？

**答案：它们不是在内核中"初始化"的，而是预先存储在根文件系统镜像中的文件。**

```
关键理解：
+===========================================================================+
|                                                                           |
|   /etc/rc 和 /bin/sh 是文件系统中的普通文件                               |
|                                                                           |
|   它们不是内核创建的，而是：                                               |
|   1. 在制作根文件系统镜像时就已经存在                                      |
|   2. 存储在硬盘（或软盘）的文件系统分区中                                  |
|   3. 内核只是"挂载"文件系统，然后"读取"这些文件                           |
|                                                                           |
+===========================================================================+
```

### 1.2 为什么在 mount_root 里没看到相关初始化逻辑？

**答案：mount_root() 只负责挂载根文件系统，不负责创建文件。**

| 函数 | 作用 | 不负责 |
|------|------|--------|
| mount_root() | 读取超级块、挂载根目录 | 创建 /etc/rc、/bin/sh |
| setup() | 初始化块设备、调用 mount_root | 创建任何文件 |
| init() | 打开文件、执行程序 | 创建文件系统内容 |

### 2.2 制作文件系统镜像的方法

**方法一：使用 mkfs 工具**

```bash
# 在宿主机上制作文件系统镜像
dd if=/dev/zero of=rootimage bs=1024 count=360  # 创建空白镜像
mkfs.minix rootimage                            # 格式化为 Minix 文件系统
mount -o loop rootimage /mnt                    # 挂载镜像
mkdir /mnt/bin /mnt/etc /mnt/dev                # 创建目录
cp sh /mnt/bin/                                 # 复制 shell
cp rc /mnt/etc/                                 # 复制启动脚本
mknod /mnt/dev/tty0 c 4 0                       # 创建设备节点
umount /mnt                                     # 卸载镜像
```

**方法二：Linux 0.11 实际使用的镜像**

Linux 0.11 发布时提供了一个预制的根文件系统镜像，包含：
- `/bin/sh` - Bourne Shell
- `/etc/rc` - 启动脚本
- `/dev/tty0` - 控制台设备

## 3. mount_root 的真正作用

### 3.1 mount_root 函数分析

定义在 **`fs/super.c`**：

```c
void mount_root(void)
{
    int i, free;
    struct super_block * p;
    struct m_inode * mi;

    /* 1. 初始化文件表 */
    for (i = 0; i < NR_FILE; i++)
        file_table[i].f_count = 0;

    /* 2. 初始化超级块数组 */
    for (p = &super_block[0]; p < &super_block[NR_SUPER]; p++) {
        p->s_dev = 0;
        p->s_lock = 0;
        p->s_wait = NULL;
    }

    /* 3. 读取根文件系统的超级块 */
    if (!(p = read_super(ROOT_DEV)))
        panic("Unable to mount root");

    /* 4. 读取根目录的 inode */
    if (!(mi = iget(ROOT_DEV, ROOT_INO)))
        panic("Unable to read root i-node");

    /* 5. 设置挂载点 */
    mi->i_count += 3;
    p->s_isup = p->s_imount = mi;

    /* 6. 设置当前进程的根目录和当前目录 */
    current->pwd = mi;
    current->root = mi;

    /* 7. 统计空闲块和空闲 inode */
    /* ... */
}
```

### 3.2 mount_root 做了什么

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 初始化文件表 | 清空 file_table |
| 2 | 初始化超级块数组 | 清空 super_block |
| 3 | 读取超级块 | 从磁盘读取文件系统元数据 |
| 4 | 读取根 inode | 获取根目录的 inode |
| 5 | 设置挂载点 | 关联超级块和根 inode |
| 6 | 设置进程目录 | 设置 current->root 和 current->pwd |

### 3.3 mount_root 不做什么

| 不做的事情 | 原因 |
|-----------|------|
| 创建 /bin/sh | 文件已存在于文件系统镜像中 |
| 创建 /etc/rc | 文件已存在于文件系统镜像中 |
| 创建 /dev/tty0 | 设备节点已存在于文件系统镜像中 |
| 格式化文件系统 | 文件系统已预先格式化 |

## 4. 文件查找机制

### 4.1 namei 函数 - 路径名解析

定义在 **`fs/namei.c`**：

```c
struct m_inode * namei(const char * pathname)
{
    const char * basename;
    int inr, dev, namelen;
    struct m_inode * dir;
    struct buffer_head * bh;
    struct dir_entry * de;

    /* 1. 获取父目录 inode 和文件名 */
    if (!(dir = dir_namei(pathname, &namelen, &basename)))
        return NULL;
    
    /* 2. 如果是根目录，直接返回 */
    if (!namelen)
        return dir;

    /* 3. 在目录中查找文件 */
    bh = find_entry(&dir, basename, namelen, &de);
    if (!bh) {
        iput(dir);
        return NULL;
    }

    /* 4. 获取文件 inode 号 */
    inr = de->inode;
    dev = dir->i_dev;
    brelse(bh);
    iput(dir);

    /* 5. 读取文件 inode */
    dir = iget(dev, inr);
    if (dir) {
        dir->i_atime = CURRENT_TIME;
        dir->i_dirt = 1;
    }
    return dir;
}
```

## 5. execve 执行流程

### 5.1 execve 系统调用

定义在 **`fs/exec.c`**：

```c
int do_execve(unsigned long * eip, long tmp, char * filename,
    char ** argv, char ** envp)
{
    struct m_inode * inode;
    struct buffer_head * bh;
    struct exec ex;
    unsigned long page[MAX_ARG_PAGES];
    int i, argc, envc;
    int e_uid, e_gid;
    int retval;
    int sh_bang = 0;
    unsigned long p = PAGE_SIZE * MAX_ARG_PAGES - 4;

    /* 1. 清空参数页 */
    for (i = 0; i < MAX_ARG_PAGES; i++)
        page[i] = 0;

    /* 2. 获取可执行文件的 inode */
    if (!(inode = namei(filename)))
        return -ENOENT;

    /* 3. 统计参数和环境变量数量 */
    argc = count(argv);
    envc = count(envp);

    /* 4. 检查文件类型和权限 */
    if (!S_ISREG(inode->i_mode)) {
        retval = -EACCES;
        goto exec_error2;
    }

    /* 5. 读取可执行文件头 */
    if (!(bh = bread(inode->i_dev, inode->i_zone[0]))) {
        retval = -EACCES;
        goto exec_error2;
    }
    ex = *((struct exec *) bh->b_data);

    /* 6. 检查脚本文件 #! */
    if ((bh->b_data[0] == '#') && (bh->b_data[1] == '!') && (!sh_bang)) {
        /* 处理脚本文件 */
        /* ... */
    }

    /* 7. 检查可执行文件格式 */
    if (N_MAGIC(ex) != ZMAGIC || ex.a_trsize || ex.a_drsize ||
        ex.a_text + ex.a_data + ex.a_bss > 0x3000000 ||
        inode->i_size < ex.a_text + ex.a_data + ex.a_syms + N_TXTOFF(ex)) {
        retval = -ENOEXEC;
        goto exec_error2;
    }

    /* 8. 复制参数和环境变量 */
    if (!sh_bang) {
        p = copy_strings(envc, envp, page, p, 0);
        p = copy_strings(argc, argv, page, p, 0);
    }

    /* 9. 设置新进程的内存空间 */
    if (current->executable)
        iput(current->executable);
    current->executable = inode;

    /* 10. 清除信号处理函数 */
    for (i = 0; i < 32; i++)
        current->sigaction[i].sa_handler = NULL;

    /* 11. 关闭执行时关闭的文件 */
    for (i = 0; i < NR_OPEN; i++)
        if ((current->close_on_exec >> i) & 1)
            sys_close(i);
    current->close_on_exec = 0;

    /* 12. 释放旧的内存页表 */
    free_page_tables(get_base(current->ldt[1]), get_limit(0x0f));
    free_page_tables(get_base(current->ldt[2]), get_limit(0x17));

    /* 13. 设置新的 LDT */
    p += change_ldt(ex.a_text, page) - MAX_ARG_PAGES * PAGE_SIZE;

    /* 14. 创建参数表 */
    p = (unsigned long) create_tables((char *) p, argc, envc);

    /* 15. 设置进程信息 */
    current->brk = ex.a_bss + (current->end_data = ex.a_data + 
                   (current->end_code = ex.a_text));
    current->start_stack = p & 0xfffff000;
    current->euid = e_uid;
    current->egid = e_gid;

    /* 16. 设置新的执行入口 */
    eip[0] = ex.a_entry;    /* eip = 程序入口地址 */
    eip[3] = p;             /* esp = 栈指针 */

    return 0;

exec_error2:
    iput(inode);
exec_error1:
    for (i = 0; i < MAX_ARG_PAGES; i++)
        free_page(page[i]);
    return retval;
}
```

### 6.3 按需加载（Demand Loading）

Linux 0.11 实现了按需加载机制：

```c
/*
 * 可执行文件的 inode 保存在 current->executable 中
 * 当进程访问未加载的页面时，触发缺页中断
 * 缺页中断处理程序从文件中读取相应页面
 */
current->executable = inode;  /* 保存可执行文件的 inode */
```

### 7.2 init() 函数详解

定义在 **`init/main.c`**：

```c
static char * argv_rc[] = { "/bin/sh", NULL };
static char * envp_rc[] = { "HOME=/", NULL };

static char * argv[] = { "-/bin/sh", NULL };
static char * envp[] = { "HOME=/usr/root", NULL };

void init(void)
{
    int pid, i;

    /* 1. 挂载根文件系统 */
    setup((void *) &drive_info);

    /* 2. 打开控制台 */
    (void) open("/dev/tty0", O_RDWR, 0);
    (void) dup(0);  /* stdout */
    (void) dup(0);  /* stderr */

    /* 3. 打印内存信息 */
    printf("%d buffers = %d bytes buffer space\n\r", NR_BUFFERS,
        NR_BUFFERS * BLOCK_SIZE);
    printf("Free mem: %d bytes\n\r", memory_end - main_memory_start);

    /* 4. 执行启动脚本 */
    if (!(pid = fork())) {
        close(0);
        if (open("/etc/rc", O_RDONLY, 0))
            _exit(1);
        execve("/bin/sh", argv_rc, envp_rc);
        _exit(2);
    }
    if (pid > 0)
        while (pid != wait(&i))
            /* nothing */;

    /* 5. 启动登录 shell */
    while (1) {
        if ((pid = fork()) < 0) {
            printf("Fork failed in init\r\n");
            continue;
        }
        if (!pid) {
            close(0); close(1); close(2);
            setsid();
            (void) open("/dev/tty0", O_RDWR, 0);
            (void) dup(0);
            (void) dup(0);
            _exit(execve("/bin/sh", argv, envp));
        }
        while (1)
            if (pid == wait(&i))
                break;
        printf("\n\rchild %d died with code %04x\n\r", pid, i);
        sync();
    }
    _exit(0);
}
```

## 8. 总结

### 8.1 关键理解

| 问题 | 答案 |
|------|------|
| /etc/rc 和 /bin/sh 在哪里初始化？ | 不是内核初始化的，是文件系统镜像中预存的 |
| 为什么 mount_root 没有相关逻辑？ | mount_root 只挂载文件系统，不创建文件 |
| 文件从哪里来？ | 在制作文件系统镜像时预先创建 |
| execve 如何执行？ | 查找 inode → 读取文件头 → 设置内存 → 跳转执行 |

### 8.2 关键函数

| 函数 | 文件 | 作用 |
|------|------|------|
| mount_root() | super.c | 挂载根文件系统 |
| namei() | namei.c | 路径名解析，获取 inode |
| iget() | inode.c | 读取 inode |
| bread() | buffer.c | 读取数据块 |
| do_execve() | exec.c | 执行程序 |

### 8.3 文件来源总结

```
文件来源：
```

## 1. 核心问题解答

### 1.1 超级块是什么时候被创建出来的？

**答案：超级块是在文件系统格式化时创建的，不是内核运行时创建的。**

```
关键理解：
+===========================================================================+
|                                                                           |
|   超级块的创建时机：                                                      |
|                                                                           |
|   1. 文件系统格式化时（mkfs 命令）                                        |
|      - 在宿主机上执行 mkfs.minix /dev/hd6                                |
|      - 创建超级块、inode 位图、数据块位图、根目录                         |
|                                                                           |
|   2. 内核运行时                                                           |
|      - read_super() 只是"读取"已存在的超级块                             |
|      - 不是"创建"超级块                                                  |
|                                                                           |
+===========================================================================+
```

### 1.2 Linux 是怎么读取文件系统镜像的？

**答案：通过块设备驱动和缓冲区机制读取硬盘数据。**

```
读取流程：
+===========================================================================+
|                                                                           |
|   read_super(ROOT_DEV)                                                    |
|         |                                                                 |
|         v                                                                 |
|   bread(dev, 1)              <-- 读取第 1 块（超级块）                    |
|         |                                                                 |
|         v                                                                 |
|   getblk(dev, 1)             <-- 获取缓冲区                               |
|         |                                                                 |
|         v                                                                 |
|   ll_rw_block(READ, bh)      <-- 发起块设备读请求                         |
|         |                                                                 |
|         v                                                                 |
|   hd_request()               <-- 硬盘驱动处理请求                         |
|         |                                                                 |
|         v                                                                 |
|   硬盘中断                   <-- 数据读取完成                             |
|         |                                                                 |
|         v                                                                 |
|   返回缓冲区                 <-- 数据在 bh->b_data 中                     |
|                                                                           |
+===========================================================================+
```

## 2. 超级块是什么时候创建的

### 2.1 文件系统格式化过程

**超级块是在文件系统格式化时创建的**，这是在宿主机上完成的操作，不是内核运行时的操作。

```
文件系统格式化流程：
+===========================================================================+
|                                                                           |
|   宿主机上执行：                                                          |
|   $ mkfs.minix /dev/hd6                                                   |
|                                                                           |
|   格式化过程：                                                            |
|   +------------------+                                                    |
|   | 1. 创建超级块    |  写入第 1 块                                       |
|   |    s_magic=0x137F|                                                    |
|   |    s_ninodes=... |                                                    |
|   |    s_nzones=...  |                                                    |
|   +------------------+                                                    |
|            |                                                              |
|            v                                                              |
|   +------------------+                                                    |
|   | 2. 创建 inode    |  写入第 2~N 块                                     |
|   |    位图          |  每位表示一个 inode 是否空闲                       |
|   +------------------+                                                    |
|            |                                                              |
|            v                                                              |
|   +------------------+                                                    |
|   | 3. 创建数据块    |  写入后续块                                        |
|   |    位图          |  每位表示一个数据块是否空闲                        |
|   +------------------+                                                    |
|            |                                                              |
|            v                                                              |
|   +------------------+                                                    |
|   | 4. 创建根目录    |  创建 inode 1                                      |
|   |    inode         |  创建根目录数据块                                  |
|   +------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 3.2 内存中的超级块结构

```c
struct super_block {
    /* 从磁盘读取的字段 */
    unsigned short s_ninodes;
    unsigned short s_nzones;
    unsigned short s_imap_blocks;
    unsigned short s_zmap_blocks;
    unsigned short s_firstdatazone;
    unsigned short s_log_zone_size;
    unsigned long  s_max_size;
    unsigned short s_magic;

    /* 仅在内存中存在的字段 */
    struct buffer_head * s_imap[8];  /* inode 位图缓冲区 */
    struct buffer_head * s_zmap[8];  /* 数据块位图缓冲区 */

    struct m_inode * s_isup;         /* 被挂载的 inode */
    struct m_inode * s_imount;       /* 挂载点 inode */
    unsigned long s_time;
    struct task_struct * s_wait;
    unsigned char s_lock;            /* 锁定标志 */
    unsigned char s_rd_only;         /* 只读标志 */
    unsigned char s_dirt;            /* 脏标志 */
};
```

| 字段 | 大小 | 说明 |
|------|------|------|
| s_ninodes | 2 字节 | 文件系统中 inode 的总数 |
| s_nzones | 2 字节 | 文件系统中数据块的总数 |
| s_imap_blocks | 2 字节 | inode 位图占用的块数 |
| s_zmap_blocks | 2 字节 | 数据块位图占用的块数 |

| s_log_zone_size | 2 字节 | 日志区大小（Linux 0.11 未使用） |
| s_max_size | 4 字节 | 文件最大大小 |
| s_magic | 2 字节 | 魔数，Minix 文件系统为 0x137F |

## 4. read_super 函数分析

### 4.1 read_super 完整代码

定义在 **`fs/super.c`**：

```c
static struct super_block * read_super(int dev)
{
    struct super_block * s;
    struct buffer_head * bh;
    int i, block;

    if (!dev)
        return NULL;

    /* 1. 检查磁盘是否更换 */
    check_disk_change(dev);

    /* 2. 检查是否已读取过 */
    if (s = get_super(dev))
        return s;

    /* 3. 找一个空闲的超级块槽位 */
    for (s = 0 + super_block; ; s++) {
        if (s >= NR_SUPER + super_block)
            return NULL;
        if (!s->s_dev)
            break;
    }

    /* 4. 初始化超级块 */
    s->s_dev = dev;
    s->s_isup = NULL;
    s->s_imount = NULL;
    s->s_time = 0;
    s->s_rd_only = 0;
    s->s_dirt = 0;

    /* 5. 锁定超级块 */
    lock_super(s);

    /* 6. 读取磁盘上的超级块（第 1 块） */
    if (!(bh = bread(dev, 1))) {
        s->s_dev = 0;
        free_super(s);
        return NULL;
    }

    /* 7. 复制超级块数据 */
    *((struct d_super_block *) s) =
        *((struct d_super_block *) bh->b_data);
    brelse(bh);

    /* 8. 验证魔数 */
    if (s->s_magic != SUPER_MAGIC) {
        s->s_dev = 0;
        free_super(s);
        return NULL;
    }

    /* 9. 初始化位图指针 */
    for (i = 0; i < I_MAP_SLOTS; i++)
        s->s_imap[i] = NULL;
    for (i = 0; i < Z_MAP_SLOTS; i++)
        s->s_zmap[i] = NULL;

    /* 10. 读取 inode 位图 */
    block = 2;
    for (i = 0; i < s->s_imap_blocks; i++)
        if (s->s_imap[i] = bread(dev, block))
            block++;
        else
            break;

    /* 11. 读取数据块位图 */
    for (i = 0; i < s->s_zmap_blocks; i++)
        if (s->s_zmap[i] = bread(dev, block))
            block++;
        else
            break;

    /* 12. 检查是否读取完整 */
    if (block != 2 + s->s_imap_blocks + s->s_zmap_blocks) {
        for (i = 0; i < I_MAP_SLOTS; i++)
            brelse(s->s_imap[i]);
        for (i = 0; i < Z_MAP_SLOTS; i++)
            brelse(s->s_zmap[i]);
        s->s_dev = 0;
        free_super(s);
        return NULL;
    }

    /* 13. 标记 inode 0 和数据块 0 为已使用 */
    s->s_imap[0]->b_data[0] |= 1;
    s->s_zmap[0]->b_data[0] |= 1;

    /* 14. 解锁超级块 */
    free_super(s);

    return s;
}
```

## 5. 文件系统镜像读取流程

### 5.1 bread 函数 - 读取数据块

定义在 **`fs/buffer.c`**：

```c
struct buffer_head * bread(int dev, int block)
{
    struct buffer_head * bh;

    /* 1. 获取缓冲区 */
    if (!(bh = getblk(dev, block)))
        panic("bread: getblk returned NULL\n");

    /* 2. 如果数据已有效，直接返回 */
    if (bh->b_uptodate)
        return bh;

    /* 3. 发起块设备读请求 */
    ll_rw_block(READ, bh);

    /* 4. 等待读取完成 */
    wait_on_buffer(bh);

    /* 5. 检查是否成功 */
    if (bh->b_uptodate)
        return bh;

    /* 6. 失败则释放缓冲区 */
    brelse(bh);
    return NULL;
}
```

## 6. 块设备读取机制

### 6.1 ll_rw_block 函数

定义在 **`kernel/blk_drv/ll_rw_blk.c`**：

```c
void ll_rw_block(int rw, struct buffer_head * bh)
{
    unsigned int major;

    /* 1. 检查设备号 */
    if ((major = MAJOR(bh->b_dev)) >= NR_BLK_DEV ||
        !(blk_dev[major].request_fn)) {
        printk("Trying to read nonexistent block-device\n\r");
        return;
    }

    /* 2. 创建请求 */
    make_request(major, rw, bh);
}

static void make_request(int major, int rw, struct buffer_head * bh)
{
    struct request * req;

    /* 1. 锁定缓冲区 */
    lock_buffer(bh);

    /* 2. 找一个空闲的请求槽位 */
    /* ... */

    /* 3. 填充请求信息 */
    req->dev = bh->b_dev;
    req->cmd = rw;
    req->sector = bh->b_blocknr << 1;  /* 块号转扇区号 */
    req->nr_sectors = 2;               /* 每块 2 个扇区 */
    req->buffer = bh->b_data;
    req->bh = bh;

    /* 4. 添加到请求队列 */
    add_request(major + blk_dev, req);
}
```

### 6.2 块设备请求结构

```c
struct request {
    int dev;                /* 设备号 */
    int cmd;                /* READ 或 WRITE */
    int errors;             /* 错误计数 */
    unsigned long sector;   /* 起始扇区号 */
    unsigned long nr_sectors; /* 扇区数 */
    char * buffer;          /* 数据缓冲区 */
    struct task_struct * waiting; /* 等待进程 */
    struct buffer_head * bh; /* 缓冲区头 */
    struct request * next;  /* 下一个请求 */
};
```

```c
/* 硬盘中断处理 */
void hd_interrupt(void)
{
    /* 1. 读取硬盘数据 */
    port_read(HD_DATA, CURRENT->buffer, 256);

    /* 2. 唤醒等待进程 */
```

## 8. 总结

### 8.1 关键理解

| 问题 | 答案 |
|------|------|
| 超级块何时创建？ | 文件系统格式化时（mkfs 命令） |
| read_super 做什么？ | 从磁盘读取已存在的超级块 |
| 如何读取文件系统镜像？ | 通过块设备驱动读取硬盘数据 |
| ROOT_DEV 从哪来？ | setup.s 从 BIOS 读取，保存到 0x901FC |

### 8.2 关键函数

| 函数 | 文件 | 作用 |
|------|------|------|
| read_super() | super.c | 读取超级块 |
| bread() | buffer.c | 读取数据块 |
| ll_rw_block() | ll_rw_blk.c | 发起块设备请求 |
| do_hd_request() | hd.c | 硬盘驱动处理 |
| hd_interrupt() | hd.c | 硬盘中断处理 |

### 8.3 文件系统镜像读取总结

```
文件系统镜像读取总结：
+===========================================================================+

|   1. 超级块：                                                             |
```

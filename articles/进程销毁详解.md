## 进程销毁概述

### 什么是进程销毁

进程销毁是指进程结束执行后，系统回收其占用的资源并从系统中移除的过程。进程销毁包括：

- 释放进程占用的内存
- 关闭打开的文件
- 通知父进程
- 回收进程控制块

### 进程销毁的时机

| 触发条件 | 说明 |
|----------|------|
| 正常退出 | 调用 exit() 或从 main 返回 |
| 异常退出 | 收到致命信号（如 SIGSEGV） |
| 内核错误 | 发生不可恢复的内核错误 |
| 被其他进程杀死 | 通过 kill 系统调用 |

### 进程销毁流程概览

```
进程调用 exit()
      │
      ▼
sys_exit() → do_exit()
      │
      ├─ 释放内存页表
      ├─ 关闭打开的文件
      ├─ 处理子进程（过继给 init）
      ├─ 设置僵尸状态
      └─ 通知父进程
      │
      ▼
父进程调用 wait()
      │
      ▼
release() → 释放 task_struct
```

## exit系统调用

### 1. 系统调用入口

**定义位置**：**`kernel/exit.c:137-139`**

```c
int sys_exit(int error_code)
{
	return do_exit((error_code&0xff)<<8);
}
```

**参数处理**：
- `error_code` 是用户传入的退出码（0-255）
- 左移 8 位是为了与信号导致的退出区分
- 最终退出码格式：`(error_code & 0xff) << 8`

### 2. 退出码格式

```
退出码格式（16位）：
┌─────────────────────────────────────┐
│  高8位：退出状态  │  低8位：终止信号  │
├─────────────────────────────────────┤
│  正常退出：(code<<8) │      0        │
│  信号退出：   0      │   signal      │
└─────────────────────────────────────┘
```

### 3. 系统调用表注册

**定义位置**：**`include/linux/sys.h:74`**

```c
fn_ptr sys_call_table[] = { 
    sys_setup,   // 0

    // ...
};
```

## do_exit核心函数

### do_exit() 完整实现

**定义位置**：**`kernel/exit.c:102-135`**

```c
int do_exit(long code)
{
	int i;

	/* 1. 释放进程的内存页表 */
	free_page_tables(get_base(current->ldt[1]), get_limit(0x0f));
	free_page_tables(get_base(current->ldt[2]), get_limit(0x17));
	
	/* 2. 处理子进程（过继给进程1） */
	for (i = 0; i < NR_TASKS; i++)
		if (task[i] && task[i]->father == current->pid) {
			task[i]->father = 1;  /* 过继给 init 进程 */
			if (task[i]->state == TASK_ZOMBIE)
				/* 通知 init 进程 */
				(void) send_sig(SIGCHLD, task[1], 1);
		}
	
	/* 3. 关闭所有打开的文件 */
	for (i = 0; i < NR_OPEN; i++)
		if (current->filp[i])
			sys_close(i);
	
	/* 4. 释放 inode 资源 */
	iput(current->pwd);
	current->pwd = NULL;
	iput(current->root);
	current->root = NULL;
	iput(current->executable);
	current->executable = NULL;
	
	/* 5. 释放终端控制 */
	if (current->leader && current->tty >= 0)
		tty_table[current->tty].pgrp = 0;
	
	/* 6. 清除协处理器状态 */
	if (last_task_used_math == current)
		last_task_used_math = NULL;
	
	/* 7. 如果是会话组长，终止会话 */
	if (current->leader)
		kill_session();
	
	/* 8. 设置僵尸状态 */
	current->state = TASK_ZOMBIE;
	current->exit_code = code;
	
	/* 9. 通知父进程 */
	tell_father(current->father);
	
	/* 10. 调度其他进程运行 */
	schedule();
	
	return (-1);	/* 不会执行到这里 */
}
```

### do_exit() 执行步骤详解

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 释放页表 | 回收进程占用的内存页表 |
| 2 | 处理子进程 | 将子进程过继给 init 进程 |
| 3 | 关闭文件 | 关闭所有打开的文件描述符 |
| 4 | 释放 inode | 释放当前目录、根目录、执行文件 |
| 5 | 释放终端 | 如果是控制终端，释放控制权 |
| 6 | 清除协处理器 | 清除 FPU 状态 |
| 7 | 终止会话 | 如果是会话组长，向会话进程发送 SIGHUP |
| 8 | 设置僵尸 | 将状态设为 TASK_ZOMBIE |
| 9 | 通知父进程 | 发送 SIGCHLD 信号 |
| 10 | 调度 | 调用 schedule() 切换进程 |

## 僵尸进程

### 什么是僵尸进程

僵尸进程（Zombie Process）是指已经执行完毕，但其父进程尚未调用 wait() 收集其退出状态的进程。

### 僵尸进程的特点

- 进程状态为 `TASK_ZOMBIE`
- task_struct 仍然存在
- 内存和文件资源已释放
- 保留退出状态供父进程查询

### 僵尸进程的创建

```c
/* 在 do_exit() 中设置僵尸状态 */
current->state = TASK_ZOMBIE;
current->exit_code = code;
tell_father(current->father);
schedule();  /* 进程不再运行，但 task_struct 仍存在 */
```

### 僵尸进程的危害

| 影响 | 说明 |
|------|------|
| 占用进程槽 | task 数组中的槽位被占用 |
| 资源泄漏 | 如果大量僵尸进程累积，可能导致无法创建新进程 |
| 系统性能 | 影响进程调度效率 |

### 避免僵尸进程

1. **父进程及时调用 wait()**
2. **使用信号处理 SIGCHLD**
3. **将子进程过继给 init**

## wait系统调用

### sys_waitpid() 函数

**定义位置**：**`kernel/exit.c:141-195`**

```c
int sys_waitpid(pid_t pid, unsigned long * stat_addr, int options)
{
	int flag, code;
	struct task_struct ** p;

	verify_area(stat_addr, 4);
repeat:
	flag = 0;
	
	/* 遍历所有进程，查找符合条件的子进程 */
	for(p = &LAST_TASK; p > &FIRST_TASK; --p) {
		if (!*p || *p == current)
			continue;
		if ((*p)->father != current->pid)
			continue;
		
		/* 根据 pid 参数筛选 */
		if (pid > 0) {
			if ((*p)->pid != pid)
				continue;
		} else if (!pid) {
			if ((*p)->pgrp != current->pgrp)
				continue;
		} else if (pid != -1) {
			if ((*p)->pgrp != -pid)
				continue;
		}
		
		/* 根据子进程状态处理 */
		switch ((*p)->state) {
			case TASK_STOPPED:
				if (!(options & WUNTRACED))
					continue;
				put_fs_long(0x7f, stat_addr);
				return (*p)->pid;
				
			case TASK_ZOMBIE:
				/* 累加子进程的时间统计 */
				current->cutime += (*p)->utime;
				current->cstime += (*p)->stime;
				flag = (*p)->pid;
				code = (*p)->exit_code;
				
				/* 释放僵尸进程 */
				release(*p);
				put_fs_long(code, stat_addr);
				return flag;
				
			default:
				flag = 1;  /* 有运行中的子进程 */
				continue;
		}
	}
	
	/* 处理等待逻辑 */
	if (flag) {
		if (options & WNOHANG)
			return 0;  /* 不阻塞，立即返回 */
		
		/* 阻塞等待子进程状态变化 */
		current->state = TASK_INTERRUPTIBLE;
		schedule();
		
		/* 检查是否收到信号 */
		if (!(current->signal &= ~(1<<(SIGCHLD-1))))
			goto repeat;
		else
			return -EINTR;
	}
	
	return -ECHILD;  /* 没有子进程 */
}
```

### waitpid 参数说明

| 参数 | 说明 |
|------|------|
| `pid` | 进程 ID 筛选条件 |
| `stat_addr` | 存储退出状态的地址 |
| `options` | 选项标志 |

**pid 参数含义**：

| 值 | 含义 |
|-----|------|
| `pid > 0` | 等待指定 PID 的子进程 |
| `pid == 0` | 等待与当前进程同组的子进程 |
| `pid == -1` | 等待任意子进程 |
| `pid < -1` | 等待进程组 ID 为 -pid 的子进程 |

**options 标志**：

| 标志 | 说明 |
|------|------|
| `WNOHANG` | 非阻塞，如果没有子进程退出则立即返回 0 |
| `WUNTRACED` | 报告停止的子进程 |

### wait() 系统调用

wait() 是 waitpid() 的简化版本：

```c
int sys_wait(int *stat_addr)
{
    return sys_waitpid(-1, stat_addr, 0);
}
```

### wait 的工作流程

```
父进程调用 wait()
        │
        ▼
遍历子进程列表
        │
        ├─ 发现僵尸子进程？
        │       │
        │       ├─ 是 → 累加时间统计
        │       │      释放 task_struct
        │       │      返回子进程 PID
        │       │
        │       └─ 否 → 继续
        │
        ├─ 有运行中的子进程？
        │       │
        │       ├─ 是 → 阻塞等待
        │       │      收到 SIGCHLD 后重试
        │       │
        │       └─ 否 → 返回 -ECHILD
        │
        └─ WNOHANG 标志？
                │
                ├─ 是 → 立即返回 0
                │
                └─ 否 → 阻塞等待
```

## 孤儿进程处理

### 什么是孤儿进程

孤儿进程是指父进程已经退出，但子进程仍在运行的进程。

### 孤儿进程的处理

**定义位置**：**`kernel/exit.c:108-115`**

```c
/* 在 do_exit() 中处理子进程 */
for (i = 0; i < NR_TASKS; i++)
	if (task[i] && task[i]->father == current->pid) {
		task[i]->father = 1;  /* 过继给 init 进程（进程1） */
		if (task[i]->state == TASK_ZOMBIE)
			/* 通知 init 进程回收僵尸子进程 */
			(void) send_sig(SIGCHLD, task[1], 1);
	}
```

### 处理流程

```
父进程退出
    │
    ▼
遍历所有进程
    │
    ├─ 发现子进程？
    │       │
    │       ├─ 是 → 将 father 设为 1（init 进程）
    │       │      如果子进程是僵尸，通知 init
    │       │
    │       └─ 否 → 继续
    │
    ▼
init 进程负责回收孤儿进程
```

### init 进程的责任

进程1（init）负责：
- 回收所有孤儿进程
- 调用 wait() 清理僵尸子进程
- 确保系统不会累积僵尸进程

## 信号与进程终止

### 致命信号

某些信号会导致进程终止：

| 信号 | 编号 | 说明 |
|------|------|------|
| SIGKILL | 9 | 强制终止（不可捕获） |
| SIGSEGV | 11 | 段错误 |
| SIGABRT | 6 | 异常终止 |
| SIGTERM | 15 | 正常终止请求 |
| SIGHUP | 1 | 挂起 |
| SIGBUS | 7 | 总线错误 |
| SIGFPE | 8 | 浮点异常 |

### 信号处理导致退出

**定义位置**：**`kernel/signal.c:89-119`**

```c
void do_signal(long signr, long eax, long ebx, long ecx, long edx,
	long fs, long es, long ds,
	long eip, long cs, long eflags,
	unsigned long * esp, long ss)
{
	unsigned long sa_handler;
	long old_eip = eip;
	struct sigaction * sa = current->sigaction + signr - 1;
	int longs;
	unsigned long * tmp_esp;

	sa_handler = (unsigned long) sa->sa_handler;
	
	/* 如果信号处理函数为 SIG_IGN (1)，忽略信号 */
	if (sa_handler == 1)
		return;
	
	/* 如果信号处理函数为 SIG_DFL (0)，执行默认动作 */
	if (!sa_handler) {
		if (signr == SIGCHLD)
			return;  /* 忽略 SIGCHLD */
		else
			do_exit(1<<(signr-1));  /* 其他信号导致退出 */
	}
	
	/* 设置信号处理函数执行环境 */
	if (sa->sa_flags & SA_ONESHOT)
		sa->sa_handler = NULL;
	
	*(&eip) = sa_handler;  /* 跳转到信号处理函数 */
	
	/* 在用户栈上设置返回信息 */
	longs = (sa->sa_flags & SA_NOMASK) ? 7 : 8;
	*(&esp) -= longs;
	verify_area(esp, longs * 4);
	tmp_esp = esp;
	
	put_fs_long((long) sa->sa_restorer, tmp_esp++);
	put_fs_long(signr, tmp_esp++);
	if (!(sa->sa_flags & SA_NOMASK))
		put_fs_long(current->blocked, tmp_esp++);
	put_fs_long(eax, tmp_esp++);
	put_fs_long(ecx, tmp_esp++);
	put_fs_long(edx, tmp_esp++);
	put_fs_long(eflags, tmp_esp++);
	put_fs_long(old_eip, tmp_esp++);
	
	current->blocked |= sa->sa_mask;
}
```

### kill 系统调用

**定义位置**：**`kernel/exit.c:48-68`**

```c
int sys_kill(int pid, int sig)
{
	struct task_struct **p = NR_TASKS + task;
	int err, retval = 0;

	/* pid == 0: 发送给同组进程 */
	if (!pid) while (--p > &FIRST_TASK) {
		if (*p && (*p)->pgrp == current->pid) 
			if (err = send_sig(sig, *p, 1))
				retval = err;
	}
	/* pid > 0: 发送给指定进程 */
	else if (pid > 0) while (--p > &FIRST_TASK) {
		if (*p && (*p)->pid == pid) 
			if (err = send_sig(sig, *p, 0))
				retval = err;
	}
	/* pid == -1: 发送给所有进程 */
	else if (pid == -1) while (--p > &FIRST_TASK)
		if (err = send_sig(sig, *p, 0))
			retval = err;
	/* pid < -1: 发送给指定进程组 */
	else while (--p > &FIRST_TASK)
		if (*p && (*p)->pgrp == -pid)
			if (err = send_sig(sig, *p, 0))
				retval = err;
	
	return retval;
}
```

### 信号导致进程终止流程

```
┌─────────────────────────────────────────────────────────────┐
│  进程收到致命信号（如 SIGSEGV, SIGKILL）                      │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  系统调用返回时检查信号                                       │
│  ret_from_system_call:                                      │
│  - 检查 current->signal                                     │
│  - 调用 do_signal()                                         │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  do_signal(signr, ...)                                      │
│  - 检查信号处理函数                                          │
│  - 如果为 SIG_DFL (0)：                                      │
│    - SIGCHLD: 忽略                                          │
│    - 其他: do_exit(1<<(signr-1))                            │
│  - 如果为 SIG_IGN (1)：忽略                                  │
│  - 否则：执行用户定义的处理函数                               │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  do_exit(信号码)                                             │
│  - 进入正常退出流程                                          │
└─────────────────────────────────────────────────────────────┘
```

### 孤儿进程处理流程

```
┌─────────────────────────────────────────────────────────────┐
│  父进程退出 (do_exit)                                        │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  遍历所有进程                                                │
│  for (i = 0; i < NR_TASKS; i++)                             │
│      if (task[i] && task[i]->father == current->pid)        │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  发现子进程                                                  │
│  - task[i]->father = 1  (过继给 init)                        │
│  - 如果子进程是僵尸：                                        │
│    - send_sig(SIGCHLD, task[1], 1)                          │
│    - 通知 init 进程回收                                      │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  init 进程 (进程1)                                           │
│  - 在循环中调用 wait()                                       │
│  - 回收所有孤儿进程                                          │
│  - 确保系统不会累积僵尸进程                                   │
└─────────────────────────────────────────────────────────────┘
```

## 总结

### 进程销毁的关键步骤

| 步骤 | 函数 | 说明 |
|------|------|------|
| 1 | sys_exit() | 系统调用入口 |
| 2 | do_exit() | 核心退出处理 |
| 3 | free_page_tables() | 释放内存 |
| 4 | sys_close() | 关闭文件 |
| 5 | iput() | 释放 inode |
| 6 | tell_father() | 通知父进程 |
| 7 | schedule() | 切换进程 |
| 8 | sys_waitpid() | 父进程回收 |
| 9 | release() | 释放 task_struct |

### 进程状态转换

```
TASK_RUNNING → do_exit() → TASK_ZOMBIE → wait() → 完全销毁
                    ↑
            信号终止 │
                    │
            TASK_STOPPED (可被 wait 回收)
```

### 关键代码文件

| 文件 | 功能 |
|------|------|
| **`kernel/exit.c`** | 进程退出和等待 |
| **`kernel/signal.c`** | 信号处理 |
| **`mm/memory.c`** | 内存释放 |
| **`include/linux/sched.h`** | 进程状态定义 |

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 僵尸进程累积 | 父进程未调用 wait | 父进程处理 SIGCHLD |
| 孤儿进程 | 父进程先于子进程退出 | 过继给 init 进程 |
| 进程无法退出 | 资源未正确释放 | 确保所有资源正确关闭 |

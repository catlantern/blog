## 1. MMU 概述

### 1.1 什么是 MMU？

```
MMU（Memory Management Unit）定义：
+===========================================================================+
|                                                                           |
|   MMU 是一种硬件组件，位于 CPU 和内存之间                                  |
|                                                                           |
|   核心职责：                                                              |
|   +-------------------+                                                    |
|   | 1. 地址转换       |  虚拟地址 → 物理地址                              |
|   | 2. 内存保护       |  权限检查、隔离保护                               |
|   | 3. 缓存管理       |  TLB 缓存加速                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 1.2 为什么需要 MMU？

```
没有 MMU 的问题：
+===========================================================================+
|                                                                           |
|   问题 1：内存安全问题                                                    |
|   +-------------------+                                                    |
|   | 程序直接访问物理内存 |                                                 |
|   | 可以修改任意内存区域 |                                                 |
|   | 一个程序崩溃可能影响其他程序 |                                         |
|   +-------------------+                                                    |
|                                                                           |
|   问题 2：内存碎片化                                                      |
|   +-------------------+                                                    |
|   | 物理内存不连续     |                                                  |
|   | 大程序难以分配连续空间 |                                               |
|   | 内存利用率低       |                                                  |
|   +-------------------+                                                    |
|                                                                           |
|   问题 3：多进程管理困难                                                  |
|   +-------------------+                                                    |
|   | 进程间无隔离       |                                                  |
|   | 无法实现虚拟内存   |                                                  |
|   | 内存共享困难       |                                                  |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

有 MMU 后的解决方案：
+===========================================================================+
|                                                                           |
|   解决方案 1：虚拟地址空间                                                |
|   +-------------------+                                                    |
|   | 每个进程独立地址空间 |                                                 |
|   | 进程间相互隔离     |                                                  |
|   | 安全性大幅提升     |                                                  |
|   +-------------------+                                                    |
|                                                                           |
|   解决方案 2：分页机制                                                    |
|   +-------------------+                                                    |
|   | 虚拟地址可以映射到不连续物理页 |                                       |
|   | 解决外部碎片问题   |                                                  |
|   | 内存利用率提高     |                                                  |
|   +-------------------+                                                    |
|                                                                           |
|   解决方案 3：内存保护                                                    |
|   +-------------------+                                                    |
|   | 权限位控制访问     |                                                  |
|   | 内核/用户空间隔离  |                                                  |
|   | 非法访问触发异常   |                                                  |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

## 2. 虚拟地址与物理地址

### 2.1 地址类型对比

```
虚拟地址 vs 物理地址：
+===========================================================================+
|                                                                           |
|   虚拟地址（Virtual Address）：                                           |
|   +-------------------+                                                    |
|   | CPU 生成的地址    |  程序中使用的地址                                 |
|   | 每个进程独立      |  0 ~ TASK_SIZE                                   |
|   | 连续的假象        |  由 MMU 转换                                     |
|   +-------------------+                                                    |
|                                                                           |
|   物理地址（Physical Address）：                                          |
|   +-------------------+                                                    |
|   | 内存芯片实际地址  |  硬件真实地址                                     |
|   | 全局唯一          |  0 ~ 物理内存大小                                |
|   | CPU 无法直接访问  |  必须经过 MMU                                    |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 2.2 Linux 地址空间布局

```
Linux 64 位虚拟地址空间布局：
+===========================================================================+
|                                                                           |
|   地址范围                    用途                                        |
|   +-------------------+-------------------+                               |
|   | 0x0000000000000000 | 用户空间         |  128 TB                       |
|   | ~0x00007FFFFFFFFFFF | (User Space)    |                               |
|   +-------------------+-------------------+                               |
|   | 0x0000800000000000 | 非规范地址       |  空洞                         |
|   | ~0xFFFF7FFFFFFFFFFF | (Non-Canonical) |                               |
|   +-------------------+-------------------+                               |
|   | 0xFFFF800000000000 | 内核空间         |  128 TB                       |
|   | ~0xFFFFFFFFFFFFFFFF | (Kernel Space)  |                               |
|   +-------------------+-------------------+                               |
|                                                                           |
|   用户空间细分：                                                          |
|   +-------------------+-------------------+                               |
|   | 代码段 (.text)    | 只读、可执行     |                               |
|   | 数据段 (.data)    | 可读、可写       |                               |
|   | BSS 段 (.bss)     | 未初始化数据     |                               |
|   | 堆 (heap)         | 动态分配         |  向上增长                     |
|   | 内存映射 (mmap)   | 共享库、文件映射 |                               |
|   | 栈 (stack)        | 局部变量         |  向下增长                     |

|                                                                           |
```

## 4. 地址转换机制

### 4.1 页式地址转换

```
页式地址转换原理：
+===========================================================================+
|                                                                           |
|   页面大小：4KB（常见）                                                    |
|   +-------------------+                                                    |
|   | 页面大小 = 4096 字节 |                                                 |
|   | 页内偏移 = 12 位     |  log2(4096) = 12                               |
|   +-------------------+                                                    |
|                                                                           |
|   虚拟地址结构（以 48 位虚拟地址为例）：                                   |
|   +-------------------+-------------------+-------------------+            |
|   | VPN（虚拟页号）   | Offset（页内偏移）|                   |            |
|   |     36 位         |      12 位        |                   |            |
|   +-------------------+-------------------+-------------------+            |
|                                                                           |
|   地址转换公式：                                                          |
|   +-------------------+                                                    |
|   | 虚拟页号 = 虚拟地址 / 页面大小                                        |
|   | 页内偏移 = 虚拟地址 % 页面大小                                        |
|   | 物理地址 = 物理页框号 × 页面大小 + 页内偏移                           |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 4.2 地址转换示例代码

```c
#include <stdio.h>
#include <stdint.h>
#include <stdbool.h>

#define PAGE_SIZE       4096
#define PAGE_SHIFT      12
#define TLB_SIZE        8

typedef struct {
    uint64_t virtual_page;
    uint64_t physical_page;
    bool valid;
} PageTableEntry;

PageTableEntry tlb[TLB_SIZE];
PageTableEntry page_table[1024];

void mmu_init(void)
{
    for (int i = 0; i < TLB_SIZE; i++) {
        tlb[i].valid = false;
    }
    
    page_table[0x00] = (PageTableEntry){0x00, 0x1000, true};
    page_table[0x01] = (PageTableEntry){0x01, 0x2000, true};
    page_table[0x02] = (PageTableEntry){0x02, 0x3000, true};
}

bool tlb_lookup(uint64_t virtual_page, uint64_t *physical_page)
{
    for (int i = 0; i < TLB_SIZE; i++) {
        if (tlb[i].valid && tlb[i].virtual_page == virtual_page) {
            *physical_page = tlb[i].physical_page;
            return true;
        }
    }
    return false;
}

void tlb_add(uint64_t virtual_page, uint64_t physical_page)
{
    static int next = 0;
    tlb[next] = (PageTableEntry){virtual_page, physical_page, true};
    next = (next + 1) % TLB_SIZE;
}

void page_fault_handler(uint64_t virtual_page)
{
    printf("[缺页异常] 虚拟页 0x%lX 未映射\n", virtual_page);
}

uint64_t mmu_translate(uint64_t virtual_addr)
{
    printf("CPU 发出虚拟地址: 0x%016lX\n", virtual_addr);
    
    uint64_t virtual_page = virtual_addr >> PAGE_SHIFT;
    uint64_t offset = virtual_addr & (PAGE_SIZE - 1);
    
    printf("  虚拟页号: 0x%lX, 页内偏移: 0x%lX\n", virtual_page, offset);
    
    uint64_t physical_page;
    
    if (tlb_lookup(virtual_page, &physical_page)) {
        printf("  TLB 命中: 物理页框号 0x%lX\n", physical_page);
    } else {
        printf("  TLB 未命中，查询页表\n");
        
        if (virtual_page >= 1024 || !page_table[virtual_page].valid) {
            page_fault_handler(virtual_page);
            return 0;
        }
        
        physical_page = page_table[virtual_page].physical_page;
        printf("  页表查询成功: 物理页框号 0x%lX\n", physical_page);
        tlb_add(virtual_page, physical_page);
    }
    
    uint64_t physical_addr = (physical_page << PAGE_SHIFT) | offset;
    printf("  物理地址: 0x%016lX\n\n", physical_addr);
    
    return physical_addr;
}

int main(void)
{
    mmu_init();
    
    mmu_translate(0x00001234);
    mmu_translate(0x00001ABC);
    mmu_translate(0x00003000);
    
    return 0;
}
```

## 5. 多级页表结构

### 5.1 为什么需要多级页表？

```
单级页表的问题：
+===========================================================================+
|                                                                           |
|   假设：                                                                  |
|   +-------------------+                                                    |
|   | 虚拟地址空间 = 48 位 |  256 TB                                        |
|   | 页面大小 = 4KB       |                                                |
|   | 页表项大小 = 8 字节  |                                                |
|   +-------------------+                                                    |
|                                                                           |
|   单级页表大小：                                                          |
|   +-------------------+                                                    |
|   | 页表项数量 = 2^36    |  64G 个                                        |
|   | 页表大小 = 64G × 8B  |  512 GB !!!                                    |
|   +-------------------+                                                    |
|                                                                           |
|   问题：                                                                  |
|   +-------------------+                                                    |
|   | 内存占用过大        |  每个进程都需要                                 |
|   | 大部分空间未使用    |  稀疏地址空间                                   |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+

多级页表的解决方案：
+===========================================================================+
|                                                                           |
|   Linux x86_64 四级页表：                                                 |
|   +-------------------+                                                    |
|   | PGD (Page Global Directory) |  页全局目录                             |
|   | P4D (Page 4th Level)        |  第四级页表                             |
|   | PUD (Page Upper Directory)  |  页上级目录                             |
|   | PMD (Page Middle Directory) |  页中级目录                             |
|   | PTE (Page Table Entry)      |  页表项                                 |
|   +-------------------+                                                    |
|                                                                           |
|   优势：                                                                  |
|   +-------------------+                                                    |
|   | 按需分配页表        |  只分配使用的部分                               |
|   | 内存占用小          |  稀疏地址空间高效                               |
|   | 支持大页面          |  减少页表层级                                   |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 5.2 四级页表地址划分

```

|                                                                           |

|   +--------+--------+--------+--------+--------+--------+                |
|   | Sign   | PGD    | PUD    | PMD    | PTE    | Offset |                |
|   | Extend | 9 bits | 9 bits | 9 bits | 9 bits | 12 bits|                |
|   | 16 bits|        |        |        |        |        |                |
|   +--------+--------+--------+--------+--------+--------+                |
|    63-48    47-39    38-30    29-21    20-12    11-0                     |
|                                                                           |
|   每级页表项数量：                                                        |
|   +-------------------+                                                    |
|   | 2^9 = 512 个      |                                                    |
|   +-------------------+                                                    |
|                                                                           |
|   每级页表大小：                                                          |
|   +-------------------+                                                    |
|   | 512 × 8 字节 = 4KB |  正好一页                                        |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

```
|       |                                                                   |
|       v                                                                   |
|   +-------------------+                                                    |
|   | PMD (页中级目录)   |  ← 使用 PMD 索引                                 |
|   +-------------------+                                                    |
|       |                                                                   |
|       v                                                                   |
|   +-------------------+                                                    |
|   | PTE (页表项)       |  ← 使用 PTE 索引                                 |

|       |                                                                   |
```

### 5.4 页表项权限位定义

```c
#define _PAGE_USER      (_AT(pteval_t, 1) << _PAGE_BIT_USER)
#define _PAGE_ACCESSED  (_AT(pteval_t, 1) << _PAGE_BIT_ACCESSED)
#define _PAGE_DIRTY     (_AT(pteval_t, 1) << _PAGE_BIT_DIRTY)
```

## 6. TLB 快表缓存

### 6.1 TLB 概述

```
TLB（Translation Lookaside Buffer）：
+===========================================================================+
|                                                                           |
|   TLB 是 MMU 内部的高速缓存                                               |
|                                                                           |
|   作用：                                                                  |
|   +-------------------+                                                    |
|   | 缓存最近使用的页表项 |                                                 |
|   | 加速地址转换过程     |                                                 |
|   | 减少内存访问次数     |                                                 |
|   +-------------------+                                                    |
|                                                                           |
|   结构：                                                                  |
|   +-------------------+                                                    |
|   | 虚拟页号 | 物理页框号 | 权限位 | 有效位 |                              |
|   +-------------------+                                                    |
|                                                                           |
|   特点：                                                                  |
|   +-------------------+                                                    |
|   | 容量小（几十到几百条）|                                                |
|   | 速度极快            |  硬件实现                                       |
|   | 自动管理            |  对软件透明                                     |
|   +-------------------+                                                    |
|                                                                           |
+===========================================================================+
```

### 6.2 TLB 命中与未命中

```
TLB 查找流程：
+===========================================================================+
|                                                                           |
|   CPU 访问虚拟地址                                                        |
|       |                                                                   |
|       v                                                                   |
|   +-------------------+                                                    |
|   | 查询 TLB          |                                                    |
|   +-------------------+                                                    |
|       |                                                                   |
|       +-----+-----+                                                       |
|       | 命中 | 未命中                                                    |
|       v       v                                                           |
|   +--------+  +-------------------+                                        |
|   | 快速   |  | 遍历多级页表     |                                        |
|   | 路径   |  | (Page Table Walk)|                                        |
|   +--------+  +-------------------+                                        |
|       |               |                                                   |
|       |               v                                                   |
|       |       +-------------------+                                        |
|       |       | 更新 TLB          |                                        |
|       |       +-------------------+                                        |
|       |               |                                                   |
|       +-------+-------+                                                   |
|               |                                                           |
|               v                                                           |
|       +-------------------+                                                |
|       | 返回物理地址      |                                                |
|       +-------------------+                                                |
|                                                                           |
+===========================================================================+

性能对比：
+===========================================================================+
|                                                                           |
|   TLB 命中：                                                              |
|   +-------------------+                                                    |
|   | 1-2 个时钟周期    |                                                    |
|   +-------------------+                                                    |
|                                                                           |
|   TLB 未命中（需要页表遍历）：                                            |
|   +-------------------+                                                    |
|   | 四级页表：4 次内存访问 |  约 100+ 时钟周期                             |
|   +-------------------+                                                    |
|                                                                           |
|   TLB 命中率对性能影响巨大！                                              |
|                                                                           |
+===========================================================================+
```

### 6.3 TLB 类型

### 7.3 内存保护示例代码

```c
#include <stdio.h>
#include <stdint.h>
#include <stdbool.h>

#define _PAGE_PRESENT   (1 << 0)
#define _PAGE_RW        (1 << 1)
#define _PAGE_USER      (1 << 2)
#define _PAGE_NX        (1 << 3)

typedef struct {
    uint64_t physical_page;
    uint32_t flags;
    bool valid;
} PageTableEntry;

typedef enum {
    ACCESS_READ,
    ACCESS_WRITE,
    ACCESS_EXECUTE
} AccessType;

typedef enum {
    MODE_USER,
    MODE_KERNEL
} PrivilegeLevel;

bool check_permission(PageTableEntry *entry, AccessType access, 
                      PrivilegeLevel level)
{
    if (!entry->valid || !(entry->flags & _PAGE_PRESENT)) {
        printf("  [错误] 页面不存在\n");
        return false;
    }
    
    if (level == MODE_USER && !(entry->flags & _PAGE_USER)) {
        printf("  [错误] 用户态无权访问内核页面\n");
        return false;
    }
    
    switch (access) {
    case ACCESS_WRITE:
        if (!(entry->flags & _PAGE_RW)) {
            printf("  [错误] 页面不可写\n");
            return false;
        }
        break;
    case ACCESS_EXECUTE:
        if (entry->flags & _PAGE_NX) {
            printf("  [错误] 页面不可执行\n");
            return false;
        }
        break;
    case ACCESS_READ:
    default:
        break;
    }
    
    printf("  [成功] 权限检查通过\n");
    return true;
}

int main(void)
{
    PageTableEntry user_code = {0x1000, _PAGE_PRESENT | _PAGE_USER, true};
    PageTableEntry user_data = {0x2000, _PAGE_PRESENT | _PAGE_USER | _PAGE_RW, true};
    PageTableEntry kernel_data = {0x3000, _PAGE_PRESENT | _PAGE_RW, true};
    
    printf("=== 测试 1: 用户态读取代码段 ===\n");
    check_permission(&user_code, ACCESS_READ, MODE_USER);
    
    printf("\n=== 测试 2: 用户态写入代码段 ===\n");
    check_permission(&user_code, ACCESS_WRITE, MODE_USER);
    
    printf("\n=== 测试 3: 用户态写入数据段 ===\n");
    check_permission(&user_data, ACCESS_WRITE, MODE_USER);
    
    printf("\n=== 测试 4: 用户态访问内核数据 ===\n");
    check_permission(&kernel_data, ACCESS_READ, MODE_USER);
    
    printf("\n=== 测试 5: 内核态访问内核数据 ===\n");
    check_permission(&kernel_data, ACCESS_WRITE, MODE_KERNEL);
    
    return 0;
}
```

### 8.3 缺页异常处理代码

```c
/*
 * 缺页异常入口函数
 * 定义在 arch/x86/mm/fault.c
 */
DEFINE_IDTENTRY_RAW_ERRORCODE(exc_page_fault)
{
    unsigned long address = read_cr2();  /* 读取触发异常的地址 */
    irqentry_state_t state;

    prefetchw(&current->mm->mmap_lock);

    /* 处理 KVM 异步缺页 */
    if (kvm_handle_async_pf(regs, (u32)address))
        return;

    state = irqentry_enter(regs);

    instrumentation_begin();
    handle_page_fault(regs, error_code, address);
    instrumentation_end();

    irqentry_exit(regs, state);
}

/*
 * 页错误处理分发函数
 */
static __always_inline void
handle_page_fault(struct pt_regs *regs, unsigned long error_code,
                  unsigned long address)
{
    trace_page_fault_entries(regs, error_code, address);

    if (unlikely(kmmio_fault(regs, address)))
        return;

    /* 判断是内核空间还是用户空间 */
    if (unlikely(fault_in_kernel_space(address))) {
        do_kern_addr_fault(regs, error_code, address);
    } else {
        do_user_addr_fault(regs, error_code, address);
    }

    local_irq_disable();
}

/*
 * 用户空间缺页处理
 */
void do_user_addr_fault(struct pt_regs *regs,
                        unsigned long error_code,
                        unsigned long address)
{
    struct vm_area_struct *vma;
    struct task_struct *tsk;
    struct mm_struct *mm;
    vm_fault_t fault;
    unsigned int flags = FAULT_FLAG_DEFAULT;

    tsk = current;
    mm = tsk->mm;

    /* 检查保留位 */
    if (unlikely(error_code & X86_PF_RSVD))
        pgtable_bad(regs, error_code, address);

    /* 检查 SMAP 违规 */
    if (unlikely(cpu_feature_enabled(X86_FEATURE_SMAP) &&
                 !(error_code & X86_PF_USER) &&
                 !(regs->flags & X86_EFLAGS_AC))) {
        page_fault_oops(regs, error_code, address);
        return;
    }

    /* 设置错误标志 */
    if (error_code & X86_PF_WRITE)
        flags |= FAULT_FLAG_WRITE;
    if (error_code & X86_PF_INSTR)
        flags |= FAULT_FLAG_INSTRUCTION;

    /* 查找 VMA */
    vma = find_vma(mm, address);
    if (unlikely(!vma))
        return bad_area(regs, error_code, address);

    /* 处理缺页 */
    fault = handle_mm_fault(vma, address, flags, regs);

    /* 检查处理结果 */
    if (fault_signal_pending(fault, regs))
        return;

    if (unlikely(fault & VM_FAULT_ERROR)) {
        mm_fault_error(regs, error_code, address, fault);
        return;
    }
}
```

### 9.2 TLB 刷新函数

```c
/*
 * TLB 刷新函数
 * 定义在 arch/x86/mm/tlb.c
 */

/*
 * 刷新所有 TLB
 */
void flush_tlb_all(void)
{
    count_vm_tlb_event(NR_TLB_REMOTE_FLUSH);
    on_each_cpu(do_flush_tlb_all, NULL, 1);
}

/*
 * 刷新指定地址范围的 TLB
 */
void flush_tlb_mm_range(struct mm_struct *mm, unsigned long start,
                        unsigned long end, unsigned int stride_shift,
                        bool freed_tables)
{
    struct flush_tlb_info *info;
    u64 new_tlb_gen;
    int cpu;

    cpu = get_cpu();

    /* 决定刷新范围 */
    if ((end == TLB_FLUSH_ALL) ||
        ((end - start) >> stride_shift) > tlb_single_page_flush_ceiling) {
        start = 0;
        end = TLB_FLUSH_ALL;
    }

    /* 更新 TLB 生成号 */
    new_tlb_gen = inc_mm_tlb_gen(mm);

    info = get_flush_tlb_info(mm, start, end, stride_shift, freed_tables,
                              new_tlb_gen);

    /* 多核刷新 */
    if (cpumask_any_but(mm_cpumask(mm), cpu) < nr_cpu_ids) {
        info->trim_cpumask = should_trim_cpumask(mm);
        flush_tlb_multi(mm_cpumask(mm), info);
    } else if (mm == this_cpu_read(cpu_tlbstate.loaded_mm)) {
        lockdep_assert_irqs_enabled();
        local_irq_disable();
        flush_tlb_func(info);
        local_irq_enable();
    }

    put_flush_tlb_info();
    put_cpu();
}

/*
 * 刷新内核地址范围
 */
void flush_tlb_kernel_range(unsigned long start, unsigned long end)
{
    if (end == TLB_FLUSH_ALL ||
        (end - start) > tlb_single_page_flush_ceiling << PAGE_SHIFT) {
        on_each_cpu(do_flush_tlb_all, NULL, 1);
    } else {
        struct flush_tlb_info *info;

        preempt_disable();
        info = get_flush_tlb_info(NULL, start, end, 0, false, 0);

        on_each_cpu(do_kernel_range_flush, info, 1);

        put_flush_tlb_info();
        preempt_enable();
    }
}
```

## 总结

### 核心要点

1. **MMU 是内存管理的硬件基础**：
   - 实现虚拟地址到物理地址的转换
   - 提供内存保护和隔离机制
   - 通过 TLB 加速地址转换

2. **多级页表解决大地址空间问题**：
   - 四级页表：PGD → P4D → PUD → PMD → PTE
   - 按需分配，节省内存
   - 支持大页面优化

3. **TLB 是性能关键**：
   - 缓存页表映射
   - 命中时极快
   - 未命中时需要多级页表遍历

4. **内存保护保障安全**：
   - 权限位控制访问
   - 内核/用户空间隔离
   - 违规触发异常

5. **缺页异常实现按需加载**：
   - 延迟分配物理页面
   - 实现写时复制
   - 支持交换和内存映射

## 概述

### Linux 0.11 调度器特点（1991年）

Linux 0.11 采用**简单的基于时间片的优先级调度算法**，具有以下特点：

- **单一调度策略**：仅支持一种简单的轮转调度
- **进程数量限制**：最多支持 64 个进程
- **单核设计**：不支持 SMP 多核系统
- **简单高效**：代码量小，逻辑清晰
- **时间片驱动**：基于固定时间片的抢占式调度

### Linux 5.15 调度器特点（2021年）

Linux 5.15 采用**完全公平调度器（CFS）**为核心，具有以下特点：

- **多种调度类**：支持 stop、deadline、realtime、fair、idle 五种调度类
- **无进程数量限制**：支持数以万计的进程
- **SMP 多核支持**：完善的负载均衡和调度域机制
- **复杂精密**：支持 NUMA、能效感知、带宽控制等高级特性
- **虚拟运行时间驱动**：基于虚拟运行时间的公平调度

## 核心数据结构对比

### task_struct 结构体演进

#### Linux 0.11 的 task_struct

```c
struct task_struct {
    long state;             /* 进程状态 */
    long counter;           /* 时间片计数器 */
    long priority;          /* 优先级（时间片基数） */
    long signal;            /* 信号位图 */
    struct sigaction sigaction[32]; /* 信号处理 */
    long blocked;           /* 阻塞信号 */
    int exit_code;          /* 退出码 */
    unsigned long start_code, end_code, end_data, brk, start_stack;
    long pid, father, pgrp, session, leader;
    unsigned short uid, euid, suid;
    unsigned short gid, egid, sgid;
    long alarm;             /* 闹钟时间 */
    long utime, stime, cutime, cstime, start_time;
    unsigned short used_math;
    int tty;
    unsigned short umask;
    struct m_inode * pwd, * root, * executable;
    unsigned long close_on_exec;
    struct file * filp[NR_OPEN];
    struct desc_struct ldt[3];  /* 局部描述符表 */
    struct tss_struct tss;      /* 任务状态段 */
};
```

**调度相关字段**：
- `counter`：剩余时间片
- `priority`：静态优先级
- `state`：进程状态

#### Linux 5.15 的 task_struct

```c
struct task_struct {
    unsigned int            __state;
    
    int                     on_rq;
    int                     prio;
    int                     static_prio;
    int                     normal_prio;
    unsigned int            rt_priority;

```

**关键新增字段**：
- `sched_class`：调度类指针
- `se/rt/dl`：不同调度类的实体
- `static_prio/normal_prio/rt_priority`：多级优先级
- `cpus_ptr/cpus_mask`：CPU亲和性
- `nr_cpus_allowed`：允许运行的CPU数量

## 调度算法演进

### Linux 0.11：简单时间片轮转

#### 核心算法

```c
void schedule(void)
{
    int i, next, c;
    struct task_struct ** p;

    /* 1. 检查闹钟和信号 */
    for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)
        if (*p) {
            if ((*p)->alarm && (*p)->alarm < jiffies) {
                (*p)->signal |= (1<<(SIGALRM-1));
                (*p)->alarm = 0;
            }
            if (((*p)->signal & ~(_BLOCKABLE & (*p)->blocked)) &&
                (*p)->state == TASK_INTERRUPTIBLE)
                (*p)->state = TASK_RUNNING;
        }

    /* 2. 选择时间片最大的就绪进程 */
    while (1) {
        c = -1;
        next = 0;
        i = NR_TASKS;
        p = &task[NR_TASKS];
        
        while (--i) {
            if (!*--p)
                continue;
            if ((*p)->state == TASK_RUNNING && (*p)->counter > c)
                c = (*p)->counter, next = i;
        }
        
        if (c) break;
        
        /* 3. 所有进程时间片用完，重新分配 */
        for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)
            if (*p)
                (*p)->counter = ((*p)->counter >> 1) + (*p)->priority;
    }
    
    switch_to(next);
}
```

#### 算法特点

1. **时间片分配**：
   - 新时间片 = 剩余时间片/2 + 优先级
   - I/O密集型进程获得更多时间片

2. **选择策略**：
   - 简单的最大时间片选择
   - O(n)复杂度遍历所有进程

3. **调度时机**：
   - 时钟中断（100Hz）
   - 进程主动睡眠
   - 进程退出

### Linux 5.15：完全公平调度器（CFS）

#### 核心概念

**虚拟运行时间（vruntime）**：
- 每个进程维护一个vruntime
- vruntime增长速度与进程权重成反比
- 调度器选择vruntime最小的进程运行

#### 核心数据结构

```c
struct sched_entity {
    struct load_weight      load;           /* 进程权重 */
    struct rb_node          run_node;       /* 红黑树节点 */
    unsigned int            on_rq;          /* 是否在运行队列 */
    
    u64                     exec_start;     /* 开始执行时间 */
    u64                     sum_exec_runtime;   /* 总执行时间 */
    u64                     vruntime;       /* 虚拟运行时间 */
    u64                     prev_sum_exec_runtime;
    
    /* 用于组调度 */
    struct sched_entity     *parent;
    struct cfs_rq           *cfs_rq;
    struct cfs_rq           *my_q;
};
```

#### CFS运行队列

```c
struct cfs_rq {
    struct load_weight      load;           /* 总负载 */
    unsigned int            nr_running;     /* 运行进程数 */
    
    u64                     min_vruntime;   /* 最小虚拟运行时间 */
    
    struct rb_root_cached   tasks_timeline; /* 红黑树根节点 */
    
    struct sched_entity     *curr;          /* 当前运行进程 */
    struct sched_entity     *next;          /* 下一个运行进程 */
    struct sched_entity     *last;          /* 上一个运行进程 */
    
    /* 带宽控制 */
    struct rq               *rq;
    struct task_group       *tg;
};
```

#### 核心调度函数

```c
static struct task_struct *
pick_next_task_fair(struct rq *rq, struct task_struct *prev, struct rq_flags *rf)
{
    struct cfs_rq *cfs_rq = &rq->cfs;
    struct sched_entity *se;
    struct task_struct *p;
    
    if (!cfs_rq->nr_running)
        return NULL;
    
    /* 从红黑树中选择vruntime最小的进程 */
    se = pick_next_entity(cfs_rq, prev);
    set_next_entity(cfs_rq, se);
    
    p = task_of(se);
    return p;
}
```

#### 算法特点

1. **公平性**：
   - 基于虚拟运行时间实现真正的公平
   - 优先级高的进程vruntime增长慢

2. **效率**：
   - 红黑树组织进程，O(log n)复杂度
   - 缓存最左节点，快速选择

3. **调度延迟**：
   - 目标延迟：6ms * (1 + ilog(ncpus))
   - 最小粒度：0.75ms * (1 + ilog(ncpus))

### 算法对比总结

| 特性 | Linux 0.11 | Linux 5.15 CFS |
|------|-----------|----------------|
| 核心指标 | 剩余时间片 | 虚拟运行时间 |
| 数据结构 | 数组 | 红黑树 |
| 选择复杂度 | O(n) | O(log n) |
| 公平性 | 相对公平 | 完全公平 |
| 响应时间 | 固定时间片 | 动态调整 |
| 适用场景 | 单核少进程 | 多核多进程 |

## 调度类与策略

### Linux 0.11：单一调度策略

Linux 0.11 只有一种调度策略，所有进程平等对待，仅通过优先级区分。

### Linux 5.15：多调度类架构

#### 调度类层次结构

```
优先级从高到低：

stop_sched_class (最高优先级)
    ↓
dl_sched_class (Deadline调度)
    ↓
rt_sched_class (实时调度)
    ↓
fair_sched_class (CFS公平调度)
    ↓
idle_sched_class (空闲调度)
```

#### 各调度类详解

##### 1. stop_sched_class

```c
const struct sched_class stop_sched_class = {
    .next           = &dl_sched_class,
    
    .enqueue_task   = enqueue_task_stop,
    .dequeue_task   = dequeue_task_stop,
    .yield          = yield_task_stop,
    
    .pick_next_task = pick_next_task_stop,
    .put_prev_task  = put_prev_task_stop,
    
    /* ... */
};
```

**特点**：
- 最高优先级
- 用于CPU热插拔等关键操作
- 每个CPU只有一个stop任务

##### 2. dl_sched_class (Deadline)

```c
struct sched_dl_entity {
    struct rb_node          rb_node;
    
    u64                     dl_runtime;     /* 运行时间 */
    u64                     dl_deadline;    /* 相对截止时间 */
    u64                     dl_period;      /* 周期 */
    
    u64                     runtime;        /* 剩余运行时间 */
    u64                     deadline;       /* 绝对截止时间 */
    
    unsigned int            flags;
    unsigned int            dl_throttled    : 1;
    unsigned int            dl_yielded      : 1;
    unsigned int            dl_non_contending : 1;
    
    struct hrtimer          dl_timer;
};
```

**特点**：
- 基于EDF（Earliest Deadline First）算法
- 保证实时任务的截止时间
- 支持带宽预留

##### 3. rt_sched_class (Real-Time)

```c
struct sched_rt_entity {
    struct list_head        run_list;
    unsigned long           timeout;
    unsigned long           watchdog_stamp;
    unsigned int            time_slice;
    unsigned short          on_rq;
    unsigned short          on_list;
    
    struct sched_rt_entity  *back;
#ifdef CONFIG_RT_GROUP_SCHED
    struct sched_rt_entity  *parent;
    struct rt_rq            *rt_rq;
    struct rt_rq            *my_q;
#endif
};
```

**策略**：
- `SCHED_FIFO`：先进先出，不时间片
- `SCHED_RR`：时间片轮转

**特点**：
- 优先级范围：0-99（数值越小优先级越高）
- 支持优先级继承
- 带宽限制机制

##### 4. fair_sched_class (CFS)

```c
const struct sched_class fair_sched_class = {
    .next           = &idle_sched_class,
    
    .enqueue_task   = enqueue_task_fair,
    .dequeue_task   = dequeue_task_fair,
    .yield          = yield_task_fair,
    
    .pick_next_task = pick_next_task_fair,
    .put_prev_task  = put_prev_task_fair,
    
    .check_preempt_wakeup = check_preempt_wakeup_fair,
    
    .select_task_rq = select_task_rq_fair,
    .migrate_task_rq = migrate_task_rq_fair,
    
    .task_tick      = task_tick_fair,
    .task_fork      = task_fork_fair,
    
    /* ... */
};
```

**策略**：
- `SCHED_NORMAL`：普通进程
- `SCHED_BATCH`：批处理进程

##### 5. idle_sched_class

```c
const struct sched_class idle_sched_class = {
    .next           = NULL,
    
    .pick_next_task = pick_next_task_idle,
    .put_prev_task  = put_prev_task_idle,
    
    /* ... */
};
```

**特点**：
- 最低优先级
- 当没有其他进程可运行时执行
- 用于省电和CPU空闲管理

### 调度策略对比

| 特性 | Linux 0.11 | Linux 5.15 |
|------|-----------|-----------|
| 调度类数量 | 1 | 5 |
| 实时支持 | 无 | SCHED_FIFO/RR + Deadline |
| 公平调度 | 简单轮转 | CFS完全公平 |
| 空闲处理 | 简单idle | 智能idle + 省电 |
| 优先级范围 | 单一范围 | 0-139（RT: 0-99, Normal: 100-139） |

- 单一运行队列

## 任务切换机制

### Linux 0.11：TSS硬件切换

```c
#define switch_to(n) {\
struct {long a,b;} __tmp; \
__asm__("cmpl %%ecx,_current\n\t" \
    "je 1f\n\t" \
    "movw %%dx,%1\n\t" \
    "xchgl %%ecx,_current\n\t" \
    "ljmp %0\n\t" \
    "cmpl %%ecx,_last_task_used_math\n\t" \
    "jne 1f\n\t" \
    "clts\n" \
    "1:" \
    ::"m" (*&__tmp.a),"m" (*&__tmp.b), \
    "d" (_TSS(n)),"c" ((long) task[n])); \
}
```

**特点**：
- 使用 `ljmp` 指令跳转到TSS
- 硬件自动保存/恢复寄存器
- 简单但效率较低

### Linux 5.15：软件上下文切换

```c
static __always_inline struct rq *
context_switch(struct rq *rq, struct task_struct *prev,
               struct task_struct *next, struct rq_flags *rf)
{
    struct mm_struct *mm, *oldmm;

    prepare_task_switch(rq, prev, next);

    mm = next->mm;
    oldmm = prev->active_mm;
    
    if (!mm) {
        next->active_mm = oldmm;
        if (prev->mm)
            mmgrab(oldmm);
        else
            oldmm = NULL;
    } else
        mmgrab(mm);

    if (!prev->mm) {
        prev->active_mm = NULL;
        rq->prev_mm = oldmm;
    }

    rq->clock_update_flags &= ~(RQCF_ACT_SKIP|RQCF_NOCLOCK);

    if (mm) {
        membarrier_switch_mm(rq, prev->mm, mm);
        switch_mm_irqs_off(oldmm, mm, next);
    }

    switch_to(prev, next, prev);
    barrier();
    
    return finish_task_switch(prev);
}

#define switch_to(prev, next, last)                    \
do {                                                   \
    ((last) = __switch_to_asm((prev), (next)));       \
} while (0)
```

**特点**：
- 软件保存/恢复寄存器
- 内核栈切换
- 内存管理上下文切换
- 更高效，更灵活

## 时间管理与定时器

### Linux 0.11：简单定时器

#### 时钟中断

```c
#define HZ 100          /* 每秒 100 次时钟中断 */

void do_timer(long cpl)
{
    if (cpl)
        current->utime++;
    else
        current->stime++;

    if (next_timer) {
        next_timer->jiffies--;
        while (next_timer && next_timer->jiffies <= 0) {
            void (*fn)(void);
            fn = next_timer->fn;
            next_timer->fn = NULL;
            next_timer = next_timer->next;
            (fn)();
        }
    }
    
    if ((--current->counter) > 0) return;
    current->counter = 0;
    if (!cpl) return;
    schedule();
}
```

## 总结与展望

### 演进总结

从 Linux 0.11 到 Linux 5.15，进程调度机制经历了翻天覆地的变化：

#### 1. 架构演进

| 维度 | Linux 0.11 | Linux 5.15 |
|------|-----------|-----------|
| 代码规模 | ~200行 | ~20000行 |
| 调度类 | 1 | 5 |
| 数据结构 | 简单数组 | 红黑树 + 多级队列 |
| 复杂度 | O(n) | O(log n) |

#### 2. 功能演进

| 功能 | Linux 0.11 | Linux 5.15 |
|------|-----------|-----------|
| 实时调度 | 无 | RT + Deadline |
| 公平调度 | 简单轮转 | CFS |
| SMP支持 | 无 | 完整支持 |
| NUMA支持 | 无 | NUMA感知 |
| 能效优化 | 无 | EAS |

#### 3. 性能演进

| 性能指标 | Linux 0.11 | Linux 5.15 |
|---------|-----------|-----------|
| 最大进程数 | 64 | 无限制 |
| 调度延迟 | 固定 | 动态调整 |
| 切换开销 | 高（TSS） | 低（软件切换） |
| 扩展性 | 单核 | 数千CPU核心 |

### 关键创新

#### 1. 完全公平调度器（CFS）

**创新点**：
- 基于虚拟运行时间的公平调度
- 红黑树组织进程
- 动态时间片分配

**影响**：
- 彻底改变了传统时间片调度
- 提供了真正的公平性保证
- 成为Linux默认调度器

#### 2. 调度类架构

**创新点**：
- 模块化的调度类设计
- 支持多种调度策略
- 清晰的优先级层次

**影响**：
- 支持实时、普通、批处理等多种工作负载
- 便于扩展新的调度策略
- 成为其他操作系统的参考

#### 3. SMP负载均衡

**创新点**：
- 多层次调度域
- 智能负载均衡
- NUMA感知调度

**影响**：
- 支持大规模多核系统
- 提高了系统整体吞吐量
- 降低了跨NUMA访问开销

#### 4. 能效感知调度

**创新点**：
- 能耗模型集成
- 异构CPU支持
- 动态功耗管理

**影响**：
- 显著降低功耗
- 延长移动设备续航
- 支持big.LITTLE等异构架构

### 未来展望

#### 1. 异构计算支持

随着ARM big.LITTLE、Intel Lakefield等异构架构的普及，调度器需要：
- 更智能的任务放置
- 更精细的能耗模型
- 更好的负载预测

#### 2. 实时性增强

实时Linux（PREEMPT_RT）的发展方向：
- 更细粒度的锁
- 更多的可抢占点
- 更低的调度延迟

#### 3. 机器学习辅助调度

可能的创新方向：
- 基于机器学习的负载预测
- 自适应调度参数调整
- 智能任务分类

#### 4. 容器与云原生

容器化环境下的调度挑战：
- 更精细的资源隔离
- 更好的QoS保证
- 多租户公平性

### 结语

从 Linux 0.11 到 Linux 5.15，进程调度机制的发展历程展示了操作系统技术的巨大进步。从简单的时间片轮转到复杂的完全公平调度，从单核到多核NUMA，从简单的进程管理到能效感知调度，Linux调度器已经成长为一个精密、高效、可扩展的核心子系统。

这种演进不仅反映了硬件技术的发展，也体现了操作系统设计理念的进步：从简单实用到精密复杂，从单一目标到多维优化。未来，随着硬件架构的持续演进和应用场景的不断丰富，Linux调度器必将继续创新和发展。

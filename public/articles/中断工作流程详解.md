## 中断概述

### 什么是中断

中断是 CPU 暂停当前程序执行，转而去处理紧急事件的一种机制。Linux 0.11 支持以下类型的中断：

| 类型 | 向量号范围 | 说明 |
|------|-----------|------|
| 异常 | 0-31 | CPU 内部事件（除零、页错误等） |
| 硬件中断 | 32-47 | 外部设备请求（时钟、键盘、硬盘等） |
| 系统调用 | 128 (0x80) | 用户程序请求内核服务 |

### 中断处理的核心步骤

```
1. CPU 工作模式转化（如需要）
2. 寄存器的保存与压栈
3. 查找中断向量表，获取处理程序地址
4. 跳转到对应的中断服务函数执行
5. 执行完毕后恢复寄存器
6. 返回被中断的程序继续执行
```

## 中断向量表初始化

### 1. IDT 结构

**定义位置**：**`boot/head.s:229-236`**

```asm
.align 3
_idt:	.fill 256,8,0		# idt is uninitialized

_gdt:	.quad 0x0000000000000000	/* NULL descriptor */
	.quad 0x00c09a0000000fff	/* 16Mb */
	.quad 0x00c0920000000fff	/* 16Mb */
	.quad 0x0000000000000000	/* TEMPORARY - don't use */
	.fill 252,8,0			/* space for LDT's and TSS's etc */
```

**IDT 描述符格式**（每个门描述符 8 字节）：

```
字节 0-1:  段内偏移地址低 16 位
字节 2-3:  段选择子（代码段选择子）
字节 4:    保留（全 0）
字节 5:    P DPL 0 Type（存在位、特权级、类型）
字节 6-7:  段内偏移地址高 16 位
```

### 2. IDT 初始化函数

**定义位置**：**`boot/head.s:62-79`**

```asm
/*
 *  setup_idt
 *
 *  sets up a idt with 256 entries pointing to
 *  ignore_int, interrupt gates. It then loads
 *  idt. Everything that wants to install itself
 *  in the idt-table may do so themselves. Interrupts
 *  are enabled elsewhere, when we can be relatively
 *  sure everything is ok. This routine will be over-
 *  written by the page tables.
 */
setup_idt:
	lea ignore_int,%edx          ; 默认处理程序地址
	movl $0x00080000,%eax
	movw %dx,%ax                 ; selector = 0x0008 = cs
	movw $0x8E00,%dx             ; interrupt gate - dpl=0, present

	lea _idt,%edi
	mov $256,%ecx                ; 256 个中断向量
rp_sidt:
	movl %eax,(%edi)             ; 设置门描述符低 32 位
	movl %edx,4(%edi)            ; 设置门描述符高 32 位
	addl $8,%edi
	dec %ecx
	jne rp_sidt
	lidt idt_descr               ; 加载 IDT 寄存器
	ret
```

**代码详解**：

| 指令 | 说明 |
|------|------|
| `lea ignore_int,%edx` | 加载默认中断处理程序地址 |
| `movl $0x00080000,%eax` | 设置段选择子为 0x0008（内核代码段） |
| `movw $0x8E00,%dx` | 设置门描述符属性：P=1, DPL=0, Type=14（中断门） |
| `lidt idt_descr` | 加载 IDT 寄存器 |

### 3. IDT 描述符结构

```asm
idt_descr:
	.word 256*8-1		# IDT 限长 = 256 * 8 - 1
	.long _idt		# IDT 基地址
```

### 4. 默认中断处理程序

**定义位置**：**`boot/head.s:152-170`**

```asm
/* This is the default interrupt "handler" :-) */
int_msg:
	.asciz "Unknown interrupt\n\r"
.align 2
ignore_int:
	pushl %eax
	pushl %ecx
	pushl %edx
	push %ds
	push %es
	push %fs
	movl $0x10,%eax
	mov %ax,%ds
	mov %ax,%es
	mov %ax,%fs
	pushl $int_msg
	call _printk
	popl %eax
	pop %fs
	pop %es
	pop %ds
	popl %edx
	popl %ecx
	popl %eax

```

**作用**：当发生未注册的中断时，打印 "Unknown interrupt" 消息。

## 中断控制器初始化

### 8259A PIC 初始化

**定义位置**：**`boot/setup.s:150-191`**

```asm
! well, that went ok, I hope. Now we have to reprogram the interrupts :-(
! we put them right after the intel-reserved hardware interrupts, at
! int 0x20-0x2F. There they won't mess up anything.

	mov	al,#0x11		! initialization sequence
	out	#0x20,al		! send it to 8259A-1 (主片)
	.word	0x00eb,0x00eb		! jmp $+2, jmp $+2 (延迟)
	out	#0xA0,al		! and to 8259A-2 (从片)
	.word	0x00eb,0x00eb
	mov	al,#0x20		! start of hardware int's (0x20)
	out	#0x21,al		! 主片起始中断号
	.word	0x00eb,0x00eb
	mov	al,#0x28		! start of hardware int's 2 (0x28)
	out	#0xA1,al		! 从片起始中断号
	.word	0x00eb,0x00eb
	mov	al,#0x04		! 8259-1 is master
	out	#0x21,al		! 主片 IR2 接从片
	.word	0x00eb,0x00eb
	mov	al,#0x02		! 8259-2 is slave
	out	#0xA1,al		! 从片标识
	.word	0x00eb,0x00eb
	mov	al,#0x01		! 8086 mode for both
	out	#0x21,al
	.word	0x00eb,0x00eb
	out	#0xA1,al
	.word	0x00eb,0x00eb
	mov	al,#0xFF		! mask off all interrupts for now
	out	#0x21,al		! 屏蔽所有中断
	.word	0x00eb,0x00eb
	out	#0xA1,al
```

### 8259A 中断向量映射

| 主片 IRQ | 中断号 | 用途 |
|----------|--------|------|
| IRQ0 | 0x20 | 时钟中断 |
| IRQ1 | 0x21 | 键盘中断 |
| IRQ2 | 0x22 | 级联从片 |
| IRQ3 | 0x23 | 串口2 |
| IRQ4 | 0x24 | 串口1 |
| IRQ5 | 0x25 | 并口2 |

| 从片 IRQ | 中断号 | 用途 |
|----------|--------|------|
| IRQ8 | 0x28 | 实时时钟 |
| IRQ9 | 0x29 | 保留 |
| IRQ10 | 0x2A | 保留 |
| IRQ11 | 0x2B | 保留 |
| IRQ12 | 0x2C | 保留 |
| IRQ13 | 0x2D | 协处理器 |
| IRQ14 | 0x2E | 硬盘 |
| IRQ15 | 0x2F | 保留 |

## 中断门描述符设置

### 1. 门描述符设置宏

**定义位置**：**`include/asm/system.h:22-33`**

```c
#define _set_gate(gate_addr,type,dpl,addr) \
__asm__ ("movw %%dx,%%ax\n\t" \
	"movw %0,%%dx\n\t" \
	"movl %%eax,%1\n\t" \
	"movl %%edx,%2" \
	: \
	: "i" ((short) (0x8000+(dpl<<13)+(type<<8))), \
	"o" (*((char *) (gate_addr))), \
	"o" (*(4+(char *) (gate_addr))), \
	"d" ((char *) (addr)),"a" (0x00080000))

#define set_intr_gate(n,addr) \
	_set_gate(&idt[n],14,0,addr)

#define set_trap_gate(n,addr) \
	_set_gate(&idt[n],15,0,addr)

#define set_system_gate(n,addr) \
	_set_gate(&idt[n],15,3,addr)
```

### 2. 三种门类型的区别

| 宏 | 类型 | DPL | 用途 |
|------|------|-----|------|
| `set_intr_gate` | 14 (中断门) | 0 | 硬件中断，只能内核调用 |
| `set_trap_gate` | 15 (陷阱门) | 0 | 异常处理，只能内核调用 |
| `set_system_gate` | 15 (陷阱门) | 3 | 系统调用，用户态可调用 |

### 3. 门描述符属性值计算

```
属性值 = 0x8000 + (DPL << 13) + (Type << 8)

中断门 (DPL=0, Type=14): 0x8000 + 0 + 0x0E00 = 0x8E00
陷阱门 (DPL=0, Type=15): 0x8000 + 0 + 0x0F00 = 0x8F00
系统门 (DPL=3, Type=15): 0x8000 + 0x6000 + 0x0F00 = 0xEF00
```

## 中断处理流程

### CPU 自动完成的步骤

当中断发生时，CPU 自动执行以下操作：

```
1. 获取中断号（从中断控制器或指令中）
2. 从 IDT 中读取对应的门描述符
3. 检查特权级（DPL >= CPL）
4. 保存当前状态到栈：
   - SS (如果是用户态)
   - ESP (如果是用户态)
   - EFLAGS
   - CS
   - EIP
   - Error Code (某些异常)
5. 加载新的 CS:EIP（从门描述符）
6. 清除某些标志位（IF、TF、VM、RF、NT）
```

### 栈结构（从用户态进入中断）

```
高地址
┌─────────────┐
│     SS      │ ← 用户栈段
├─────────────┤
│    ESP      │ ← 用户栈指针
├─────────────┤
│   EFLAGS    │ ← 标志寄存器
├─────────────┤
│     CS      │ ← 代码段
├─────────────┤
│    EIP      │ ← 返回地址
├─────────────┤
│ Error Code  │ ← 错误码（某些异常）
└─────────────┘ ← 当前 ESP
低地址
```

### 栈结构（从内核态进入中断）

```
高地址
┌─────────────┐
│   EFLAGS    │ ← 标志寄存器
├─────────────┤
│     CS      │ ← 代码段
├─────────────┤
│    EIP      │ ← 返回地址
├─────────────┤
│ Error Code  │ ← 错误码（某些异常）
└─────────────┘ ← 当前 ESP
低地址
```

## 异常处理详解

### 1. 异常处理入口

**定义位置**：**`kernel/asm.s:17-49`**

#### 无错误码异常处理（如除零错误）

```asm
_divide_error:
	pushl $_do_divide_error      ; 压入 C 处理函数地址
no_error_code:
	xchgl %eax,(%esp)            ; 交换 eax 和函数地址
	pushl %ebx                   ; 保存通用寄存器
	pushl %ecx
	pushl %edx
	pushl %edi
	pushl %esi
	pushl %ebp
	push %ds                     ; 保存段寄存器
	push %es
	push %fs
	pushl $0                     ; 压入伪错误码（对齐）
	lea 44(%esp),%edx            ; 计算栈指针（传递给 C 函数）
	pushl %edx
	movl $0x10,%edx              ; 设置内核数据段
	mov %dx,%ds
	mov %dx,%es
	mov %dx,%fs
	call *%eax                   ; 调用 C 处理函数
	addl $8,%esp                 ; 清理栈
	pop %fs                      ; 恢复段寄存器
	pop %es
	pop %ds
	popl %ebp                    ; 恢复通用寄存器
	popl %esi
	popl %edi
	popl %edx
	popl %ecx
	popl %ebx
	popl %eax
	iret                         ; 中断返回
```

#### 有错误码异常处理（如一般保护错误）

```asm
_double_fault:
	pushl $_do_double_fault
error_code:
	xchgl %eax,4(%esp)           ; 错误码 <-> %eax
	xchgl %ebx,(%esp)            ; 函数地址 <-> %ebx
	pushl %ecx
	pushl %edx
	pushl %edi
	pushl %esi
	pushl %ebp
	push %ds
	push %es
	push %fs
	pushl %eax                   ; 错误码
	lea 44(%esp),%eax            ; 栈指针
	pushl %eax
	movl $0x10,%eax
	mov %ax,%ds
	mov %ax,%es
	mov %ax,%fs
	call *%ebx
	addl $8,%esp
	pop %fs
	pop %es
	pop %ds
	popl %ebp
	popl %esi
	popl %edi
	popl %edx
	popl %ecx
	popl %ebx
	popl %eax

```

### 2. 异常处理函数表

**定义位置**：**`kernel/traps.c:176-208`**

```c
void trap_init(void)
{
	int i;

	set_trap_gate(0, &divide_error);        // 0 - 除零错误
	set_trap_gate(1, &debug);              // 1 - 调试异常
	set_trap_gate(2, &nmi);                // 2 - 不可屏蔽中断
	set_system_gate(3, &int3);             // 3 - 断点（用户可用）
	set_system_gate(4, &overflow);         // 4 - 溢出（用户可用）
	set_system_gate(5, &bounds);           // 5 - 边界检查（用户可用）
	set_trap_gate(6, &invalid_op);         // 6 - 无效操作码
	set_trap_gate(7, &device_not_available); // 7 - 设备不可用
	set_trap_gate(8, &double_fault);       // 8 - 双重错误
	set_trap_gate(9, &coprocessor_segment_overrun); // 9 - 协处理器段越界
	set_trap_gate(10, &invalid_TSS);       // 10 - 无效 TSS
	set_trap_gate(11, &segment_not_present); // 11 - 段不存在
	set_trap_gate(12, &stack_segment);     // 12 - 栈段错误
	set_trap_gate(13, &general_protection); // 13 - 一般保护错误
	set_trap_gate(14, &page_fault);        // 14 - 页错误
	set_trap_gate(15, &reserved);          // 15 - 保留
	set_trap_gate(16, &coprocessor_error); // 16 - 协处理器错误
	for (i = 17; i < 48; i++)
		set_trap_gate(i, &reserved);
	set_trap_gate(45, &irq13);             // 45 - 协处理器中断
	outb_p(inb_p(0x21) & 0xfb, 0x21);      // 允许 IRQ2
	outb(inb_p(0xA1) & 0xdf, 0xA1);        // 允许 IRQ13
	set_trap_gate(39, &parallel_interrupt); // 39 - 并口中断
}
```

## 硬件中断处理

### 1. 时钟中断处理

**定义位置**：**`kernel/system_call.s:176-197`**

```asm
_timer_interrupt:
	push %ds                    ; 保存段寄存器
	push %es
	push %fs
	pushl %edx                  ; 保存通用寄存器
	pushl %ecx
	pushl %ebx
	pushl %eax
	movl $0x10,%eax             ; 设置内核数据段
	mov %ax,%ds
	mov %ax,%es
	movl $0x17,%eax             ; fs 指向用户数据段
	mov %ax,%fs
	incl _jiffies               ; 增加系统时间计数
	movb $0x20,%al              ; EOI 命令
	outb %al,$0x20              ; 发送给中断控制器
	movl CS(%esp),%eax          ; 获取被中断代码的 CS
	andl $3,%eax                ; 提取 CPL
	pushl %eax                  ; 传递 CPL 给 do_timer
	call _do_timer              ; 调用时钟处理函数
	addl $4,%esp                ; 清理参数
	jmp ret_from_system_call    ; 跳转到系统调用返回处理
```

### 2. 硬盘中断处理

**定义位置**：**`kernel/system_call.s:221-250`**

```asm
_hd_interrupt:
	pushl %eax                  ; 保存寄存器
	pushl %ecx
	pushl %edx
	push %ds
	push %es
	push %fs
	movl $0x10,%eax             ; 设置内核数据段

	mov %ax,%es

	outb %al,$0xA0              ; EOI 给从片

1:	jmp 1f
1:	xorl %edx,%edx

	testl %edx,%edx
	jne 1f
	movl $_unexpected_hd_interrupt,%edx
1:	outb %al,$0x20              ; EOI 给主片
	call *%edx                  ; 调用硬盘处理函数
	pop %fs                     ; 恢复寄存器
	pop %es
	pop %ds
	popl %edx
	popl %ecx
	popl %eax
	iret                        ; 中断返回
```

### 4. 硬件中断处理流程

```
硬件设备发出中断信号
        │
        ▼
8259A 中断控制器接收
        │
        ▼
向 CPU 发送中断请求
        │
        ▼
CPU 响应中断，自动保存上下文
        │
        ▼
跳转到 IDT 中对应的中断处理程序
        │
        ▼
汇编入口保存更多寄存器
        │
        ▼
发送 EOI 给中断控制器
        │
        ▼
调用 C 语言处理函数
        │
        ▼
恢复寄存器，执行 iret
        │
        ▼
返回被中断的程序
```

## 系统调用中断

### 1. 系统调用入口

**定义位置**：**`kernel/system_call.s:80-128`**

```asm
_system_call:
	cmpl $nr_system_calls-1,%eax  ; 检查系统调用号
	ja bad_sys_call
	push %ds                      ; 保存段寄存器
	push %es
	push %fs
	pushl %edx                    ; 保存参数寄存器
	pushl %ecx
	pushl %ebx
	movl $0x10,%edx               ; 设置内核数据段
	mov %dx,%ds
	mov %dx,%es
	movl $0x17,%edx               ; fs 指向用户数据段
	mov %dx,%fs
	call _sys_call_table(,%eax,4) ; 调用系统调用处理函数
	pushl %eax                    ; 保存返回值
	movl _current,%eax
	cmpl $0,state(%eax)           ; 检查进程状态
	jne reschedule                ; 需要调度
	cmpl $0,counter(%eax)         ; 检查时间片
	je reschedule                 ; 时间片用完，需要调度
ret_from_system_call:
	movl _current,%eax
	cmpl _task,%eax               ; 任务0不处理信号
	je 3f
	cmpw $0x0f,CS(%esp)           ; 检查是否从用户态来
	jne 3f
	cmpw $0x17,OLDSS(%esp)        ; 检查栈段
	jne 3f
	movl signal(%eax),%ebx        ; 处理信号
	movl blocked(%eax),%ecx
	notl %ecx
	andl %ebx,%ecx
	bsfl %ecx,%ecx
	je 3f
	btrl %ecx,%ebx
	movl %ebx,signal(%eax)
	incl %ecx
	pushl %ecx
	call _do_signal
	popl %eax
3:	popl %eax                      ; 恢复返回值
	popl %ebx
	popl %ecx
	popl %edx
	pop %fs
	pop %es
	pop %ds
	iret                           ; 中断返回
```

### 2. 系统调用流程

```
用户程序调用 fork()
        │
        ▼
_syscall0 宏展开
        │
        ▼
执行 int $0x80
        │
        ▼
CPU 保存上下文，跳转到 _system_call
        │
        ▼
检查系统调用号
        │
        ▼
保存寄存器，设置内核段
        │
        ▼
通过 sys_call_table 调用 sys_fork
        │
        ▼
检查是否需要调度
        │
        ▼
处理信号（如果有）
        │
        ▼
恢复寄存器，iret 返回
        │
        ▼
用户程序继续执行
```

**定义位置**：**`include/linux/sys.h:74-86`**

```c
    sys_read,    // 3
    sys_write,   // 4

    sys_close,   // 6

};
```

## 中断返回机制

### 1. iret 指令

`iret` 指令执行以下操作：

```
1. 从栈中弹出 EIP
2. 从栈中弹出 CS
3. 从栈中弹出 EFLAGS
4. 如果返回到不同特权级，弹出 ESP 和 SS
5. 继续执行被中断的程序
```

### 2. 中断返回流程

```
执行 iret 指令
        │
        ▼
弹出 EIP（返回地址）
        │
        ▼
弹出 CS（代码段）
        │
        ▼
弹出 EFLAGS（标志寄存器）
        │
        ▼
检查 CPL 是否变化
        │
        ├─ 是：弹出 ESP 和 SS（恢复用户栈）
        │
        └─ 否：继续
        │
        ▼
恢复执行被中断的程序
```

## 总结

### 中断处理的关键技术

| 技术 | 说明 |
|------|------|
| **IDT** | 中断描述符表，存储所有中断处理程序入口 |
| **门描述符** | 描述中断处理程序的入口和属性 |
| **8259A PIC** | 可编程中断控制器，管理硬件中断 |
| **特权级检查** | 确保只有合法的中断请求被处理 |
| **上下文保存** | 保存 CPU 状态以便恢复 |
| **iret 指令** | 从中断返回，恢复上下文 |

### 中断处理的层次结构

```
┌─────────────────────────────────────────┐
│           用户程序                       │
├─────────────────────────────────────────┤
│         系统调用接口                     │
│           (int 0x80)                    │
├─────────────────────────────────────────┤
│           IDT                           │
├─────────────────────────────────────────┤
│       汇编入口（保存/恢复寄存器）         │
├─────────────────────────────────────────┤
│       C 处理函数（具体逻辑）              │
├─────────────────────────────────────────┤
│           硬件                          │
└─────────────────────────────────────────┘
```

### 关键代码文件

| 文件 | 功能 |
|------|------|
| **`boot/head.s`** | IDT 初始化，默认中断处理 |
| **`boot/setup.s`** | 8259A 初始化 |
| **`kernel/asm.s`** | 异常处理入口 |
| **`kernel/traps.c`** | 异常处理函数，trap_init |
| **`kernel/system_call.s`** | 系统调用，硬件中断处理 |
| **`include/asm/system.h`** | 门描述符设置宏 |

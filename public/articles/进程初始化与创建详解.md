## 进程概述

### 什么是进程

进程是程序在计算机上的一次执行过程，是系统进行资源分配和调度的基本单位。Linux 0.11 中每个进程都有：

- 唯一的进程标识符（PID）
- 独立的内存空间
- 执行上下文（寄存器状态）
- 打开的文件资源
- 信号处理机制

### 进程的生命周期

```
创建(fork) → 运行(running) → 睡眠(sleeping) → 唤醒(wakeup) → 退出(exit)
                              ↓
                           僵尸(zombie) → 回收
```

### 进程状态定义

**定义位置**：**`include/linux/sched.h:19-23`**

```c
#define TASK_RUNNING        0   /* 可运行状态 */
#define TASK_INTERRUPTIBLE  1   /* 可中断睡眠 */
#define TASK_UNINTERRUPTIBLE 2  /* 不可中断睡眠 */
#define TASK_ZOMBIE         3   /* 僵尸进程 */
#define TASK_STOPPED        4   /* 停止状态 */
```

## 核心数据结构

### 1. 任务控制块 (task_struct)

**定义位置**：**`include/linux/sched.h:78-107`**

```c
struct task_struct {
/* 调度相关字段 */
	long state;		/* 进程状态 */
	long counter;		/* 时间片计数器 */
	long priority;		/* 优先级 */
	long signal;		/* 信号位图 */
	struct sigaction sigaction[32]; /* 信号处理 */
	long blocked;		/* 阻塞信号 */

/* 进程标识 */
	int exit_code;		/* 退出码 */
	unsigned long start_code, end_code, end_data, brk, start_stack;
	long pid, father, pgrp, session, leader;
	unsigned short uid, euid, suid;
	unsigned short gid, egid, sgid;

/* 时间统计 */
	long alarm;
	long utime, stime, cutime, cstime, start_time;
	unsigned short used_math;

/* 文件系统 */
	int tty;
	unsigned short umask;
	struct m_inode * pwd;
	struct m_inode * root;
	struct m_inode * executable;
	unsigned long close_on_exec;
	struct file * filp[NR_OPEN];

/* CPU 相关 */
	struct desc_struct ldt[3];  /* 局部描述符表 */
	struct tss_struct tss;      /* 任务状态段 */
};
```

**关键字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `state` | long | 进程状态，-1不可运行，0可运行，>0停止 |
| `counter` | long | 剩余时间片，0时触发调度 |
| `priority` | long | 优先级，决定时间片大小 |
| `pid` | long | 进程标识符 |
| `father` | long | 父进程PID |
| `tss` | struct | 任务状态段，保存CPU上下文 |
| `ldt` | struct | 局部描述符表，定义进程的段 |

### 2. 任务状态段 (TSS)

**定义位置**：**`include/linux/sched.h:51-76`**

```c
struct tss_struct {
	long back_link;		/* 任务链，高16位为0 */
	long esp0;		/* 0级栈指针 */
	long ss0;		/* 0级栈段选择子 */
	long esp1;		/* 1级栈指针 */
	long ss1;		/* 1级栈段选择子 */
	long esp2;		/* 2级栈指针 */
	long ss2;		/* 2级栈段选择子 */
	long cr3;		/* 页目录基址 */
	long eip;		/* 指令指针 */
	long eflags;		/* 标志寄存器 */
	long eax, ecx, edx, ebx; /* 通用寄存器 */
	long esp;		/* 栈指针 */
	long ebp;		/* 基址指针 */
	long esi;		/* 源变址 */
	long edi;		/* 目的变址 */
	long es, cs, ss, ds, fs, gs; /* 段寄存器 */
	long ldt;		/* LDT 选择子 */
	long trace_bitmap;	/* 跟踪位图 */
	struct i387_struct i387; /* 协处理器状态 */
};
```

**TSS 的作用**：
- 保存进程的完整 CPU 上下文
- 实现任务切换时自动保存/恢复状态
- 存储不同特权级的栈信息

### 3. 任务联合体 (task_union)

**定义位置**：**`kernel/sched.c:53-56`**

```c
union task_union {
	struct task_struct task;
	char stack[PAGE_SIZE];
};
```

**设计原理**：
- task_struct 和内核栈共享一个 4KB 页面
- task_struct 位于页面底部
- 内核栈从页面顶部向下增长
- 这种设计节省内存，便于快速访问当前进程

```
┌─────────────────────────────────────┐ ← 页面顶部 (4KB)
│                                     │
│          内核栈空间                  │
│         (向下增长)                   │
│                                     │
├─────────────────────────────────────┤
│                                     │
│        struct task_struct           │
│          (约1KB)                     │
│                                     │
└─────────────────────────────────────┘ ← 页面底部 (0)
```

### 4. 任务数组

**定义位置**：**`kernel/sched.c:65`**

```c
struct task_struct * task[NR_TASKS] = {&(init_task.task), };
```

- `NR_TASKS` 定义为 64，系统最多支持 64 个进程
- `task[0]` 指向进程0（空闲进程）
- 其他槽位初始化为 NULL

## 进程调度初始化

### sched_init() 函数

**定义位置**：**`kernel/sched.c:383-412`**

```c
void sched_init(void)
{
	int i;
	struct desc_struct * p;

	if (sizeof(struct sigaction) != 16)
		panic("Struct sigaction MUST be 16 bytes");
	
	/* 设置进程0的 TSS 和 LDT 描述符 */
	set_tss_desc(gdt+FIRST_TSS_ENTRY, &(init_task.task.tss));
	set_ldt_desc(gdt+FIRST_LDT_ENTRY, &(init_task.task.ldt));
	
	/* 清空其他进程的 TSS 和 LDT 描述符 */
	p = gdt+2+FIRST_TSS_ENTRY;
	for(i = 1; i < NR_TASKS; i++) {
		task[i] = NULL;
		p->a = p->b = 0;
		p++;
		p->a = p->b = 0;
		p++;
	}
	
	/* 清除 EFLAGS 中的 NT 标志 */
	__asm__("pushfl ; andl $0xffffbfff,(%esp) ; popfl");
	
	/* 加载任务寄存器和 LDT 寄存器 */
	ltr(0);
	lldt(0);
	
	/* 初始化 8253 定时器 */
	outb_p(0x36, 0x43);             /* binary, mode 3, LSB/MSB, ch 0 */
	outb_p(LATCH & 0xff, 0x40);     /* LSB */
	outb(LATCH >> 8, 0x40);         /* MSB */
	
	/* 设置定时器中断门 */
	set_intr_gate(0x20, &timer_interrupt);
	outb(inb_p(0x21) & ~0x01, 0x21);  /* 允许时钟中断 */
	
	/* 设置系统调用门 */
	set_system_gate(0x80, &system_call);
}
```

**初始化步骤详解**：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 设置 TSS 描述符 | 将进程0的 TSS 描述符放入 GDT |
| 2 | 设置 LDT 描述符 | 将进程0的 LDT 描述符放入 GDT |
| 3 | 清空其他槽位 | 初始化其他进程的描述符为空 |
| 4 | 清除 NT 标志 | 避免任务嵌套问题 |
| 5 | ltr(0) | 加载任务寄存器，指向进程0的 TSS |
| 6 | lldt(0) | 加载 LDT 寄存器 |
| 7 | 初始化定时器 | 设置 100Hz 时钟中断 |
| 8 | 设置中断门 | 注册时钟中断和系统调用处理程序 |

### GDT 中 TSS 和 LDT 的布局

**定义位置**：**`include/linux/sched.h:153-157`**

```c
#define FIRST_TSS_ENTRY 4
#define FIRST_LDT_ENTRY (FIRST_TSS_ENTRY+1)
#define _TSS(n) ((((unsigned long) n)<<4)+(FIRST_TSS_ENTRY<<3))
#define _LDT(n) ((((unsigned long) n)<<4)+(FIRST_LDT_ENTRY<<3))
```

**GDT 布局**：

```
索引  内容
────────────────────────
0     空描述符
1     内核代码段
2     内核数据段
3     系统调用门
4     TSS0 (进程0)
5     LDT0 (进程0)
6     TSS1 (进程1)
7     LDT1 (进程1)
...   ...
```

### TSS/LDT 描述符设置宏

**定义位置**：**`include/asm/system.h:51-66`**

```c
#define _set_tssldt_desc(n,addr,type) \
__asm__ ("movw $104,%1\n\t" \

	"movb %%al,%3\n\t" \

	"movb $0x00,%5\n\t" \
```

## fork系统调用详解

### 1. 系统调用入口

**定义位置**：**`kernel/system_call.s:208-219`**

```asm
_sys_fork:
	call _find_empty_process    ; 寻找空进程槽
	testl %eax,%eax             ; 检查返回值
	js 1f                        ; 如果为负，跳转到返回
	push %gs                     ; 保存寄存器
	pushl %esi
	pushl %edi
	pushl %ebp
	pushl %eax                   ; 压入进程槽号
	call _copy_process          ; 调用进程复制函数
	addl $20,%esp               ; 清理栈
1:	ret
```

### 2. 寻找空进程槽

**定义位置**：**`kernel/fork.c:134-147`**

```c
int find_empty_process(void)
{
	int i;

	repeat:
		if ((++last_pid) < 0) 
			last_pid = 1;           /* PID 溢出处理 */
		for(i = 0; i < NR_TASKS; i++)
			if (task[i] && task[i]->pid == last_pid) 
				goto repeat;         /* PID 冲突，重试 */
	
	for(i = 1; i < NR_TASKS; i++)
		if (!task[i])
			return i;               /* 返回空槽索引 */
	
	return -EAGAIN;                  /* 无可用槽位 */
}
```

**功能**：
1. 分配唯一的 PID（避免冲突）
2. 查找任务数组中的空槽位
3. 返回槽位索引或错误码

### 3. 复制进程 (copy_process)

**定义位置**：**`kernel/fork.c:68-132`**

```c
int copy_process(int nr, long ebp, long edi, long esi, long gs, long none,
		long ebx, long ecx, long edx,
		long fs, long es, long ds,
		long eip, long cs, long eflags, long esp, long ss)
{
	struct task_struct *p;
	int i;
	struct file *f;

	/* 1. 分配一页内存作为 task_struct 和内核栈 */
	p = (struct task_struct *) get_free_page();
	if (!p)
		return -EAGAIN;
	
	/* 2. 在任务数组中记录新进程 */
	task[nr] = p;
	
	/* 3. 复制父进程的 task_struct */
	*p = *current;  /* 注意：这不复制内核栈 */
	
	/* 4. 设置新进程的特有字段 */
	p->state = TASK_UNINTERRUPTIBLE;  /* 先设为不可中断，防止被调度 */
	p->pid = last_pid;
	p->father = current->pid;
	p->counter = p->priority;
	p->signal = 0;
	p->alarm = 0;
	p->leader = 0;
	p->utime = p->stime = 0;
	p->cutime = p->cstime = 0;
	p->start_time = jiffies;
	
	/* 5. 设置 TSS 字段 */
	p->tss.back_link = 0;
	p->tss.esp0 = PAGE_SIZE + (long) p;  /* 内核栈顶 */
	p->tss.ss0 = 0x10;                    /* 内核栈段 */
	p->tss.eip = eip;                     /* 返回地址 */
	p->tss.eflags = eflags;
	p->tss.eax = 0;                       /* fork 返回 0 给子进程 */
	p->tss.ecx = ecx;
	p->tss.edx = edx;
	p->tss.ebx = ebx;
	p->tss.esp = esp;                     /* 用户栈指针 */
	p->tss.ebp = ebp;
	p->tss.esi = esi;
	p->tss.edi = edi;
	p->tss.es = es & 0xffff;
	p->tss.cs = cs & 0xffff;
	p->tss.ss = ss & 0xffff;
	p->tss.ds = ds & 0xffff;
	p->tss.fs = fs & 0xffff;
	p->tss.gs = gs & 0xffff;
	p->tss.ldt = _LDT(nr);                /* LDT 选择子 */
	p->tss.trace_bitmap = 0x80000000;
	
	/* 6. 保存协处理器状态 */
	if (last_task_used_math == current)
		__asm__("clts ; fnsave %0"::"m" (p->tss.i387));
	
	/* 7. 复制内存（页表） */
	if (copy_mem(nr, p)) {
		task[nr] = NULL;
		free_page((long) p);
		return -EAGAIN;
	}
	
	/* 8. 增加文件引用计数 */
	for (i = 0; i < NR_OPEN; i++)
		if (f = p->filp[i])
			f->f_count++;
	if (current->pwd)
		current->pwd->i_count++;
	if (current->root)
		current->root->i_count++;
	if (current->executable)
		current->executable->i_count++;
	
	/* 9. 设置 GDT 中的 TSS 和 LDT 描述符 */
	set_tss_desc(gdt+(nr<<1)+FIRST_TSS_ENTRY, &(p->tss));
	set_ldt_desc(gdt+(nr<<1)+FIRST_LDT_ENTRY, &(p->ldt));
	
	/* 10. 设置进程为可运行状态 */
	p->state = TASK_RUNNING;
	
	return last_pid;  /* 返回子进程 PID 给父进程 */
}
```

### 4. copy_process 参数说明

| 参数 | 来源 | 说明 |
|------|------|------|
| `nr` | find_empty_process() | 进程槽位索引 |
| `eip` | 栈上 | 返回地址（fork 后的执行位置） |
| `eflags` | 栈上 | 标志寄存器 |
| `esp` | 栈上 | 用户栈指针 |
| `cs, ds, es, fs, gs, ss` | 栈上 | 段寄存器 |
| `eax, ebx, ecx, edx` | 栈上 | 通用寄存器 |
| `ebp, esi, edi` | 栈上 | 指针和变址寄存器 |

## 进程内存管理

### 1. copy_mem() 函数

**定义位置**：**`kernel/fork.c:35-56`**

```c
int copy_mem(int nr, struct task_struct * p)
{
	unsigned long old_data_base, new_data_base, data_limit;
	unsigned long old_code_base, new_code_base, code_limit;

	/* 获取父进程的段限制和基地址 */
	code_limit = get_limit(0x0f);
	data_limit = get_limit(0x17);
	old_code_base = get_base(current->ldt[1]);
	old_data_base = get_base(current->ldt[2]);
	
	if (old_data_base != old_code_base)
		panic("We don't support separate I&D");
	if (data_limit < code_limit)
		panic("Bad data_limit");
	
	/* 计算新进程的基地址 */
	new_data_base = new_code_base = nr * 0x4000000;  /* 64MB * nr */
	p->start_code = new_code_base;
	
	/* 设置新进程的 LDT */
	set_base(p->ldt[1], new_code_base);
	set_base(p->ldt[2], new_data_base);
	
	/* 复制页表 */
	if (copy_page_tables(old_data_base, new_data_base, data_limit)) {
		free_page_tables(new_data_base, data_limit);
		return -ENOMEM;
	}
	
	return 0;
}
```

### 2. 进程地址空间布局

每个进程拥有独立的 64MB 虚拟地址空间：

```
进程地址空间布局：

进程0: 0x00000000 - 0x03FFFFFF (64MB)
进程1: 0x04000000 - 0x07FFFFFF (64MB)
进程2: 0x08000000 - 0x0BFFFFFF (64MB)
...

每个进程空间内部：
┌─────────────────────┐ 0x04000000 (进程1起始)
│     代码段          │
├─────────────────────┤
│     数据段          │
├─────────────────────┤
│                     │
│       堆            │ ↓ 向下增长
│                     │
├─────────────────────┤
│                     │
│       栈            │ ↑ 向上增长
│                     │
└─────────────────────┘ 0x07FFFFFF (进程1结束)
```

### 3. LDT 描述符设置

```c
/* LDT[0] - 空 */
/* LDT[1] - 代码段 */
/* LDT[2] - 数据/栈段 */

#define set_base(ldt,base) \
__asm__("movw %%dx,%1\n\t" \
	"rorl $16,%%eax\n\t" \
	"movb %%al,%2\n\t" \
	"movb %%ah,%3\n\t" \
	"rorl $16,%%eax" \
	::"a" (base), "m" (*((char *)&(ldt))), \
	"m" (*((char *)&(ldt)+2)), "m" (*((char *)&(ldt)+4)))
```

## 进程切换机制

### 1. switch_to 宏

**定义位置**：**`include/linux/sched.h:171-186`**

```c
#define switch_to(n) {\
struct {long a,b;} __tmp; \
__asm__("cmpl %%ecx,_current\n\t" \
	"je 1f\n\t" \                     /* 如果是当前进程，跳过 */
	"movw %%dx,%1\n\t" \              /* 设置 TSS 选择子 */
	"xchgl %%ecx,_current\n\t" \      /* 更新 current 指针 */
	"ljmp %0\n\t" \                   /* 跳转到 TSS，触发任务切换 */
	"cmpl %%ecx,_last_task_used_math\n\t" \ /* 检查协处理器 */
	"jne 1f\n\t" \
	"clts\n" \                        /* 清除 TS 标志 */
	"1:" \
	::"m" (*&__tmp.a), "m" (*&__tmp.b), \
	"d" (_TSS(n)), "c" ((long) task[n])); \
}
```

### 2. 任务切换原理

**ljmp 指令的工作流程**：

```
执行 ljmp TSS选择子
        │
        ▼
CPU 检查 TSS 描述符
        │
        ▼
保存当前状态到当前 TSS
        │
        ├─ 保存所有寄存器
        ├─ 保存段寄存器
        └─ 保存 EIP、EFLAGS
        │
        ▼
从目标 TSS 加载新状态
        │
        ├─ 加载所有寄存器
        ├─ 加载段寄存器
        ├─ 加载 CR3（页目录）
        └─ 加载 LDT
        │
        ▼
从新 EIP 处继续执行
```

### 3. 任务切换时机

| 触发条件 | 说明 |
|----------|------|
| 时间片用完 | 时钟中断检测到 counter=0 |
| 进程睡眠 | 调用 sleep_on() 或 pause() |
| 进程退出 | 调用 exit() |
| 优先级调度 | 高优先级进程就绪 |

## 总结

### 进程创建的关键步骤

| 步骤 | 函数 | 说明 |
|------|------|------|
| 1 | find_empty_process() | 分配 PID，查找空槽 |
| 2 | get_free_page() | 分配 task_struct 和内核栈 |
| 3 | 复制 task_struct | 继承父进程属性 |
| 4 | 设置 TSS | 配置任务状态段 |
| 5 | copy_mem() | 复制页表，设置 LDT |
| 6 | 增加文件引用 | 共享打开的文件 |
| 7 | 设置 GDT 描述符 | 注册 TSS 和 LDT |
| 8 | 设置状态 | 标记为可运行 |

### 关键数据结构

| 结构 | 大小 | 说明 |
|------|------|------|
| task_struct | ~1KB | 进程控制块 |
| TSS | 104字节 | 任务状态段 |
| LDT | 24字节 | 局部描述符表 |
| task_union | 4KB | task_struct + 内核栈 |

### 关键代码文件

| 文件 | 功能 |
|------|------|
| **`include/linux/sched.h`** | 进程数据结构定义 |
| **`kernel/fork.c`** | 进程创建函数 |
| **`kernel/sched.c`** | 调度器和初始化 |
| **`kernel/system_call.s`** | fork 汇编入口 |
| **`init/main.c`** | 内核启动流程 |

## 概述

Linux 0.11 采用**基于时间片的优先级调度算法**，核心思想是：

- 每个进程有一个优先级（`priority`）和时间片计数器（`counter`）
- 调度器选择时间片最大且处于就绪状态的进程运行
- 当所有就绪进程的时间片都用完时，重新分配时间片

**调度策略特点**：
- 简单高效，适合早期操作系统
- 支持 I/O 密集型进程获得更好的响应时间
- 通过时间片轮转实现公平调度

**调度相关字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `state` | long | 进程状态（运行/睡眠/僵尸等） |
| `counter` | long | 剩余时间片，0 表示需要重新调度 |
| `priority` | long | 优先级，决定时间片大小 |
| `alarm` | long | 闹钟时间，用于定时唤醒 |

## 调度算法实现

### 1. schedule() 函数

定义位置：**`kernel/sched.c:100-141`**

```c
void schedule(void)
{
    int i, next, c;
    struct task_struct ** p;

    /* 检查闹钟，唤醒收到信号的可中断睡眠进程 */
    for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)
        if (*p) {
            if ((*p)->alarm && (*p)->alarm < jiffies) {
                (*p)->signal |= (1<<(SIGALRM-1));
                (*p)->alarm = 0;
            }
            if (((*p)->signal & ~(_BLOCKABLE & (*p)->blocked)) &&
                (*p)->state == TASK_INTERRUPTIBLE)
                (*p)->state = TASK_RUNNING;
        }

    /* 调度主算法 */
    while (1) {
        c = -1;
        next = 0;
        i = NR_TASKS;
        p = &task[NR_TASKS];
        
        /* 遍历所有进程，找出时间片最大的就绪进程 */
        while (--i) {
            if (!*--p)
                continue;
            if ((*p)->state == TASK_RUNNING && (*p)->counter > c)
                c = (*p)->counter, next = i;
        }
        
        /* 如果找到可运行的进程，跳出循环 */
        if (c) break;
        
        /* 所有就绪进程的时间片都用完，重新分配时间片 */
        for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)
            if (*p)
                (*p)->counter = ((*p)->counter >> 1) + (*p)->priority;
    }
    
    /* 切换到选中的进程 */
    switch_to(next);
}
```

### 2. 调度算法详解

#### 步骤一：闹钟和信号处理

```c
for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)
    if (*p) {
        if ((*p)->alarm && (*p)->alarm < jiffies) {
            (*p)->signal |= (1<<(SIGALRM-1));
            (*p)->alarm = 0;
        }
        if (((*p)->signal & ~(_BLOCKABLE & (*p)->blocked)) &&
            (*p)->state == TASK_INTERRUPTIBLE)
            (*p)->state = TASK_RUNNING;
    }
```

**作用**：
- 检查每个进程的闹钟是否到期
- 如果到期，发送 `SIGALRM` 信号
- 唤醒收到信号且处于可中断睡眠状态的进程

#### 步骤二：选择下一个运行的进程

```c
while (--i) {
    if (!*--p)
        continue;
    if ((*p)->state == TASK_RUNNING && (*p)->counter > c)
        c = (*p)->counter, next = i;
}
```

**选择标准**：
- 进程状态必须是 `TASK_RUNNING`
- 选择 `counter` 值最大的进程
- 如果多个进程 `counter` 相同，选择数组下标较小的

#### 步骤三：时间片重新分配

```c
for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)
    if (*p)
        (*p)->counter = ((*p)->counter >> 1) + (*p)->priority;
```

**分配策略**：
- 新时间片 = 剩余时间片/2 + 优先级
- 这种策略使得 I/O 密集型进程（经常睡眠，剩余时间片多）获得更多时间片
- CPU 密集型进程（时间片用完）只能获得基础优先级

## 任务切换机制

### 1. switch_to 宏

定义位置：**`include/linux/sched.h:171-186`**

```c
#define switch_to(n) {\
struct {long a,b;} __tmp; \
__asm__("cmpl %%ecx,_current\n\t" \
    "je 1f\n\t" \                     /* 如果切换到当前进程，跳过 */
    "movw %%dx,%1\n\t" \              /* 设置 TSS 选择子 */
    "xchgl %%ecx,_current\n\t" \      /* 更新 current 指针 */
    "ljmp %0\n\t" \                   /* 跳转到 TSS，触发任务切换 */
    "cmpl %%ecx,_last_task_used_math\n\t" \ /* 检查协处理器 */
    "jne 1f\n\t" \
    "clts\n" \                        /* 清除 TS 标志 */
    "1:" \
    ::"m" (*&__tmp.a),"m" (*&__tmp.b), \
    "d" (_TSS(n)),"c" ((long) task[n])); \
}
```

### 2. 任务切换原理

**关键指令：`ljmp`**

```
ljmp 指令格式：ljmp TSS选择子
```

当执行 `ljmp` 到一个 TSS 描述符时，CPU 会：

1. **保存当前状态**：将当前所有寄存器保存到当前 TSS
2. **加载新状态**：从目标 TSS 加载所有寄存器
3. **切换栈**：切换到新进程的内核栈
4. **继续执行**：从新进程的 EIP 处继续执行

### 3. TSS 和 LDT 在 GDT 中的布局

```c
#define FIRST_TSS_ENTRY 4
#define FIRST_LDT_ENTRY (FIRST_TSS_ENTRY+1)
#define _TSS(n) ((((unsigned long) n)<<4)+(FIRST_TSS_ENTRY<<3))
#define _LDT(n) ((((unsigned long) n)<<4)+(FIRST_LDT_ENTRY<<3))
```

**GDT 布局**：

| 索引 | 内容 |
|------|------|
| 0 | 空描述符 |
| 1 | 内核代码段 |
| 2 | 内核数据段 |
| 3 | 系统调用门 |
| 4 | TSS0 (任务0) |
| 5 | LDT0 (任务0) |
| 6 | TSS1 (任务1) |
| 7 | LDT1 (任务1) |
| ... | ... |

int sys_pause(void)

#### sleep_on() 函数

定义位置：**`kernel/sched.c:149-162`**

```c
void sleep_on(struct task_struct **p)
{
    struct task_struct *tmp;

    if (!p)
        return;
    if (current == &(init_task.task))
        panic("task[0] trying to sleep");
    tmp = *p;
    *p = current;
    current->state = TASK_UNINTERRUPTIBLE;
    schedule();
    if (tmp)
        tmp->state = 0;
}
```

#### interruptible_sleep_on() 函数

定义位置：**`kernel/sched.c:164-181`**

```c
void interruptible_sleep_on(struct task_struct **p)
{
    struct task_struct *tmp;

    if (!p)
        return;
    if (current == &(init_task.task))
        panic("task[0] trying to sleep");
    tmp = *p;
    *p = current;
repeat:
    current->state = TASK_INTERRUPTIBLE;
    schedule();
    if (*p && *p != current) {
        (**p).state = 0;
        goto repeat;
    }
    *p = NULL;
    if (tmp)
        tmp->state = 0;
}
```

### 2. 被动调度（抢占式）

#### 定时器中断触发调度

定义位置：**`kernel/sched.c:318-337`**

```c
void do_timer(long cpl)
{
    extern int beepcount;
    extern void sysbeepstop(void);

    if (beepcount)
        if (!--beepcount)
            sysbeepstop();

    if (cpl)
        current->utime++;    /* 用户态时间 */
    else
        current->stime++;    /* 内核态时间 */

    if (next_timer) {
        next_timer->jiffies--;
        while (next_timer && next_timer->jiffies <= 0) {
            void (*fn)(void);
            fn = next_timer->fn;
            next_timer->fn = NULL;
            next_timer = next_timer->next;
            (fn)();
        }
    }
    if (current_DOR & 0xf0)
        do_floppy_timer();
    if ((--current->counter) > 0) return;  /* 时间片未用完 */
    current->counter = 0;
    if (!cpl) return;                       /* 内核态不抢占 */
    schedule();                             /* 用户态触发调度 */
}
```

**关键点**：
- 每次时钟中断（100Hz）调用 `do_timer()`
- 递减当前进程的时间片
- 时间片用完且在用户态时触发调度

### 1. 时钟频率

### 2. jiffies 全局变量

定义位置：**`kernel/sched.c:60`**

```c

```

- 记录系统启动以来的时钟中断次数
- 每次时钟中断加 1
- 用于计算时间和超时

### 3. 定时器链表

定义位置：**`kernel/sched.c:274-278`**

```c
static struct timer_list {
    long jiffies;           /* 到期时间 */
    void (*fn)();           /* 回调函数 */
    struct timer_list * next; /* 下一个定时器 */
} timer_list[TIME_REQUESTS], * next_timer = NULL;
```

### 4. 添加定时器

定义位置：**`kernel/sched.c:280-307`**

```c
void add_timer(long jiffies, void (*fn)(void))
{
    struct timer_list * p;

    if (!fn)
        return;
    cli();
    if (jiffies <= 0)
        (fn)();
    else {
        for (p = timer_list ; p < timer_list + TIME_REQUESTS ; p++)
            if (!p->fn)
                break;
        if (p >= timer_list + TIME_REQUESTS)
            panic("No more time requests free");
        p->fn = fn;
        p->jiffies = jiffies;
        p->next = next_timer;
        next_timer = p;
        while (p->next && p->next->jiffies < p->jiffies) {
            p->jiffies -= p->next->jiffies;
            fn = p->fn;
            p->fn = p->next->fn;
            p->next->fn = fn;
            jiffies = p->jiffies;
            p->jiffies = p->next->jiffies;
            p->next->jiffies = jiffies;
            p = p->next;
        }
    }
    sti();
}
```

## 调度初始化

### sched_init() 函数

定义位置：**`kernel/sched.c:383-412`**

```c
void sched_init(void)
{
    int i;
    struct desc_struct * p;

    if (sizeof(struct sigaction) != 16)
        panic("Struct sigaction MUST be 16 bytes");
    
    /* 设置任务0的 TSS 和 LDT 描述符 */
    set_tss_desc(gdt+FIRST_TSS_ENTRY, &(init_task.task.tss));
    set_ldt_desc(gdt+FIRST_LDT_ENTRY, &(init_task.task.ldt));
    
    /* 清空其他任务的 TSS 和 LDT 描述符 */
    p = gdt+2+FIRST_TSS_ENTRY;
    for(i = 1; i < NR_TASKS; i++) {
        task[i] = NULL;
        p->a = p->b = 0;
        p++;
        p->a = p->b = 0;
        p++;
    }
    
    /* 清除 NT 标志 */
    __asm__("pushfl ; andl $0xffffbfff,(%esp) ; popfl");
    
    /* 加载任务寄存器和 LDT 寄存器 */
    ltr(0);
    lldt(0);
    
    /* 初始化 8253 定时器 */
    outb_p(0x36, 0x43);             /* binary, mode 3, LSB/MSB, ch 0 */
    outb_p(LATCH & 0xff, 0x40);     /* LSB */
    outb(LATCH >> 8, 0x40);         /* MSB */
    
    /* 设置定时器中断门 */
    set_intr_gate(0x20, &timer_interrupt);
    outb(inb_p(0x21) & ~0x01, 0x21);  /* 允许时钟中断 */
    
    /* 设置系统调用门 */
    set_system_gate(0x80, &system_call);
}
```

**初始化步骤**：

1. 设置任务0的 TSS 和 LDT 描述符
2. 清空其他任务的描述符
3. 清除 EFLAGS 中的 NT 标志
4. 加载 TR 和 LDTR 寄存器
5. 初始化 8253 定时器芯片
6. 设置时钟中断门（0x20）
7. 设置系统调用门（0x80）

## 完整调度流程

### 1. 系统启动流程

```
bootsect.s → setup.s → head.s → main()
                              │
                              ▼
                         sched_init()
                              │
                              ▼
                         设置定时器中断
                              │
                              ▼
                         sti() 开中断
                              │
                              ▼
                         move_to_user_mode()
                              │
                              ▼
                         fork() 创建 init 进程
                              │
                              ▼
                         任务0 执行 pause()
                              │
                              ▼
                         schedule() 调度
```

### 2. 时钟中断调度流程

```
时钟中断发生
    │
    ▼
_timer_interrupt (汇编)
    │
    ▼
保存寄存器，调用 do_timer()
    │
    ▼
do_timer():
    │
    ├─ 更新进程时间统计
    │
    ├─ 处理软件定时器
    │
    ├─ 递减当前进程 counter
    │
    └─ 如果 counter == 0 且在用户态
         │
         ▼
       schedule()
         │
         ├─ 检查闹钟和信号
         │
         ├─ 选择下一个进程
         │
         └─ switch_to(next)
              │
              ▼
          任务切换完成
```

### 3. 进程睡眠与唤醒流程

```
进程调用 sleep_on()
    │
    ▼
设置状态为 TASK_UNINTERRUPTIBLE
    │
    ▼
加入等待队列
    │
    ▼
调用 schedule()
    │
    ▼
调度器选择其他进程运行
    │
    ... (进程睡眠中) ...
    │
其他进程调用 wake_up()
    │
    ▼
设置睡眠进程状态为 TASK_RUNNING
    │
    ▼
下次调度时，睡眠进程可能被选中运行
```

## 总结

### 调度算法特点

| 特点 | 说明 |
|------|------|
| **简单高效** | 算法简洁，代码量少 |
| **公平性** | 时间片轮转保证公平 |
| **响应性** | I/O 密集型进程获得更多时间片 |
| **非抢占内核** | 内核态不会被抢占 |

### 关键技术

1. **TSS 任务切换**：利用 x86 硬件实现任务切换
2. **时间片调度**：基于优先级的时间片分配
3. **时钟中断**：100Hz 定时器驱动调度
4. **等待队列**：实现进程睡眠和唤醒

### 局限性

1. 不支持对称多处理（SMP）
2. 内核不可抢占
3. 调度算法较简单，不适合复杂场景
4. 最大进程数限制（64个）

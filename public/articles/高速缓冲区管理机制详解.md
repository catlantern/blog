## 1. 概述

### 1.1 高速缓冲区的目的

高速缓冲区（Buffer Cache）是内核与块设备之间的中间层，主要目的：

```
+===================+     +===================+     +===================+
|    用户进程       |     |   高速缓冲区      |     |    块设备         |
|   read/write      |---->|   Buffer Cache    |---->|   磁盘/软盘       |
+===================+     +===================+     +===================+
                              |              |
                              v              v
                       +-----------+  +-----------+
                       | 减少磁盘  |  | 提高访问  |
                       | I/O 次数  |  | 速度      |
                       +-----------+  +-----------+
```

### 1.2 学习目的

| 目的 | 说明 |
|------|------|
| **理解文件交互流程** | 用户与磁盘文件交互的完整路径 |
| **理解缓冲区与磁盘关系** | 缓冲区如何缓存块设备数据 |
| **加深块设备驱动理解** | 块设备请求队列的工作方式 |
| **学习链表使用方法** | 哈希链表、循环双链表、单链表 |

### 1.3 高速缓冲区工作原理

```
读取数据流程：
+-------------------+
| 用户请求数据      |
+---------+---------+
          |
          v
+---------+---------+
| 检查缓冲区        |
| hash 查找         |
+---------+---------+
          |
    +-----+-----+
    | 存在      | 不存在
    v           v
+-------+   +-----------+
| 直接  |   | 从磁盘读取|
| 返回  |   | 存入缓冲区|
+-------+   +-----------+
                |
                v
            +-------+
            | 返回  |
            +-------+
```

## 2. 缓冲区数据结构

### 2.1 buffer_head 结构

定义在 **`include/linux/fs.h`**：

```c
struct buffer_head {
    char * b_data;              /* 指向数据块的指针（1024字节） */
    unsigned long b_blocknr;    /* 块号 */
    unsigned short b_dev;       /* 设备号（0表示空闲） */
    unsigned char b_uptodate;   /* 数据是否有效（最新） */
    unsigned char b_dirt;       /* 是否脏（已修改，需写回） */
    unsigned char b_count;      /* 使用计数 */
    unsigned char b_lock;       /* 是否锁定（正在I/O） */
    struct task_struct * b_wait;/* 等待该缓冲区的进程队列 */
    
    /* 哈希链表指针（双向链表） */
    struct buffer_head * b_prev;
    struct buffer_head * b_next;
    
    /* 空闲链表指针（双向循环链表） */
    struct buffer_head * b_prev_free;
    struct buffer_head * b_next_free;
};
```

### 2.2 缓冲区字段详解

| 字段 | 类型 | 说明 |
|------|------|------|
| **b_data** | char* | 指向 1024 字节的数据区 |
| **b_blocknr** | long | 逻辑块号 |
| **b_dev** | short | 设备号（主设备号<<8 \| 次设备号） |
| **b_uptodate** | char | 1=数据有效，0=数据无效 |
| **b_dirt** | char | 1=已修改需写回，0=干净 |
| **b_count** | char | 引用计数，0=空闲 |
| **b_lock** | char | 1=正在I/O，0=未锁定 |

### 2.3 全局变量

定义在 **`fs/buffer.c`**：

```c
struct buffer_head * start_buffer;      /* 缓冲区起始地址 */
struct buffer_head * hash_table[NR_HASH]; /* 哈希表（307项） */
static struct buffer_head * free_list;  /* 空闲链表头 */
static struct task_struct * buffer_wait;/* 等待缓冲区的进程队列 */
int NR_BUFFERS = 0;                     /* 缓冲区总数 */

#define NR_HASH 307                     /* 哈希表大小（质数） */
```

### 2.4 缓冲区内存布局

```
内存布局（buffer_init 初始化后）：
+===========================================================================+
|                                                                           |
|   低地址                                                                  |

|   +-------------------+                                                   |
|   | ...               |                                                   |
|   +-------------------+                                                   |
|   | end 符号          | <-- start_buffer 从这里开始                      |
|   +-------------------+                                                   |
|   | buffer_head[0]    |                                                   |
```

## 3. 为什么使用哈希表+循环双链表

### 3.1 两种链表的作用

Linux 0.11 使用了**两种链表结构**来管理缓冲区：

| 链表类型 | 结构 | 用途 |
|---------|------|------|
| **哈希链表** | 双向链表 | 快速查找特定缓冲区 |
| **空闲链表** | 双向循环链表 | LRU 算法管理空闲缓冲区 |

### 3.2 哈希表的作用

**问题**：如何快速找到特定设备和块号对应的缓冲区？

**解决方案**：使用哈希表 + 链表解决冲突

```c
#define _hashfn(dev,block) (((unsigned)(dev^block))%NR_HASH)
#define hash(dev,block) hash_table[_hashfn(dev,block)]
```

**哈希函数分析**：
- 输入：设备号 dev + 块号 block
- 计算：`(dev ^ block) % 307`
- 输出：哈希表索引（0-306）

**为什么选择 307？**
- 307 是质数，能更好地分散哈希值
- 减少哈希冲突

### 3.3 哈希链表结构

```
哈希表结构：
+===========================================================================+
|                                                                           |
|   hash_table[0] -----> [buf1] <---> [buf2] <---> [buf3] <---> NULL       |
|                                                                           |
|   hash_table[1] -----> [buf4] <---> [buf5] <---> NULL                    |
|                                                                           |
|   hash_table[2] -----> [buf6] <---> NULL                                 |
|                                                                           |
|   ...                                                                     |
|                                                                           |
|   hash_table[306] ----> [bufN] <---> NULL                                |
|                                                                           |
+===========================================================================+

每个哈希桶是一个双向链表：
+-------+     +-------+     +-------+
| buf A |<--->| buf B |<--->| buf C |
+-------+     +-------+     +-------+
  b_prev       b_next
  b_next       b_prev
```

### 3.4 循环双链表的作用

**问题**：如何高效实现 LRU（最近最少使用）算法？

**解决方案**：使用循环双向链表

```
空闲链表（循环双向链表）：
+===========================================================================+
|                                                                           |
|   +-------+     +-------+     +-------+     +-------+                    |
|   | free  |---->| buf 1 |---->| buf 2 |---->| buf 3 |----+               |
|   | list  |     |       |     |       |     |       |    |               |
|   +-------+<----+-------+<----+-------+<----+-------+<---+               |
|       |                                               ^                   |
|       +-----------------------------------------------+                   |
|                                                                           |
|   free_list 指向最近使用的缓冲区                                          |
|   free_list->b_prev_free 指向最久未使用的缓冲区（LRU淘汰候选）            |
|                                                                           |
+===========================================================================+
```

### 3.5 为什么是双向链表

**双向链表的优势**：

1. **O(1) 时间删除节点**
```
单向链表删除节点：需要遍历找到前驱节点，O(n)
双向链表删除节点：直接通过 prev 指针，O(1)

删除节点操作：
bh->b_prev->b_next = bh->b_next;
bh->b_next->b_prev = bh->b_prev;
```

2. **双向遍历能力**
```
可以从任意节点向前或向后遍历，方便实现 LRU 算法
```

### 3.6 为什么是循环链表

**循环链表的优势**：

1. **快速找到最久未使用的节点**
```
free_list->b_prev_free 直接指向最久未使用的节点
无需遍历整个链表
```

2. **简化边界处理**
```
没有 NULL 终止符，所有节点都相同处理
插入和删除操作无需特殊判断
```

3. **实现 LRU 算法**
```
新使用的缓冲区插入到链表头部（free_list 之前）
淘汰时从链表尾部（free_list->b_prev_free）选择
```

### 3.7 两种链表的关系

```
一个缓冲区同时存在于两个链表中：
+===========================================================================+
|                                                                           |
|   哈希链表（用于查找）          空闲链表（用于LRU）                       |
|                                                                           |
|   hash_table[i]                 free_list                                |
|        |                             |                                    |
|        v                             v                                    |
|   +-------+                     +-------+                                 |
|   | buf A |<------------------->| buf A |<------------------+             |
|   +-------+                     +-------+                 |             |
|        |                             |                    |             |
|        v                             v                    |             |
|   +-------+                     +-------+                 |             |
|   | buf B |<------------------->| buf B |<----------------+             |
|   +-------+                     +-------+                               |
|        |                             ^                                  |
|        v                             |                                  |
|   +-------+                     +-----+-----+                            |
|   | buf C |<------------------->| buf C |<----------------+             |
|   +-------+                     +-------+                 |             |
|        |                             |                    |             |
|       NULL                           +--------------------+             |
|                                                                           |
+===========================================================================+

注意：
- b_prev/b_next 用于哈希链表
- b_prev_free/b_next_free 用于空闲链表
- 两个链表完全独立
```

## 4. 缓冲区查找机制

### 4.1 find_buffer 函数

定义在 **`fs/buffer.c`**：

```c
static struct buffer_head * find_buffer(int dev, int block)
{
    struct buffer_head * tmp;

    /* 从哈希表对应桶开始遍历 */
    for (tmp = hash(dev, block); tmp != NULL; tmp = tmp->b_next)
        if (tmp->b_dev == dev && tmp->b_blocknr == block)
            return tmp;     /* 找到了 */
    return NULL;            /* 没找到 */
}
```

### 4.2 get_hash_table 函数

```c
struct buffer_head * get_hash_table(int dev, int block)
{
    struct buffer_head * bh;

    for (;;) {
        /* 1. 在哈希表中查找 */
        if (!(bh = find_buffer(dev, block)))
            return NULL;
        
        /* 2. 增加引用计数 */
        bh->b_count++;
        
        /* 3. 等待缓冲区解锁 */
        wait_on_buffer(bh);
        
        /* 4. 再次验证（防止睡眠期间被修改） */
        if (bh->b_dev == dev && bh->b_blocknr == block)
            return bh;
        
        /* 5. 验证失败，减少计数重试 */
        bh->b_count--;
    }
}
```

### 4.3 哈希查找流程图

```
哈希查找流程：
+-------------------+
| 输入: dev, block  |
+---------+---------+
          |
          v
+---------+---------+
| 计算 hash 索引    |
| idx = (dev^block) |
|      % NR_HASH    |
+---------+---------+
          |
          v
+---------+---------+
| tmp = hash_table  |
|      [idx]        |
+---------+---------+
          |
          v
+---------+---------+
| tmp == NULL ?     |
+---------+---------+
          |
    +-----+-----+
    | Yes       | No
    v           v
+-------+   +-----------+
| 返回  |   | 比较 dev  |
| NULL  |   | 和 block  |
+-------+   +-----+-----+
                  |
            +-----+-----+
            | 匹配      | 不匹配
            v           v
        +-------+   +-----------+
        | 返回  |   | tmp = tmp |
        | tmp   |   | ->b_next  |
        +-------+   +-----+-----+
                            |
                            v
                    +---------------+
                    | 继续循环      |
                    +---------------+
```

## 5. 缓冲区获取与释放

### 5.1 getblk 函数（核心）

`getblk()` 是获取缓冲区的核心函数：

```c
#define BADNESS(bh) (((bh)->b_dirt<<1)+(bh)->b_lock)

struct buffer_head * getblk(int dev, int block)
{
    struct buffer_head * tmp, * bh;

repeat:
    /* 1. 首先在哈希表中查找 */
    if (bh = get_hash_table(dev, block))
        return bh;

    /* 2. 在空闲链表中寻找合适的缓冲区 */
    tmp = free_list;
    do {
        /* 跳过正在使用的 */
        if (tmp->b_count)
            continue;
        
        /* 选择"最干净"的缓冲区 */
        if (!bh || BADNESS(tmp) < BADNESS(bh)) {
            bh = tmp;
            /* 如果既不脏也不锁，直接使用 */
            if (!BADNESS(tmp))
                break;
        }
    } while ((tmp = tmp->b_next_free) != free_list);

    /* 3. 没找到空闲缓冲区，等待 */
    if (!bh) {
        sleep_on(&buffer_wait);
        goto repeat;
    }

    /* 4. 等待缓冲区解锁 */
    wait_on_buffer(bh);
    if (bh->b_count)
        goto repeat;

    /* 5. 如果脏，先写回磁盘 */
    while (bh->b_dirt) {
        sync_dev(bh->b_dev);
        wait_on_buffer(bh);
        if (bh->b_count)
            goto repeat;
    }

    /* 6. 再次检查是否已被其他进程添加 */
    if (find_buffer(dev, block))
        goto repeat;

    /* 7. 初始化缓冲区 */
    bh->b_count = 1;
    bh->b_dirt = 0;
    bh->b_uptodate = 0;
    
    /* 8. 从旧队列移除，插入新位置 */
    remove_from_queues(bh);
    bh->b_dev = dev;
    bh->b_blocknr = block;
    insert_into_queues(bh);
    
    return bh;
}
```

### 5.2 BADNESS 宏分析

```c
#define BADNESS(bh) (((bh)->b_dirt<<1)+(bh)->b_lock)

/* BADNESS 值的含义：
 * 0: 干净且未锁定（最佳选择）
 * 1: 干净但锁定
 * 2: 脏但未锁定
 * 3: 脏且锁定（最差选择）
 */

优先级选择：
+--------+--------+----------+
| b_dirt | b_lock | BADNESS  |
+--------+--------+----------+
|   0    |   0    |    0     |  <-- 最佳
|   0    |   1    |    1     |
|   1    |   0    |    2     |
|   1    |   1    |    3     |  <-- 最差
+--------+--------+----------+
```

### 5.3 remove_from_queues 函数

```c
static inline void remove_from_queues(struct buffer_head * bh)
{
    /* 1. 从哈希链表中移除 */
    if (bh->b_next)
        bh->b_next->b_prev = bh->b_prev;
    if (bh->b_prev)
        bh->b_prev->b_next = bh->b_next;
    if (hash(bh->b_dev, bh->b_blocknr) == bh)
        hash(bh->b_dev, bh->b_blocknr) = bh->b_next;

    /* 2. 从空闲链表中移除 */
    if (!(bh->b_prev_free) || !(bh->b_next_free))
        panic("Free block list corrupted");
    bh->b_prev_free->b_next_free = bh->b_next_free;
    bh->b_next_free->b_prev_free = bh->b_prev_free;
    if (free_list == bh)
        free_list = bh->b_next_free;
}
```

### 5.4 insert_into_queues 函数

```c
static inline void insert_into_queues(struct buffer_head * bh)
{
    /* 1. 插入到空闲链表末尾（LRU头部） */
    bh->b_next_free = free_list;
    bh->b_prev_free = free_list->b_prev_free;
    free_list->b_prev_free->b_next_free = bh;
    free_list->b_prev_free = bh;

    /* 2. 插入到哈希链表 */
    bh->b_prev = NULL;
    bh->b_next = NULL;
    if (!bh->b_dev)
        return;
    bh->b_next = hash(bh->b_dev, bh->b_blocknr);
    hash(bh->b_dev, bh->b_blocknr) = bh;
    bh->b_next->b_prev = bh;
}
```

### 5.5 brelse 函数（释放缓冲区）

```c
void brelse(struct buffer_head * buf)
{
    if (!buf)
        return;
    
    /* 等待缓冲区解锁 */
    wait_on_buffer(buf);
    
    /* 减少引用计数 */
    if (!(buf->b_count--))
        panic("Trying to free free buffer");
    
    /* 唤醒等待缓冲区的进程 */
    wake_up(&buffer_wait);
}
```

## 6. 块设备与缓冲区交互

### 6.1 bread 函数（读取块）

```c
struct buffer_head * bread(int dev, int block)
{
    struct buffer_head * bh;

    /* 1. 获取缓冲区 */
    if (!(bh = getblk(dev, block)))
        panic("bread: getblk returned NULL\n");

    /* 2. 如果数据有效，直接返回 */
    if (bh->b_uptodate)
        return bh;

    /* 3. 发起读请求 */
    ll_rw_block(READ, bh);

    /* 4. 等待读取完成 */
    wait_on_buffer(bh);

    /* 5. 检查是否成功 */
    if (bh->b_uptodate)
        return bh;

    /* 6. 读取失败，释放缓冲区 */
    brelse(bh);
    return NULL;
}
```

### 6.2 ll_rw_block 函数

定义在 **`kernel/blk_drv/ll_rw_blk.c`**：

```c
void ll_rw_block(int rw, struct buffer_head * bh)
{
    unsigned int major;

    /* 1. 检查设备是否存在 */
    if ((major = MAJOR(bh->b_dev)) >= NR_BLK_DEV ||
        !(blk_dev[major].request_fn)) {
        printk("Trying to read nonexistent block-device\n\r");
        return;
    }

    /* 2. 创建请求并加入队列 */
    make_request(major, rw, bh);
}
```

### 6.3 make_request 函数

```c
static void make_request(int major, int rw, struct buffer_head * bh)
{
    struct request * req;
    int rw_ahead;

    /* 1. 处理预读/预写 */
    if (rw_ahead = (rw == READA || rw == WRITEA)) {
        if (bh->b_lock)
            return;     /* 如果已锁定，直接返回 */
        if (rw == READA)
            rw = READ;
        else
            rw = WRITE;
    }

    /* 2. 锁定缓冲区 */
    lock_buffer(bh);

    /* 3. 检查是否需要操作 */
    if ((rw == WRITE && !bh->b_dirt) || 
        (rw == READ && bh->b_uptodate)) {
        unlock_buffer(bh);
        return;
    }

repeat:
    /* 4. 获取空闲请求项 */
    /* 读请求可以使用全部请求项 */
    /* 写请求只能使用前 2/3 */
    if (rw == READ)
        req = request + NR_REQUEST;
    else
        req = request + ((NR_REQUEST * 2) / 3);

    /* 5. 查找空闲请求项 */
    while (--req >= request)
        if (req->dev < 0)
            break;

    /* 6. 没有空闲请求项 */
    if (req < request) {
        if (rw_ahead) {
            unlock_buffer(bh);
            return;
        }
        sleep_on(&wait_for_request);
        goto repeat;
    }

    /* 7. 填充请求 */
    req->dev = bh->b_dev;
    req->cmd = rw;
    req->errors = 0;
    req->sector = bh->b_blocknr << 1;  /* 块号转扇区号 */
    req->nr_sectors = 2;               /* 每块2个扇区 */
    req->buffer = bh->b_data;
    req->waiting = NULL;
    req->bh = bh;
    req->next = NULL;

    /* 8. 加入请求队列 */
    add_request(major + blk_dev, req);
}
```

### 6.5 电梯调度算法

```c
/* 电梯算法：按扇区顺序排序请求 */
#define IN_ORDER(s1, s2) \
((s1)->cmd < (s2)->cmd || (s1)->cmd == (s2)->cmd && \
((s1)->dev < (s2)->dev || ((s1)->dev == (s2)->dev && \
(s1)->sector < (s2)->sector)))

/* 插入请求时使用电梯算法排序 */
static void add_request(struct blk_dev_struct * dev, struct request * req)
{
    struct request * tmp;

    req->next = NULL;
    cli();
    if (req->bh)
        req->bh->b_dirt = 0;

    /* 如果队列为空，直接执行 */
    if (!(tmp = dev->current_request)) {
        dev->current_request = req;
        sti();
        (dev->request_fn)();
        return;
    }

    /* 按电梯算法插入合适位置 */
    for (; tmp->next; tmp = tmp->next)
        if ((IN_ORDER(tmp, req) ||
            !IN_ORDER(tmp, tmp->next)) &&
            IN_ORDER(req, tmp->next))
            break;
    req->next = tmp->next;
    tmp->next = req;
    sti();
}
```

## 7. LRU 算法实现

### 7.1 LRU 原理

LRU（Least Recently Used，最近最少使用）算法：

- **最近使用的缓冲区**：放在链表头部
- **最久未使用的缓冲区**：放在链表尾部
- **淘汰时**：选择链表尾部的缓冲区

### 7.2 空闲链表的 LRU 实现

```
空闲链表 LRU 实现：
+===========================================================================+
|                                                                           |
|   初始状态（所有缓冲区空闲）：                                            |
|                                                                           |
|   free_list --> [buf1] <--> [buf2] <--> [buf3] <--> [buf4]               |
|                   ^                                            |          |
|                   +--------------------------------------------+          |
|                                                                           |
|   free_list->b_prev_free 指向 buf4（最久未使用）                         |
|                                                                           |
+===========================================================================+
|                                                                           |
|   使用 buf2 后：                                                          |
|                                                                           |
|   1. 从原位置移除 buf2                                                   |
|   2. 插入到 free_list 之前                                               |
|                                                                           |
|   free_list --> [buf1] <--> [buf3] <--> [buf4] <--> [buf2]               |
|                   ^                                            |          |
|                   +--------------------------------------------+          |
|                                                                           |
|   现在 buf4 是最久未使用的                                               |
|                                                                           |
+===========================================================================+
|                                                                           |
|   淘汰时：                                                                |
|                                                                           |
|   从 free_list->b_prev_free 开始查找（buf4）                             |
|   选择 BADNESS 最小的缓冲区                                              |
|                                                                           |
+===========================================================================+
```

## 9. 总结

### 9.1 关键设计要点

| 设计 | 原因 |
|------|------|
| **哈希表** | O(1) 平均查找时间，快速定位缓冲区 |
| **双向链表** | O(1) 删除节点，支持双向遍历 |
| **循环链表** | 快速找到 LRU 候选，简化边界处理 |
| **BADNESS** | 优先选择干净、未锁定的缓冲区 |
| **读优先** | 读请求可使用全部请求项，写只能用 2/3 |
| **电梯算法** | 按扇区顺序处理请求，减少磁头移动 |

### 9.2 关键函数总结

| 函数 | 文件 | 作用 |
|------|------|------|
| buffer_init() | buffer.c | 初始化缓冲区 |
| getblk() | buffer.c | 获取缓冲区（核心） |
| brelse() | buffer.c | 释放缓冲区 |
| bread() | buffer.c | 读取数据块 |
| find_buffer() | buffer.c | 哈希查找 |
| ll_rw_block() | ll_rw_blk.c | 底层读写 |
| make_request() | ll_rw_blk.c | 创建请求 |
| add_request() | ll_rw_blk.c | 添加请求（电梯算法） |

### 9.3 为什么是哈希表+循环双链表

**总结**：

1. **哈希表 + 双向链表**：
   - 快速查找：O(1) 平均时间
   - 解决冲突：链地址法
   - 高效删除：双向链表 O(1)

2. **循环双向链表**：
   - LRU 实现：头部插入，尾部淘汰
   - 快速定位：free_list->b_prev_free 直接指向最久未使用
   - 边界简化：无 NULL 终止，统一处理

3. **两种链表独立**：
   - 哈希链表：用于快速查找
   - 空闲链表：用于 LRU 管理
   - 一个缓冲区同时存在于两个链表中
